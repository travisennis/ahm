# Add opt-in task work completion handoff

## Blocker

Awaiting design decisions on:
- Completion mechanism: separate prompt, `--complete` flag, or `task work complete` subcommand.
- Whether `ahm` itself runs `task complete` or delegates to the agent.
- How incomplete acceptance notes, failed verification, or strict acceptance settings are surfaced.
- Whether resumable session support (task 056) is a prerequisite.
- Behavior for tasks already `In Progress` before `task work` started.

These need agreement before implementation can begin. See the Design Questions section below.

## Summary

Explore and implement an opt-in task completion handoff that asks the delegated
agent to fill acceptance notes, run verification, and complete the task through
`ahm` when work is done.

## Background

The MVP `task work` prompt tells the external agent to follow the workflow, but
`ahm` does not orchestrate or enforce completion after the handoff. The shell
workflow asks the agent to complete the task before committing. This could be a
useful addition, but it needs a clear boundary so `ahm` does not silently mark
work complete or hide failed validation.

## Design Questions

- Should completion be a separate prompt after review, a `--complete` flag, or
  a separate `task work complete` command?
- Should `ahm` itself ever run `task complete`, or should the delegated agent be
  instructed to run it?
- How should incomplete acceptance notes, failed verification, or strict
  acceptance settings be surfaced?
- Should this require resumable session support first?
- How should completion handoff behave for tasks that were already `In Progress`
  before `task work` started?
