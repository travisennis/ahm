# Require workflow status in delegated task work handoff

## Summary

`ahm task work` delegates implementation, review, completion, and optional commit handoff to an external agent, but its built prompts only say to run task-specific verification and mark the task complete. They do not require `ahm --check workflow status` before commit handoff, so workflow-record drift can be committed after a successful implementation and preflight review. Update the task-work prompts and shipped preflight guidance so the normal delegated path surfaces workflow validation warnings before completion is considered ready.

Written against commit `5cbd271`.

## Evidence

- `internal/ahm/task_work.go:161` - the implementation prompt asks the agent to load task context and inspect the task.
- `internal/ahm/task_work.go:163` - the same prompt says to fill Acceptance Notes, run required verification, and mark the task complete, but does not mention `ahm status` or workflow-scoped validation.
- `internal/ahm/task_session.go:130` - the ref-backed commit prompt only asks the agent to make sure the task is marked completed before committing.
- `internal/ahm/task_session.go:136` - the legacy commit prompt likewise asks only that the task is marked completed and included in the commit.
- `internal/templates/workflow/preflight-SKILL.md:205` - the preflight workflow reruns validation chosen by the reviewer, but does not make ahm workflow validation a required completion check.

## Fix Direction

Update `buildTaskWorkPrompt`, `buildTaskWorkCommitPrompt`, and the shipped preflight template so delegated agents run `ahm --check workflow status` after task completion and before commit handoff. Keep this scoped to workflow validation rather than default `ahm status`, because link-scope warnings currently include known false positives tracked by task 161. Update docs and fixture tests that assert task-work prompt text. This is a prompt/template behavior change, so bump template metadata only if the embedded template changes require it under the existing versioning convention.
