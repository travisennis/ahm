# Add live ahm context preflight procedure scope

## Summary

Add `ahm context preflight`: the pre-commit review-readiness procedure
currently installed as the `preflight` skill, emitted from the binary with
the change size pre-computed and an ahm-managed completion checklist added.

## Problem

The procedure lives in
`internal/templates/workflow/preflight-SKILL.md` (~223 lines), installed to
`.agents/skills/preflight/SKILL.md`. Its first step is running
`git diff --stat` / `git status --short` to pick an effort scale
(XS/S = one pass, M = two, L/XL = three). Per the 156a ADR it becomes a
scoped context, and per the design decision (2026-07-01) its role sharpens:
preflight is how an agent reviews work done on an ahm-managed task —
checking not just the diff but that everything ahm manages was closed out
(acceptance notes, ExecPlans, docs).

## Design

Output = procedure instructions (adapted from the SKILL.md body) with live
state:

- Change-size summary from read-only git commands (`git diff --stat`,
  `git status --short`, including untracked files per the skill's own
  caveat) and the derived effort scale, so the prompt arrives with the
  pass count already chosen. Follow the `readGitContext` precedent in
  `internal/ahm/context.go`; read-only git only — the safety boundary that
  ahm never runs mutating git operations holds.
- In Progress task lines (the work presumably under review), so the
  reviewer can connect the diff to the task record.
- A managed-work completion checklist, new in this adaptation: task
  acceptance notes checked off, `exec_plan` (if any) updated or completed,
  documentation touched when the change requires it, no hand-edited
  generated indexes, task state moved via `ahm task` commands.

Keep the skill's core structure: scaled sequential passes (rules
conformance, correctness/source-of-truth, overengineering), then synthesize
and apply worthwhile fixes; goal is the smallest clear diff.

Mechanics: same scope-registry pattern as 156c/156d in
`internal/ahm/context.go`; instruction body and live sections as separate
fields for JSON parity.

## Relevant Files
