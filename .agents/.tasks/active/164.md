# Render agent instructions for the active storage mode

## Problem

Task 145 made the managed-work references and agent guidance storage-mode aware, but the resulting copy often tells agents to look under either `.agents/` or `.ahm/`. That wording is accurate as a fallback, but it is noisy in live `ahm` output because `ahm` already knows the repository's current storage mode.

Live instruction surfaces should tailor path guidance to the active repository mode. Legacy repositories should get direct `.agents/...` paths. Ref-backed repositories should get direct `.ahm/...` paths. Only generic documentation or migration docs should need paired-path wording.

## Relevant Files

- `internal/ahm/context.go` - renders scoped managed-work references and live briefings.
- `internal/ahm/agents.go` - renders `ahm agents suggestions` advisory output.
- `internal/templates/workflow/` - embedded task, research, ExecPlan, docs, and managed skill guidance templates.
- `internal/ahm/workflow_paths.go` - resolves storage-mode-specific workflow paths.
- `docs/references/workflow-spec.md`, `docs/guides/workflow-upgrades.md`, `ARCHITECTURE.md` - document the durable behavior and upgrade impact.
- `internal/ahm/*context*_test.go`, `internal/ahm/*agents*_test.go`, and template tests - cover rendered guidance output.

## Fix Direction

Make live `ahm` instruction output use the repository's resolved workflow paths instead of static either/or wording. Prefer injecting storage-mode-specific path values into rendered references over duplicating whole templates. Keep storage-mode-neutral wording only where the output is intentionally generic or applies before a repository mode can be known.

Review scoped contexts for task, research, ExecPlan, ADR, and docs guidance; `ahm prime`; managed skills; and `ahm agents suggestions`. Update tests so legacy-mode output names `.agents/...` directly and ref-mode output names `.ahm/...` directly, without pairing both roots in ordinary live guidance.

## Acceptance Notes

- [ ] `ahm context task`, `ahm context research`, `ahm context plan`, `ahm context docs`, and `ahm prime` render direct path guidance for the active storage mode.
- [ ] `ahm agents suggestions` and managed skill guidance avoid unnecessary `.agents/`-or-`.ahm/` phrasing when the repository mode is known.
- [ ] Generic docs still explain both storage modes where that distinction is durable user-facing behavior.
- [ ] Tests cover legacy and ref-backed rendering so regressions in path tailoring are caught.
- [ ] Template version and upgrade notes are updated if embedded workflow templates change.
