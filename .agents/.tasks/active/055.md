# Add optional task work review orchestration

## Blocker

Awaiting design decisions on:
- Review orchestration mechanism: flag on `task work`, separate subcommand, or workflow profile.
- Whether the review agent is the same as the work agent or separately selectable.
- Non-`cake` agent support for review.
- Behavior when review pass fails, produces no feedback, or suggests no changes.
- Whether deslop should be the only supported review mode initially.

These need agreement before implementation can begin. See the Design Questions section below.

## Summary

Explore and implement an opt-in review orchestration step for `ahm task work`.
The goal is to let a task work handoff run an independent review pass, such as
the existing deslop workflow, without making review orchestration part of the
default MVP behavior.

## Background

`scripts/task-workflow.sh` currently runs a separate `cake --no-session
--skills deslop` review after the initial work session, then feeds that review
back to the original session. ADR 006 intentionally kept this out of the MVP so
`ahm task work` could establish a clear delegation boundary first.

## Design Questions

- Should review orchestration be a flag on `task work`, a separate subcommand,
  or a configured workflow profile?
- Should the review agent always be the same agent as the work agent, or should
  it be separately selectable?
- Can this support non-`cake` agents without requiring provider-specific session
  parsing?
- What should happen when the review pass fails, produces no feedback, or
  suggests no changes?
- Should deslop be the only supported review mode initially?
