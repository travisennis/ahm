# Add Claude Code agent support to ahm task work

## Blocker

Awaiting a recorded decision on the Claude Code CLI contract: exact JSON
session field names, resume flags, and whether `--review` is supported or
intentionally left unsupported. The `claude` executable exists locally at
grooming time, so this can be unblocked by a short CLI verification note.

## Problem

`ahm task work` supports `cake`, `codex`, and `cursor`, but not Claude Code.
Claude Code's CLI (`claude`) supports headless print mode with JSON output and
session resume, so it can participate in full orchestration (sessions, review,
completion handoff, commit handoff) the same way `cake` does today.

## Relevant Files

- `internal/ahm/task_commands.go` — `taskWorkAgent` struct,
  `parseTaskWorkAgent`, `taskWorkWithSession`, cake session/review parsers.
- `internal/ahm/task_commands_test.go` — agent selection, dry-run, and
  orchestration tests.
- `docs/cli.md` — `task work` supported-agents table, flags, examples, and
  agent selection precedence.
- `docs/adr/006-task-work-agent-delegation.md` — enumerates the supported
  agent set and session-capable agents.

## Fix Direction

- Add a `claude` case to `parseTaskWorkAgent`:
  - executable `claude`
  - work args along the lines of `-p --output-format json <prompt>`
  - resume args along the lines of `-p --resume <sessionID> --output-format json <prompt>`
- Claude Code's `--output-format json` response carries `session_id` and
  `result` fields, the same shape `parseCakeSessionID` and
  `parseCakeReviewFeedback` already parse. Reuse or generalize those parsers
  rather than duplicating them. Verify the exact flags and JSON field names
  against the installed `claude` CLI before locking them in.
- Decide the review invocation: a fresh `claude -p --output-format json
  <review prompt>` run is already an independent session, so it can mirror
  cake's review pass; pick the closest equivalent of cake's `--no-session
  --skills deslop` review invocation and document the choice.
- Set `supportsSessions` and (if review is included) `supportsReview` so
  `--review`, `--complete`, and `--commit` work without warnings.
- Update the `--agent` flag help text, the unsupported-agent error message,
  and `default_work_agent` validation surface if any.
- Update `docs/cli.md` (supported-agents table, precedence list, examples) and
  amend `docs/adr/006-task-work-agent-delegation.md` per the supersession
  workflow, since the ADR enumerates the supported agent set.

## Acceptance Notes

- [ ] `ahm task work <id> --agent claude` invokes `claude` with JSON output
      and captures the session ID.
- [ ] `--review`, `--complete`, and `--commit` run for `claude` without
      non-session warnings.
- [ ] `--dry-run` previews the claude executable and args without invoking it.
- [ ] Unit tests cover agent selection, arg construction, and session/review
      parsing for claude.
- [ ] `docs/cli.md` and ADR 006 updated; `just ci` passes.
