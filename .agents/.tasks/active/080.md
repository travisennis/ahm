# Rewrite ADR template for MADR and update agent suggestions

## Summary

Rewrite the embedded ADR workflow template for MADR-only guidance and update
the advisory AGENTS.md suggestions so agents route ADR work through `ahm adr`
commands.

## Background

Design per ADR 009 (task 074). `internal/templates/workflow/adr-README.md`
currently documents the custom Nygard-style format; after this task it
documents only the constrained MADR profile and the `ahm adr` command
workflow. This is a managed-template content change with upgrade
implications.

## Implementation Notes

- Rewrite `internal/templates/workflow/adr-README.md`: constrained MADR
  template and section guidance, when to write an ADR (keep the existing
  trigger rules), numbering/naming, the status set with
  `superseded by ADR-NNN`, supersession rules including how partial
  supersession is expressed per ADR 009, `ahm adr` command usage
  (create/list/show/accept/reject/deprecate/supersede/migrate), and a note
  that `docs/adr/index.md` is generated and never hand-edited.
- Update `agentSuggestions` in `internal/templates/templates.go`: the
  `ahm-owned-files` suggestion must cover `docs/adr/index.md`; workflow
  routing should direct ADR creation and status changes through `ahm adr`.
  The starter AGENTS.md renders from these suggestions, so keep it coherent.
- Bump `templates.Version` and document the template content change and
  upgrade behavior in `docs/upgrades.md` (upgrade overwrites
  `docs/adr/README.md` following the existing modified-file conflict
  semantics).
- Keep embedded `workflow/TASKS.md` ADR guidance and this repo's `AGENTS.md`
  and `README.md` consistent with the new format (the "superseded in part"
  wording lives in several places).
- Template change rule: run `go test ./internal/templates ./internal/ahm`
  before `just ci`.

## Relevant Files

- `internal/templates/workflow/adr-README.md`
- `internal/templates/workflow/TASKS.md`
- `internal/templates/templates.go`
- `docs/upgrades.md`
- `AGENTS.md`, `README.md`
