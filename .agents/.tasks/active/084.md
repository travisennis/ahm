# Upgrade cursor agent to full task work orchestration

## Problem

`ahm task work --agent cursor` is basic handoff only: it runs
`cursor-agent -p --output-format text <prompt>` without session capture, so
`--review`, `--complete`, and `--commit` print warnings and are skipped. The
Cursor CLI supports JSON output and chat resume, so cursor can be upgraded to
full orchestration like `cake`.

## Relevant Files

- `internal/ahm/task_commands.go` — `cursor` case in `parseTaskWorkAgent`,
  `taskWorkAgent` struct, `taskWorkWithSession`.
- `internal/ahm/task_commands_test.go` — agent selection and orchestration
  tests.
- `docs/cli.md` — `task work` supported-agents table (cursor row currently
  says "Basic handoff only" / "Not supported").
- `docs/adr/006-task-work-agent-delegation.md` — says session-capable agents
  are "currently cake".

## Fix Direction

- Verify the installed `cursor-agent` headless interface before locking in
  flags. As of recent versions:
  - `cursor-agent -p --output-format json <prompt>` returns a JSON result that
    includes the chat/session identifier.
  - `cursor-agent -p --resume <chatID> <prompt>` resumes a prior chat.
- Switch the work invocation from `--output-format text` to
  `--output-format json`, add a cursor-specific `parseSessionID` for whatever
  field name the CLI uses (verify: it may be `session_id`, `chat_id`, or a
  stream-json event), and wire `resumeArgs`.
- Decide review support: cursor has no deslop equivalent, so an independent
  review pass would be a fresh `cursor-agent -p --output-format json
  <review prompt>` whose result text is the feedback. If that is not reliable,
  ship sessions + `--complete` + `--commit` first and leave `supportsReview`
  false with the existing warning; record the choice in the ADR amendment.
- Note the output-format change is user-visible for plain
  `ahm task work --agent cursor` runs (JSON instead of text on stdout);
  preserve the tee-to-terminal behavior and call the change out in docs.
- Update `docs/cli.md` (cursor row in the supported-agents table, examples)
  and amend `docs/adr/006-task-work-agent-delegation.md` since the
  session-capable agent set changes.

## Acceptance Notes

- [ ] `ahm task work <id> --agent cursor` captures a session/chat ID from
      cursor-agent JSON output.
- [ ] `--complete` and `--commit` resume the cursor session instead of
      warning.
- [ ] `--review` either runs a cursor review pass or the documented warning
      behavior is intentional and recorded in ADR 006.
- [ ] `--dry-run` previews the cursor-agent executable and args without
      invoking it.
- [ ] Unit tests cover cursor arg construction and session-ID parsing.
- [ ] `docs/cli.md` and ADR 006 updated; `just ci` passes.
