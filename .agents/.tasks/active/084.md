# Upgrade cursor agent to full task work orchestration

## Decision

Use Cursor Agent print mode with realtime JSON streaming:

- Work args: `cursor-agent -p --output-format stream-json --trust <prompt>`.
- Resume args: `cursor-agent -p --output-format stream-json --trust --resume
  <sessionID> <prompt>`.
- Session ID source: first `system` event with `subtype: "init"` and
  `session_id`.
- Final result source: final `result` event with `result`, `is_error`,
  `session_id`, and usage fields.
- Review support should be included by running a fresh Cursor Agent
  stream-json review prompt in ask mode and parsing the final `result` event,
  then resuming the original session with that feedback.

Local verification on 2026-06-12 with Cursor Agent
2026.06.12-01-15-52-7244546 confirmed `cursor-agent` is on PATH and
`cursor-agent -p --output-format stream-json --mode ask --trust <prompt>`
emits `system/init`, `user`, `assistant`, and final `result` events. The
`system/init` and `result` events both include the same `session_id`.

Official docs were checked at https://cursor.com/docs, but the site renders as
a client-side app and the specific CLI reference was not machine-fetchable from
the current environment. The installed CLI help is the verified source for the
exact contract below.

## Problem

`ahm task work --agent cursor` is basic handoff only: it runs
`cursor-agent -p --output-format text <prompt>` without session capture, so
`--review`, `--complete`, and `--commit` print warnings and are skipped. The
Cursor CLI supports JSON output and chat resume, so cursor can be upgraded to
full orchestration like `cake`.

## Relevant Files

- `internal/ahm/task_commands.go` — `cursor` case in `parseTaskWorkAgent`,
  `taskWorkAgent` struct, `taskWorkWithSession`.
- `internal/ahm/task_commands_test.go` — agent selection and orchestration
  tests.
- `docs/cli.md` — `task work` supported-agents table (cursor row currently
  says "Basic handoff only" / "Not supported").
- `docs/adr/006-task-work-agent-delegation.md` — says session-capable agents
  are "currently cake".

## Fix Direction

- Switch the work invocation from `--output-format text` to
  `--output-format stream-json`, add a Cursor JSONL parser that scans
  `system/init` for `session_id`, and wire `resumeArgs`.
- Add review support with a fresh `cursor-agent -p --output-format stream-json
  --mode ask --trust <review prompt>` run. Parse feedback from the final
  `result` event.
- Note the output-format change is user-visible for plain
  `ahm task work --agent cursor` runs (stream-json instead of text on stdout);
  preserve the tee-to-terminal behavior and call the change out in docs.
- Update `docs/cli.md` (cursor row in the supported-agents table, examples)
  and amend `docs/adr/006-task-work-agent-delegation.md` since the
  session-capable agent set changes.

## Acceptance Notes

- [ ] `ahm task work <id> --agent cursor` captures a session ID from the
      cursor-agent `system/init` stream-json event.
- [ ] `--complete` and `--commit` resume the cursor session instead of

- [ ] `--review` runs a Cursor Agent review pass and resumes the captured

- [ ] `--dry-run` previews the cursor-agent executable and args without
      invoking it.
