# Add committed .ahm migration and topology regression coverage

## Problem

The ref-backed happy-path suite passed while missing the clone, worktree,
divergence, attachment, initial-remote, and clean-config cases that exposed the
release blockers. The committed `.ahm/` transition needs end-to-end tests around
the repository topologies and legacy committed migration states users actually
have. Ref-backed fixtures do not need to survive because that mode is unreleased.

## Relevant Files

- `internal/ahm/records_migrate_test.go`
- `internal/ahm/records_workflow_test.go`
- `internal/ahm/prime_test.go`
- install, root, validation, and CLI tests
- `migrate-issues.md`

## Fix Direction

Replace ref-sync scenarios with fixture-backed migration coverage for legacy
committed, partially migrated, fresh-clone, ordinary branch, and linked-worktree
repositories. Include non-Markdown attachments, custom record content, dirty
source records, dry-run, conflicts, interruption/resume, Git index/HEAD
invariants, and deterministic index regeneration. Remove remote and divergence
tests with the unreleased synchronization behavior they covered.

Cover branch checkout with stale ignored workflow indexes, normal integration
of records changed on separate branches, and visible add/add conflict behavior
when branches independently create the same sequential task ID. The tests do
not need to invent cross-branch allocation or repository-global backlog state.

## Acceptance Notes

- [ ] Every reproduced blocker in `migrate-issues.md` has a regression test or is made impossible by removed behavior.
- [ ] Legacy committed repositories retain all source files through migration.
- [ ] Fresh clones and linked worktrees use normal committed-file semantics.
- [ ] Branch checkout regenerates workflow indexes from the checked-out source records.
- [ ] Concurrent same-ID creation is documented or tested as a normal visible Git conflict.
- [ ] Workflow indexes remain ignored while `docs/adr/index.md` remains committed.
- [ ] Migration and routine commands preserve project `HEAD`, index, branches, and unrelated worktree changes.
- [ ] Successful routine operations leave committed config and source records clean unless the user changed workflow state.
- [ ] Focused tests and the full CI suite pass.
