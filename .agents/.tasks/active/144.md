# Integrate ref-backed records with workflow write paths

## Blocker

Blocked until tasks 139, 140, and 141 provide the private-ref plumbing, records
metadata model, and records command behavior this integration depends on.

## Summary

Integrate record snapshot behavior with task, research, and ExecPlan write paths while keeping generated indexes local-only and preserving legacy committed-record behavior until migration.

## Relevant Records

- ADR: `docs/adr/013-use-ref-backed-workflow-record-storage.md`
- ExecPlan: `.agents/exec-plans/active/138-ref-backed-workflow-records.md`

## Work Direction

Wire ref-backed record behavior into task, research, and ExecPlan writes without
changing legacy behavior for repositories that have not opted in. Ensure
generated indexes remain local-only. In ref-backed mode, source records live
under `.ahm/`; project-owned `.agents/` content remains committed and outside
the record snapshot set. Review `ahm task work` commit handoff so gitignored
task state is not swept into normal project commits after migration.

## Acceptance Notes

- [ ] Legacy committed-record repositories behave as before.
- [ ] Ref-backed repositories snapshot `.ahm/` source records after supported
      task, research, and ExecPlan mutations.
- [ ] Generated indexes are regenerated locally but excluded from ref snapshots.
- [ ] `ahm task work` commit handoff remains coherent when task files are
      gitignored and synced through records refs.
