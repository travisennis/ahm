# Link validator flags markdown links inside code spans

## Problem

The `links` validator parses markdown link syntax inside inline code spans
(and possibly fenced code blocks). Example: `.agents/.tasks/completed/157.md`
line 25 quotes `[ADRs](adr/index.md)` in backticks as a before/after example,
and `ahm status` reports `markdown_link_missing` for `adr/index.md` resolved
relative to the task directory. Quoted example links in task bodies, ADRs,
and docs are common; they are text, not navigation.

## Fix

Skip link extraction inside inline code spans and fenced code blocks in the
`links` and `project-docs` validators. Add regression tests with quoted
links in both span and fence form.

Note for task 160b: the new docs-check validators must inherit this
behavior from the start.

## Verification

`ahm status` reports no findings for `.agents/.tasks/completed/157.md`; a
real broken link outside a code span is still reported.
