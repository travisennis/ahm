# Add ahm records sync command surface

## Summary

Add ahm records status, pull, push, sync, and doctor behavior for explicit GitHub-backed refs/ahm/records synchronization and diagnostics.

## Relevant Records

- ADR: `docs/adr/013-use-ref-backed-workflow-record-storage.md`
- ExecPlan: `.agents/exec-plans/active/138-ref-backed-workflow-records.md`

## Work Direction

Add an explicit `ahm records` command family for ref-backed workflow records.
The command surface should include status reporting, pull/fetch,
push, sync, and diagnostics for missing remotes, unsupported remotes,
credential failures, non-fast-forward remote refs, and stale local state. Initial
remote support is GitHub only.

## Acceptance Notes

- [ ] `ahm records status` reports local/remote records state without mutating
      files.
- [ ] `ahm records pull`, `push`, and `sync` work against GitHub
      `refs/ahm/records`.
- [ ] JSON/plain/text output shapes are documented and tested.
- [ ] Remote failures produce actionable diagnostics.
