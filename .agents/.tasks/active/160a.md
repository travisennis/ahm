# Write ADR for ahm docs check and project-docs deprecation

## Summary

Record the decision to add the `ahm docs` command group with `docs check` as
the front door for project documentation validation, expand the checked
surface and finding codes, deprecate the `--check project-docs` scope value,
and add the `projectDocs` config block to `.agents/ahm.json`.

## Decisions to record

- New `ahm docs check` command runs the (expanded) `project-docs` validators
  through the existing validation engine; one implementation, and `status`/
  `doctor --check project-docs` becomes a deprecated alias that warns and
  points at `ahm docs check`. Removal of the alias is explicitly deferred.
- Scanned surface expands to include root `AGENTS.md`, `CLAUDE.md`, and
  nested `AGENTS.md` files (today the routing layer is not scanned at all).
- New finding codes: `project_doc_link_not_portable` (error; `file://`,
  absolute, or home-directory link targets), `entry_point_over_budget`
  (warning; crude line count on root AGENTS.md, skipping CLAUDE.md when it
  is an import/symlink), `doc_unindexed` (warning; generalizes the
  design-docs index check to any docs subdirectory with an `index.md` —
  `design_doc_unindexed` keeps its code for compatibility),
  `doc_review_suggested` (info; diff mode), `doc_map_target_missing`
  (error; docMap entry points at a nonexistent doc).
- `projectDocs` block in `.agents/ahm.json`: `entryPointBudget` (default
  150), `exclude` globs, `docMap` (path globs → docs to review). All
  optional; zero config runs static checks with defaults.
- Diff mode (`--staged`, `--diff <range>`) is advisory-only and read-only
  (git reads within the existing safety boundary); exit code is unaffected
  by `doc_review_suggested` findings. `--strict` promotes warnings to
  errors for CI.
- `ahm docs check` never calls models and never edits files, matching the
  existing `project-docs` contract.
