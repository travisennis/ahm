# Add read-only ahm prime session briefing command

## Blocker

Blocked until the ADR from task 156a is accepted.

## Summary

Add `ahm prime`, the canonical read-only session-start command for coding
agents: warnings, live backlog state, and the managed-work routing table.
Change unscoped `ahm context` to a usage error that points at `ahm prime`.

## Problem

Today unscoped `ahm context` prints a short status summary (counts plus a
command list). The design decision (see task 156 and the 156a ADR) is a
state-rich, instruction-light prime modeled on beadwork's `bw prime`, under
the `ahm prime` name that ADR 013 already reserves for the session-start
command. Task 143 (ref-backed records) later layers a records-sync step onto
this command; this task builds only the read-only briefing.

## Output Design

Sections in order (omit empty sections):

1. Dirty-worktree warning, only when `git status` shows changes: warn the
   agent to resolve uncommitted changes with the user before starting new
   work. Reuse `readGitContext` in `internal/ahm/context.go` (read-only git
   only тАФ respect the safety boundary).
2. Root, workflow/template version, validation findings тАФ as today's
   `contextReport`.
3. `## In Progress` тАФ full task lines for In Progress tasks.
4. `## Ready` тАФ full task lines, capped at 5, with an overflow pointer
   ("run `ahm task ready` for N more").
5. Blocked and Open counts (counts only, with the commands to expand).
6. `## Managed Work Intake` тАФ the routing table that currently lives in
   AGENTS.md's "Managed Work Intake With `ahm`" section, moved here:
   - Work a task тЖТ `ahm context task`, then `ahm task show <id>`
   - ExecPlan work тЖТ `ahm context plan`
   - ADR work тЖТ `ahm context adr`
   - Research notes тЖТ `ahm context research`
   - Documentation work тЖТ `ahm context docs`
   - Groom the backlog тЖТ `ahm context groom`
   - Audit for improvements тЖТ `ahm context improve`
   - Review finished work тЖТ `ahm context preflight`
   Plus two fixed lines: "ahm manages work records, not implementation;
   after intake, classify the implementation under the project's own
   workflow routing (AGENTS.md)" and the plans-are-scratch nudge: "Before
   executing a multi-step plan, materialize it as ahm tasks (or an
   ExecPlan) тАФ plans in context die at compaction; records survive."
7. Useful commands, as today.

The groom/improve/preflight routing entries land with tasks 156c-156e; if
this task ships first, include the entries only once those scopes exist
(coordinate ordering тАФ simplest is to list only existing scopes and extend
the table in 156c-156e).

## Behavior Changes

- New `ahm prime` command wired in `internal/ahm/cli.go`.
- Unscoped `ahm context` returns a usage error: session briefing moved to
  `ahm prime`; list valid scopes. Breaking CLI change recorded in the 156a
  ADR. Update `docs/references/cli/commands.md` and `docs/cli.md`.
- JSON parity: define a prime report struct (extend or parallel
  `contextReport`) and render text from it, following the `textRenderer`
  pattern in `internal/output` (see commit ff15056). `--plain` supported.
