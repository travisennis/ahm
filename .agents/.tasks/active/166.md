# Add task work model override flag

## Problem

`ahm task work <id>` can select the external coding-agent CLI with `--agent`, but it cannot select the model used by that agent. Users currently have to rely on each provider CLI's ambient configuration, which makes one-off task runs harder to reproduce and prevents `ahm --dry-run task work` from showing the intended model.

Add a `--model <name>` flag to `ahm task work` so a user can request the model for the selected agent for a single invocation.

## Relevant Files

- `internal/ahm/task_commands.go` - Cobra wiring, usage, examples, and flag validation for `task work`.
- `internal/ahm/task_work.go` - task-work argument flow, dry-run preview, prompt handoff, and orchestration entry point.
- `internal/ahm/task_agents.go` - supported agent definitions and provider-specific argument builders.
- `internal/ahm/task_session.go` - review, feedback resume, and commit handoff argument paths that must stay consistent with model selection semantics.
- `internal/ahm/task_commands_test.go`, `internal/ahm/task_commands_golden_test.go`, and `internal/ahm/testdata/agents/` - focused task-work, dry-run, parser, and golden transcript coverage.
- `docs/references/cli/task-commands.md`, `docs/cli.md`, `docs/guides/testing.md`, `docs/references/workflow-spec.md`, and ADR 006 - documented task-work contract and external agent testing expectations.

## Fix Direction

Thread an optional model value through `taskWorkArgs` and the selected `taskWorkAgent` invocation builders. Define and document how each supported agent maps model selection to its CLI flags, and make unsupported or empty model values fail as usage errors only when the selected agent cannot accept a model override.

Decide whether `--model` applies only to the initial implementation session or also to review, feedback resume, and commit handoff. Prefer a consistent per-invocation rule unless a provider CLI makes that impossible, and document the rule clearly. Include the selected model in dry-run output so users can verify the exact handoff before invoking an external agent.

Update ADR 006 or add a focused follow-up ADR before code changes if the implementation changes the durable agent-delegation boundary.

## Acceptance Notes

- [ ] `ahm task work <id> --model <name>` is accepted and passed to supported agent CLI invocations using each provider's documented model flag shape.
- [ ] Model-selection precedence for this task is explicit: CLI `--model` affects the current run only and does not mutate repository metadata.
- [ ] Dry-run output includes the selected model and the final agent arguments that would be executed.
- [ ] Review, feedback resume, and commit handoff either receive the same model override or have a documented reason and tests for a different behavior.
- [ ] Unsupported agent/model combinations produce usage errors with stable, actionable messaging.
- [ ] CLI docs, workflow spec/ADR notes, and task-work tests are updated; live agent smoke evidence is recorded if argument builders or orchestration contracts change.
