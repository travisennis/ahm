# Add atomic writes and concurrency protection

## Summary

Concurrent `ahm` invocations (or a crash mid-write) can interleave or truncate
metadata and generated index writes, producing corrupt JSON or stale indexes.
Every persistence path currently calls `os.WriteFile` directly with no
tmp+rename and no locking.

## Current state

All write sites in `internal/ahm/cli.go` write in place:

- `.agents/ahm.json` metadata: `writeMetadata` (~L359).
- Installed/upgraded templates: `runInstall` / `runUpgrade` (~L246, L266, L1305).
- Generated indexes (tasks, research, exec-plans): `runIndex` write loops
  (~L1367, L2038).
- Task file writes: `taskCreate` (~L1833) and task mutation paths (~L1913).

No `flock`, `sync.Mutex`, or file-lock helper guards `.agents` mutations.

## Acceptance

- Introduce a single atomic write helper (write to a sibling `*.tmp` in the
  same directory, `fsync`, then `os.Rename`) and route all managed writes
  through it: `ahm.json`, generated indexes, task files, and installed
  templates.
- Evaluate advisory locking around `.agents` mutations (e.g. a `.agents/.lock`
  via `flock`/`syscall.Flock`). Implement it if the evaluation concludes it is
  warranted; otherwise document the decision and why in `docs/spec.md` or a
  short ADR under `docs/adr/`.
- Ensure partial-failure cleanup: stale `*.tmp` files left by a crash should
  not break subsequent runs.
- Add tests for the atomic helper covering: successful replace, failure before
  rename leaves the original intact, and rejection of unsafe targets (e.g.
  outside the repo root if that constraint is adopted).
- Update `docs/spec.md` (and `docs/upgrades.md` if upgrade semantics shift) to
  describe the new write/lock guarantees.

## Notes for implementer

- Keep dry-run behavior intact for every command that currently supports it.
- Windows has no `flock`; if advisory locking is implemented, gate it behind a
  build tag or use a cross-platform abstraction and document the Windows
  behavior.
- `risk:migration` is on this task because rollout introduces `.tmp` and
  possibly `.lock` files inside `.agents/`; confirm `.gitignore` and template
  ownership rules cover them.
