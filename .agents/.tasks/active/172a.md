# Supersede ADR 013 with committed .ahm records decision

## Problem

Accepted ADR 013 makes gitignored `.ahm/` files plus a private Git ref the
durable records model. Release-readiness review showed that this turns ahm into
a synchronization engine with unsafe clone, pull, and worktree behavior. The
project has decided to retain the `.ahm/` ownership boundary while returning
source records to normal project Git tracking.

## Relevant Files

- `docs/adr/013-use-ref-backed-workflow-record-storage.md`
- `docs/references/workflow-spec.md`
- `ARCHITECTURE.md`
- `migrate-issues.md`

## Fix Direction

Create a new ADR that supersedes ADR 013. Define committed `.ahm/` source
records, ignored derived and machine-local state, local-only `prime` behavior,
and `ahm records migrate` as the namespace migration trigger for legacy
committed `.agents/` repositories. Ref-backed mode is unreleased and should be
replaced directly rather than retained as a compatibility mode. Use the ADR
lifecycle command to record supersession instead of rewriting ADR 013's history.

Define records as branch-scoped state governed by normal Git checkout and merge
semantics. A task created on one branch is not globally visible until integrated,
and simultaneous creation of the same sequential ID may produce an ordinary
add/add conflict. Also distinguish ephemeral workflow indexes under `.ahm/`
from the committed project-facing `docs/adr/index.md`.

## Acceptance Notes

- [ ] A new ADR explicitly supersedes ADR 013.
- [ ] Source records, generated artifacts, configuration, and project-owned `.agents/` content have unambiguous ownership and tracking rules.
- [ ] The decision explains why normal Git history is preferred over ahm-owned synchronization.
- [ ] Branch-scoped records, sequential-ID conflicts, migration, rollback, and linked-worktree behavior are decided before implementation.
- [ ] Ephemeral workflow indexes and the committed ADR index are explicitly distinguished.
