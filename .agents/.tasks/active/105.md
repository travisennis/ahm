# Surface non-fatal parse errors as structured warnings instead of inline stderr

## Summary

When task files have parse errors (malformed front matter, unknown status values), commands like `ahm task list`, `ahm task next`, and `ahm adr list` emit a one-line warning to stderr and continue with partial or empty results. The warning scrolls past in terminal output and is invisible in JSON mode. There are ~20 such inline `fmt.Fprintln(a.err, "warning: ...")` calls across `task_commands.go`, `adr_commands.go`, and `task_deps.go`.

The fix: collect non-fatal errors into an accumulated warnings field on the `app` struct, then flush them at command exit so they're visible regardless of output mode.

## Files in scope

- `internal/ahm/cli.go` — `app` struct (add `warnings` field), `Main` function (flush warnings on exit)
- `internal/ahm/task_commands.go` — ~10 callers that print warnings and continue
- `internal/ahm/adr_commands.go` — ~6 callers with the same pattern
- `internal/ahm/task_deps.go` — 2 callers
- `internal/ahm/output.go` — if JSON mode needs structured warnings

## Files out of scope

- `internal/ahm/validation.go` — errors there already use structured `validationReport` with codes
- `internal/ahm/write.go`, `internal/ahm/root.go`, `internal/ahm/install.go`
- Test files — existing tests may need output-string updates

## Current pattern

```go
func (a *app) taskList(mode string, statuses []string, labels []string) error {
    tasks, err := a.getTasks()
    if err != nil {
        fmt.Fprintln(a.err, "warning: some task files could not be parsed and were skipped")
    }
    // ... continues with partial tasks ...
```

## Steps

### Step 1: Add a structured warnings field to `app`

In `internal/ahm/cli.go`, add:

```go
type app struct {
    opts       options
    out        io.Writer
    err        io.Writer
    in         io.Reader
    tasksCache []Task
    warnings   []string  // non-fatal errors accumulated during a command
}
```

Add methods:

```go
func (a *app) addWarning(format string, args ...any) {
    a.warnings = append(a.warnings, fmt.Sprintf(format, args...))
}

func (a *app) emitWarnings() {
    for _, w := range a.warnings {
        fmt.Fprintln(a.err, "warning:", w)
    }
}
```

### Step 3: Flush warnings at command exit

For each affected command, call `a.emitWarnings()` at the end of the handler, before returning.

For commands that currently emit warnings inline during iteration (like `taskList`), defer the flush to the end:

```go
func (a *app) taskList(mode string, statuses []string, labels []string) error {
    tasks, err := a.getTasks()
    if err != nil {
        a.addWarning("some task files could not be parsed and were skipped")
    }
    // ... existing list logic ...
    a.emitWarnings()
    return nil
}
```

### Step 4: Handle JSON mode

For JSON mode, decide whether warnings should appear in the JSON output. Recommended approach: keep warnings on stderr only for now (the same output channel as before, just accumulated and flushed rather than interleaved). Structured JSON warnings can be added later if consumers need them.
