# Add ahm onboard, remove ahm agents suggestions, add doctor check

## Blocker

Blocked until `ahm prime` (task 156b) exists, since the snippet this command
prints references `ahm prime`.

## Summary

Add `ahm onboard`, which prints a minimal paste-ready AGENTS.md snippet;
remove the `ahm agents` command group; add a low-severity `ahm doctor`
check that flags an AGENTS.md with no `ahm prime` reference.

## Problem

`ahm agents suggestions` (see `internal/ahm/agents.go`, wired in
`internal/ahm/cli.go` `agentsCommand` ~line 289, decided in ADR 002) prints
a long adaptive advisory for merging ahm guidance into an existing
AGENTS.md, with `--all` detection of already-present sections. Under the
new content architecture (156a ADR), the long-form guidance it adapts no
longer exists: routing lives in `ahm prime` output, full instructions in
scoped contexts. What remains for AGENTS.md is a small static snippet, so
the advisory machinery is dropped, not renamed.

## Design

`ahm onboard` prints this snippet (agreed draft; adjust wording only with
user sign-off):

```markdown
## Managed Work (ahm)

This project tracks tasks, ExecPlans, ADRs, and research notes with `ahm`,
stored as files under `.agents/` and `docs/adr/`.

ALWAYS run `ahm prime` before starting work, and re-run it after context
compaction. It reports workflow state, in-progress and ready tasks,
validation warnings, and which scoped `ahm context <scope>` command covers
the work at hand. Work done without it often conflicts with tracked
in-progress work.

Never hand-edit ahm-generated indexes; update source records and run the
appropriate `ahm` command. Use `ahm task` for task state changes and
`ahm adr` for ADR lifecycle changes.
```

Deliberately absent: operating-loop guidance, workflow routing, the scope
routing table (lives in `ahm prime`), per-scope entry points. The snippet
is the layer that survives in every repo and can drift, so it carries only
the bootstrap invariant and safety invariants.

Output modes, following ahm's existing output conventions rather than TTY
sniffing: default text mode wraps the snippet with brief framing ("paste
into AGENTS.md; repos using CLAUDE.md typically import AGENTS.md");
`--plain` prints the bare snippet only; `--json` carries the snippet as a
field. Beadwork's equivalent (`bw onboard`) uses TTY detection for the
same distinction — ahm's flag-based modes are the local idiom.

Removal: delete the `agents` command group, `internal/ahm/agents.go`, and
`internal/ahm/agents_test.go`. No deprecation alias — breaking CLI change
recorded in the 156a ADR (supersedes ADR 002).

Doctor check: when AGENTS.md exists at the root and does not mention
`ahm prime`, emit a low-severity (info/warning per doctor's existing
severity vocabulary) finding suggesting `ahm onboard`. Follow existing
validation-code naming conventions (see `ahm doctor` output and
`docs/references/workflow-spec.md` validation codes). No finding when
AGENTS.md is absent — ahm does not require AGENTS.md to exist.
