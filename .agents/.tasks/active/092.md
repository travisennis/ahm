# Omit duplicate top-level heading from task create body files

## Problem

When `ahm task create` is used with `--body-file`, the supplied body should not be expected to include a top-level task title heading. The command already owns the task title in front matter and generated indexes, and duplicating it as an H1 in the body creates noisy task records.

Observed while creating this task: `ahm task create --body-file -` injected a top-level heading even though the supplied body started at `## Problem`. This may still require instruction updates, but the CLI behavior needs to be checked directly.

## Relevant Files

- `.agents/TASKS.md`
- `docs/cli.md`
- `internal/templates/workflow/`
- `internal/ahm/task_commands.go`
- Task creation tests under `internal/ahm/`

## Fix Direction

Audit the task creation workflow docs and any embedded instructions that tell agents or users to include a title heading in body files. Update them so full body files start with useful task content such as `## Problem`, `## Relevant Files`, `## Fix Direction`, and `## Acceptance Notes`.

If the CLI itself injects a top-level header when `--body-file` is used, remove that behavior and add a regression test.

## Acceptance Notes

- [ ] `ahm task create --body-file` guidance no longer asks for a top-level H1 in the body.
- [ ] Generated or example body-file tasks start with section headings, not a duplicate task title.
- [ ] Relevant docs and templates agree on the body-file shape.
- [ ] Focused tests or docs verification cover the changed behavior.
