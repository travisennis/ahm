# Switch Cake task work to stream-json orchestration

## Summary

`ahm task work` currently runs Cake with `--output-format json`, which emits a
single JSON object only after the session completes. Change Cake to
`--output-format stream-json` so `ahm` can capture and display early session
metadata as soon as work starts, including the session ID and task ID.

## Problem

The current Cake integration waits until completion to parse `session_id`.
That blocks `ahm` from surfacing initial session information when the delegated
work session starts. Cake supports `stream-json`, and the local probe confirmed
the first event includes the session identifier.

Local verification on 2026-06-12 with Cake 0.1.0:

- `cake --help` documents output formats `text`, `json`, and `stream-json`.
- `cake --output-format stream-json <prompt>` emitted a first
  `task_start` event with `session_id`, `task_id`, and `timestamp`.
- The same run emitted `message`, `reasoning`, assistant `message`, and final
  `task_complete` events.
- The final `task_complete` event included `result`, `session_id`, `task_id`,
  duration, turn/tool counts, and token usage.

## Relevant Files

- `internal/ahm/task_commands.go` - Cake args, resume args, session parser,
  review parser, and `taskWorkWithSession` tee/capture behavior.
- `internal/ahm/task_commands_test.go` - agent arg construction, session
  parsing, review parsing, and orchestration tests.
- `docs/cli.md` - supported-agents table and task work behavior.
- `docs/adr/006-task-work-agent-delegation.md` - update if the output format
  choice is treated as part of the durable task-work delegation contract.

## Fix Direction

- Change Cake work, resume, and review args from `--output-format json` to
  `--output-format stream-json`.
- Replace the single-object Cake parser with a JSONL parser:
  - session ID from the first `task_start.session_id`;
  - review feedback from the final `task_complete.result`;
  - tolerate intermediate event types.
- Consider emitting a short human-readable line on stderr after `task_start`,
  such as the agent name, truncated session ID, and Cake task ID. Keep stdout
  as the raw agent stream because `taskWorkWithSession` currently tees agent
  output to the terminal.
- Preserve current completion and commit handoff behavior by resuming the same
  captured `session_id`.
- Update docs to call out that Cake output is streamed JSON events rather than
  one final JSON object.

## Acceptance Notes

- [ ] `ahm task work <id> --agent cake` invokes Cake with
