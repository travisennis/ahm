# Add task comment command

## Summary

Add `ahm task comment <id> <text>` so humans and agents can append lightweight timestamped notes to an existing task without manually editing the task Markdown file.

## Problem

Tasks are durable working records, but adding progress breadcrumbs currently requires opening and editing the task file by hand. That is workable for large updates, but too heavy for short notes such as discoveries, handoff context, blocked reasons, review observations, or implementation breadcrumbs.

A first-class comment command would make it cheap to preserve context across sessions while keeping task files as the source of truth.

## Proposed Direction

Add a `task comment` subcommand that resolves the task ID using the same rules as `task show`, appends a comment to a dedicated body section, preserves front matter and unknown fields, updates the task's `updated` timestamp, and writes the task atomically.

Suggested command shape:

```bash
ahm task comment <id> <text>
ahm task comment <id> --body-file -
ahm task comment <id> --author "Name" <text>
```

The implementation should start narrow:

- Store comments in the task Markdown body, not in a separate database or generated index.
- Store comments under a `## Comments` section.
- Append newest comments after existing comments.
- Include timestamp metadata in a stable Markdown format.
- Omit author metadata unless `--author` is provided.
- Support `--body-file -` for multiline comments.
- Keep comments available on completed and cancelled tasks.
- Regenerate generated indexes after a comment append because the task's `updated` metadata changes.
- JSON output should return the appended comment record. Text and plain output should return a concise task/path summary.

Do not introduce a separate event log, git commits, orphan branch storage, or generated comment index unless the implementation task explicitly expands scope.

## Resolved Decisions

- Comments live in a `## Comments` section.
- Author is omitted by default and included only when `--author` is provided.
- Multiline comments are supported through `--body-file -`.
- Comment appends regenerate generated indexes because the task's `updated` metadata changes.
- JSON output returns the appended comment record. Text and plain output return a concise task/path summary.

## Acceptance Notes

- [ ] `ahm task comment <id> <text>` appends a timestamped comment to the target task.
- [ ] Multiline comments are supported with `--body-file -`.
- [ ] Existing front matter, unknown fields, task status, bucket location, and body content are preserved.
- [ ] Comments work for active, completed, and cancelled tasks.
- [ ] The command uses atomic writes and respects `--dry-run` without mutating disk.
