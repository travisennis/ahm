# Remove --complete flag from ahm task work (completion is agent-driven)

## Summary

Remove the `--complete` flag and its completion-handoff orchestration from
`ahm task work`. Completion is already agent-driven: the base work prompt
(`buildTaskWorkPrompt`) instructs the delegated agent to fill Acceptance Notes,
run verification, and mark the task complete with `ahm` as part of the normal
workflow. The separate `--complete` resume step duplicates that instruction.

This is a focused cleanup that lands before inverting the remaining
`ahm task work` defaults (see follow-up task).

## Motivation

`ahm task work` delegates to a coding agent that owns the ahm completion
workflow. A dedicated `--complete` handoff is redundant with the base work
prompt and adds a flag/step to maintain. Removing it simplifies the
orchestration surface.

## Scope / Requirements

- Remove the `--complete` flag registration in
  `internal/ahm/task_commands.go` and the `complete` field from
  `taskWorkArgs`.
- Remove the completion plumbing in `internal/ahm/task_work.go`
  (the `--complete` capability warning, the dry-run `complete` preview key, and
  the `complete` argument passed to `taskWorkWithSession`).
- Remove `runCompletion` and `buildTaskWorkCompletionPrompt` and the
  `complete` parameter/branch in `taskWorkWithSession`
  (`internal/ahm/task_session.go`).
- Confirm the base work prompt still instructs the agent to complete the task;
  if anything was only covered by the completion handoff prompt, fold the
  essential wording into the base prompt.

## Compatibility surface (must update together)

- `docs/references/cli/task-commands.md` — remove the `--complete` documentation
  (around the completion-handoff section, ~L441-L457).
- `docs/cli.md` / `docs/references/cli/commands.md` if they list `--complete`.
- CHANGELOG.md — breaking CLI change: `--complete` removed.
- Tests/fixtures under `internal/ahm` that assert `--complete` behavior or the
  dry-run `complete` preview key.

## Acceptance Criteria

- `ahm task work` no longer accepts `--complete` (unknown-flag error).
- No `runCompletion` / `buildTaskWorkCompletionPrompt` code remains.
- Dry-run preview no longer emits a `complete` key.
- Docs, CHANGELOG, and tests updated and consistent.
- `just ci` passes.
