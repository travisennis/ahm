# Treat .git files as repository markers in root detection

## Problem

`detectManagedRoot` (`root.go:43`) only accepts `.git` when it is a directory
(`stat.IsDir()`). In git worktrees and submodules, `.git` is a regular file
containing a `gitdir:` pointer, so detection falls through to the
`.agents/ahm.json` check. In a worktree of a repo that is not yet managed,
`ahm status`/`doctor` fail with the confusing "not in a managed repository"
error, and lenient commands (`ahm init`) fall back to the CWD instead of the
worktree root.

## Fix Direction

Accept any `.git` entry (file or directory) as a repository marker — drop the
`IsDir()` requirement.
