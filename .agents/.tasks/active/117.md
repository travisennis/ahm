# Redesign ahm context role around managed-work intake

## Problem

Recent work added `ahm context`, scoped context commands, and stronger
`ahm agents suggestions` integration guidance, but the resulting contract is
still muddled. Unscoped `ahm context` currently mixes live repository state
with instruction authority, scoped text and scoped JSON disagree about what a
scoped command returns, and several docs/templates still describe
`ahm context` as canonical workflow guidance.

The project needs a primitives-first split:

- `AGENTS.md` owns early agent operating-loop and workflow-routing instructions.
- Unscoped `ahm context` provides a live repository/session briefing.
- Scoped `ahm context task|plan|adr|research|docs` provides reference material
  for ahm-managed artifacts and workflows.
- `ahm agents suggestions` helps maintainers adapt project-owned `AGENTS.md`
  files to those primitives without replacing project-specific guidance.

This needs a deliberate contract update so agents know when to use unscoped
context, when to use scoped references, and when to return to project
`AGENTS.md` routing.

## Decisions

- Unscoped `ahm context` is a live briefing primitive only. It should not
  print an `## Instructions` section or claim workflow authority.
- Scoped `ahm context task|plan|adr|research|docs` text output remains pure
  reference output with no live briefing wrapper.
- Scoped JSON should match scoped text semantically: include scope,
  instructions, and optionally scoped command discovery, but no live workflow,
  git, or task briefing fields.
- Unscoped JSON may intentionally change as part of this task. It should keep
  root, workflow, git, tasks, and commands, and remove instructions.
- Replace "canonical workflow guidance" language with narrower wording such as
  "managed-work reference" or "workflow reference for ahm-managed artifacts."
- `ahm agents suggestions` should describe the primitives model: project
  `AGENTS.md` owns routing, unscoped `ahm context` gives live state, scoped
  context gives ahm-managed references, and suggestions are advisory bridge
  text.
- ADR 011 should get a short 2026-06-20 refinement note rather than a
  superseding ADR.
- Scoped reference documents should stop claiming session-start control. They
  should say they are used when project routing, managed-work intake, or the
  user request points at that artifact type.
- CLI help/docs should describe two modes while keeping the current syntax:
  `ahm context` for live briefing and `ahm context <scope>` for managed-work
  reference.
- `ahm context` should remain root-aware: it works in a detected git repo even
  before `ahm init`, but it should not become a generic current-directory
  briefing outside a repo unless `--root` is provided.
- Scoped references should work anywhere unscoped `ahm context` can detect a
  repo root, including before `ahm init`.
- Because embedded workflow references and suggestions change, bump
  `internal/templates.Version` from `0.4.2` to `0.4.3` and document the
  upgrade note.
- `ahm task show <id>` is the normal task inspection primitive because it
  prints the task file contents. Do not instruct agents to also open the task
  file unless `ahm task show` is unavailable or they are manually editing the
  task record.
- The managed `preflight` skill should require `ahm task show <id>` output for
  task context, with the active task file only as fallback when `ahm` is
  unavailable or when reviewing manual task-file edits.
- In text output, `validation: ok` means zero errors and zero warnings. A
  warnings-only state should show the warning count, recommend `ahm doctor`,
  and print the first few findings.
- Context findings remain capped at five total findings, ordered errors first
  and then warnings.

## Fix Direction

Follow `.agents/exec-plans/active/117-redesign-ahm-context-role.md`.

At a high level:

1. Change unscoped `ahm context` text and JSON output to be a compact live
   briefing without instruction authority.
2. Change scoped JSON to align with scoped text semantics.
3. Update validation summary text so warnings-only state is visible.
4. Update command help, tests, CLI docs, workflow spec, upgrade docs,
   architecture notes, ADR 011, embedded workflow references, suggestions,
   this repo's `AGENTS.md`, and managed preflight skill wording.
5. Bump the embedded template version to `0.4.3`.
