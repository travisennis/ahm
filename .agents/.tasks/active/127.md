# Add scope descriptions to context --help

## Summary

The `context` command's `--help` doesn't describe what each scope returns. The scopes are in the usage line (`ahm context [task|adr|research|plan|docs]`) but not described. An agent has to run each one to discover what they return. The docs have excellent descriptions of each scope.

## Evidence

```
$ ahm context --help
Repository briefing or managed-work reference

Usage:
  ahm context [task|adr|research|plan|docs] [flags]
```

No description of what `task`, `adr`, `research`, `plan`, or `docs` scopes return.

- `internal/ahm/cli.go` — `contextCommand()` sets `Short: "Repository briefing or managed-work reference"` but has no `Long` field.
- `docs/references/cli/commands.md` — has a full description of each scope.

## Fix Direction

Add a `Long` field to the `context` command that lists each scope with a one-line description:

```
Print a repository briefing or a managed-work reference.

With no scope, prints a live repository briefing with repository state,
validation status, git branch, task counts, and useful follow-up commands.

Scopes:
  task     Task workflow reference for creating, choosing, and working on tasks
  adr      ADR workflow reference for numbering, naming, and template rules
  research Research workflow reference for organizing and using research notes
  plan     ExecPlan workflow reference for planning larger tasks
  docs     Documentation workflow reference for auditing and updating docs

Examples:
  ahm context
  ahm context task
  ahm --json context adr
```
