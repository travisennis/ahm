# Normalize exec plan field in one layer only

## Summary

Review issue 42. exec_plan is normalized with defaultDash in both parseTask and renderTask.

## Decision

Approach: **Normalize in parse (read path) only; remove `defaultDash` from render.**

This keeps `exec_plan` consistent with every other `defaultDash` field (Status,
Priority, Effort, Labels) — all are normalized in `parseTaskFromData` and trusted
by `renderTask` without re-normalization.

Change needed: remove the `defaultDash()` wrapper around `task.ExecPlan` on the
`renderTask` line that writes `exec_plan`. All other render lines for defaultDash
fields already trust the struct value.
