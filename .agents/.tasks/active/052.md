# Validate design-doc indexes when present

## Summary

Add optional design-doc index validation for repositories that already have a
`docs/design-docs/` convention.

## Problem

Some repositories maintain design docs with a generated or curated index.
`ahm` currently validates task, research, and ExecPlan indexes because those
are managed workflow artifacts. It does not recognize design-doc indexes, even
when a repository clearly has a `docs/design-docs/` tree.

This should remain optional and convention-sensitive. `ahm` must not impose a
design-doc structure on projects that do not already use one.

## Relevant Files

- `internal/ahm/validation.go` for validation orchestration.
- `internal/ahm/validation_test.go` for coverage.
- `docs/cli.md` for finding codes and command behavior.
- `docs/spec.md` if design-doc validation becomes durable workflow semantics.

## Fix Direction

Only run this validation when a repository has an obvious `docs/design-docs/`
surface, such as a directory plus an `index.md` file. Validate simple, generic
invariants rather than this project's local preferences:

- Markdown files in the design-doc directory are represented in the index.
- Index entries point at files that exist.
- Relative Markdown links inside the design-doc surface resolve.

Do not generate or rewrite design-doc indexes in this task. If auto-generation
is desirable, capture it separately after the validation contract exists.

This likely depends on the scoped validation work if the checks should run only
for project documentation surfaces. Keep default workflow validation quiet for
projects without design docs.

## Acceptance Notes

- [ ] Repositories without `docs/design-docs/` get no design-doc findings.
- [ ] Repositories with `docs/design-docs/index.md` can be validated without
      assuming this project's specific design-doc content.
- [ ] Missing indexed design-doc source files produce a clear finding.
- [ ] Design-doc Markdown files missing from the index produce a clear finding.
- [ ] Broken relative links inside the design-doc surface produce a clear
      finding or reuse the existing Markdown link finding where appropriate.
- [ ] `ahm` does not create, rewrite, or format design-doc indexes as part of
      this task.
- [ ] `docs/cli.md` documents any new finding codes or scoped behavior.
- [ ] `go test ./internal/ahm` covers absent, valid, and invalid design-doc
      surfaces.
- [ ] `just ci` is green before handoff.
