## Comments

**2026-07-02T10:32:05-04:00** — Scope addition for the ADR revision (design discussion 2026-07-02): decide the storage namespace before implementation. Recommendation: move ahm-managed state (tasks, research, exec-plans, generated indexes, sync state) from .agents/ to a tool-owned .ahm/ directory as part of the same opt-in migration. Rationale: .agents/ is becoming an ecosystem-standard home for project-owned content agents read (skills, prompt.md — see task 162); mixing ahm's mutable state into it forces per-file gitignore classification that fails silently. .ahm/ lets ahm ship a managed .ahm/.gitignore inside its own directory (records ignored, committed config like .ahm/config.json not), so consumer root .gitignore is never touched. Alternative considered and disfavored: selective ignore rules within .agents/. Consequences to cover: root detection, ahm.json rename/relocation, template and doc path churn (task 145 scope), coordination with 156g (after skills removal, ahm's .agents/ footprint drops to zero).

**2026-07-02T10:35:35-04:00** — Decision 2026-07-02: maintainer approved the .ahm/ namespace. Record it in the ADR revision as decided — ahm-managed state (records, indexes, sync state, config) moves to tool-owned .ahm/ with a managed internal .gitignore; .agents/ remains project-owned committed content that ahm reads but never manages. docs/VISION.md ownership table and git-safety boundary updated to match.
