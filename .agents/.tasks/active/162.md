# Include project instructions file in ahm task work prompts

## Summary

Let `ahm task work` carry project-owned standing instructions to the
delegated agent: when a project instructions file exists, append its
content to the built work prompt. The maintainer already keeps such a
file at `.agents/prompt.md` (orientation and vision-focus instructions
for the 138/156/160 work); today it must be pasted manually because
`task work` accepts no additional instructions.

## Design

- When the project instructions file exists, `buildTaskWorkPrompt`
  (`internal/ahm/task_work.go`) appends its content after the base
  prompt under a `## Project Instructions` heading. Base prompt first:
  the task pointer and workflow mechanics stay the lead; the project
  preamble follows.
- Default path `.agents/prompt.md`; configurable via a `taskWork`
  block in `.agents/ahm.json` (e.g. `"promptFile": "path.md"`).
  Missing file is not an error — the feature is opt-in by presence.
- `--no-project-prompt` flag skips inclusion for a single run.
- `ahm` never creates, templates, or upgrades this file. It is
  project-owned judgment content, same ownership rule as `AGENTS.md`
  (see `docs/VISION.md` ownership table and non-goals). `ahm init`
  must not scaffold it.
- Scope: the work prompt only. Whether review orchestration
  (`reviewArgs` prompts in `internal/ahm/task_agents.go`) should also
  include it is deliberately out of scope; note it in the ADR-less
  decision record below if revisited.
- `--dry-run task work` already surfaces the assembled prompt in its
  output; inclusion is therefore previewable with no extra work —
  cover it in tests.

## Decision

Storage mode (decided 2026-07-02 with the `.ahm/` namespace decision on
task 137): `.agents/prompt.md` is project-owned content agents read, so
it stays in `.agents/` and is **committed** — not gitignored, not
ref-synced. No snapshot protection is needed. The earlier coordination
comment on tracker 138 about snapshotting this file is superseded (see
the amendment comment there).

## Relevant Files
