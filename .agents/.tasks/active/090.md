# Document agent integration smoke checklist in workflow docs

## Problem

The bf6c1ae cake stream-json regression (fixed in task 087) shipped because
the implementing agent never ran the real cake binary; unit tests round-tripped
the parser's own invented schema. The repository workflow has no requirement
to live-test agent integration changes, so the same failure mode applies to
codex today and to the pending Claude Code (082) and cursor (084) work.

## Relevant Files

- `AGENTS.md` — Required Workflow section (the enforcement point agents read)
- `docs/testing.md` — new home for the checklist detail (or extend an existing
  doc if one fits better; check `.agents/DOCS.md` conventions first)
- Tasks 088 (golden fixtures) and 089 (smoke test) — reference once landed

## Implementation Notes

- Document a per-agent smoke checklist for changes touching `taskWorkAgent`
  arg builders, output parsers, or orchestration flow in
  `internal/ahm/task_commands.go`:
  1. Run the real agent end-to-end via `ahm task work` (or
     `just smoke-agents` once task 089 lands) for each affected agent.
  2. Verify "session started:" on stderr and the absence of the
     "no session ID returned by" / "could not capture session ID" warnings.
  3. Record the agent version and a transcript snippet in the task's
     Acceptance Notes.
- Wire the requirement into AGENTS.md so it binds agent-driven work: tasks
  labeled `area:agent` + `risk:external-service` must include the live-run
  evidence in Acceptance Notes before `ahm task complete`.
- Cross-reference `just capture-agent-fixtures` (task 088) as the way to
  refresh goldens when an agent CLI version changes its output.
- Docs-only change: full CI not required, but verify Markdown and links.
