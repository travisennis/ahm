# Record sessions for every task work agent run

## Problem

`ahm task work` should produce a resumable, auditable session for every external agent invocation it starts. At least one cake path currently passes `--no-session`, which prevents session creation and undermines auditability for delegated work.

The goal is to remove session-suppression from task-work agent invocations and make session capture a required invariant across supported session-capable agents.

## Relevant Files

- `internal/ahm/task_agents.go` - agent definitions and argument builders; the cake invocation currently includes `--no-session`.
- `internal/ahm/task_parsers.go` - session ID parsers and resume argument helpers for supported agents.
- `internal/ahm/task_session.go` - orchestration/session persistence path for `ahm task work`.
- `internal/ahm/task_work_smoke_test.go` - live smoke coverage for session capture and resume behavior.
- `internal/ahm/testdata/agents/` - golden transcripts and provenance sidecars if any agent output schema changes.
- `docs/guides/testing.md` - live agent smoke checklist and required evidence.

## Fix Direction

- Audit every `ahm task work` agent command path, including work, review, completion, and commit handoff where relevant, for flags or modes that suppress session creation.
- Remove the cake `--no-session` usage from task-work invocations and ensure cake still emits stream JSON with a capturable session ID.
- Add or update focused tests so session-capable agent definitions cannot silently opt out of session creation.
- Keep resume behavior consistent: the captured session ID should be used for follow-up prompts, review, completion, and commit handoff wherever the orchestration expects a session.
- Refresh golden transcripts only if the real CLI output schema changes.

## Acceptance Notes

- [ ] No `ahm task work` agent invocation suppresses session creation for a session-capable agent.
- [ ] Cake task-work runs create and report a session ID instead of running with `--no-session`.
- [ ] Focused unit/parser/orchestration tests cover the no-session regression risk.
- [ ] `just ci` passes.
- [ ] Live agent smoke evidence is recorded before completion per `docs/guides/testing.md`, including installed agent versions and transcript snippets showing `session started:` without `no session ID returned by` or `could not capture session ID`.
