# Fix empty list output in text mode and JSON null vs empty array

## Summary

`task list`, `task ready`, and `task blocked` produce zero output in text mode when there are no matching tasks. An agent can't distinguish "command succeeded, no results" from "command failed silently." `task next` gets this right with "No ready tasks." — the list commands should follow the same pattern.

Additionally, JSON mode returns `null` for empty task lists while `task labels` returns `[]`. An empty list is still a list; `[]` is more correct than `null` for JSON consumers.

## Evidence

```
$ ahm task list --status Open
(no output, exit 0)

$ ahm task ready
(no output, exit 0)

$ ahm task ready --json
null

$ ahm task labels --json
[]
```

- `internal/ahm/task_list.go` — `taskList()` iterates `filtered` and prints lines, but when `filtered` is empty, nothing is printed. No empty-state message.
- `internal/ahm/task_list.go` — `taskNext()` handles this correctly: prints "No ready tasks." in text mode and emits `nil` (which becomes `null`) in JSON mode.
- `internal/ahm/output.go` — `emitJSON()` prints `"{}"` for `nil`, but `taskList` passes an empty `[]Task` slice (not `nil`) to `emit`, which marshals as `null` because Go's `json.Marshal` produces `null` for `nil` slices.

## Fix Direction

1. **Text mode**: In `taskList()`, after filtering, if `filtered` is empty, print "No tasks found." (or a mode-specific message like "No ready tasks." / "No blocked tasks.") and return. Match the pattern already used by `taskNext()`.

2. **JSON mode**: Ensure empty task lists marshal as `[]` instead of `null`. This can be done by initializing the slice as `[]Task{}` instead of `nil` when empty, or by checking for `nil` before marshaling in the emit path.

## Acceptance Notes

- [ ] `task list` with no matching tasks prints "No tasks found." in text mode
