# Support returning an ADR to proposed via ahm adr

## Problem

`ahm adr` exposes forward lifecycle transitions only: `accept`, `reject`,
`deprecate`, and `supersede`. There is no command that returns an ADR to
`proposed`, so undoing a premature status change requires hand-editing the
front matter and running `ahm index` — exactly the manual fallback the ADR
command surface exists to remove.

Real occurrence (2026-07-11): ADR 016 was accepted during task 160a, the
maintainer wanted it back at `proposed` for review, and the only path was a
manual front-matter edit plus `ahm index`.

## Relevant Files

- `internal/ahm/adr_commands.go` — command wiring; the status-verb spec table
  (`accept`/`reject`/`deprecate`) and the generic `adrSetStatus(id, status)`
  helper already exist, so the change is mostly exposing another transition.
- `docs/cli.md` and the `docs/references/cli/` ADR page — document the new
  subcommand.
- Embedded `ahm context adr` instruction content — the command table lists the
  lifecycle commands and currently implies status can only move forward.

## Fix Direction

Add a `propose` verb to the existing status-spec table (`ahm adr propose <id>`
sets `status: proposed`), reusing `adrSetStatus` and index regeneration.
Alternative considered: a generic `ahm adr status <id> <value>`; the verb form
matches the existing surface and keeps `superseded by ADR-NNN` reachable only
through `ahm adr supersede`, which preserves the reciprocal-reference
invariant.

Decide at acceptance whether this needs its own ADR or is an anticipated
extension of ADR 009 (MADR ADR Management) recordable as an ADR 009 body
update — the lifecycle-command contract is user-visible CLI surface.

Behavior notes:

- `propose` on an ADR whose status is `superseded by ADR-NNN` should either be
  refused or also clean up the reciprocal supersession reference; refusing is
  simpler and keeps supersession edits explicit.
- Respect `--dry-run` and regenerate `docs/adr/index.md` like the other
  lifecycle commands.

## Acceptance Notes

- [ ] `ahm adr propose <id>` (or the chosen equivalent) sets an ADR back to
      `proposed` and regenerates `docs/adr/index.md` without hand edits.
- [ ] Transition from `superseded by ADR-NNN` is explicitly handled (refused
      or reciprocal references cleaned up), with a test.
- [ ] `docs/cli.md`, the CLI reference page, and the embedded `ahm context adr`
      content document the new transition.
- [ ] ADR-or-ADR-009-update decision made and recorded before implementation.
