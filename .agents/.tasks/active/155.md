# Add ahm config command for ahm.json settings

## Summary

The two repository-scoped settings ŌĆö `strict_acceptance` and `default_work_agent` ŌĆö are configurable only by hand-editing `.agents/ahm.json` (documented in docs/references/workflow-spec.md and docs/references/cli/task-commands.md). Corrupt metadata deliberately fails every command loudly (task 061), so the only supported edit path is also the one most likely to brick the workflow on a typo. A small validating `ahm config` command closes that gap. Written against commit ff15056.

## Evidence

- `internal/ahm/install.go:18-23` ŌĆö metadata struct with `StrictAcceptance` and `DefaultWorkAgent`.
- `docs/references/workflow-spec.md:76-91` ŌĆö documents hand-editing these fields.
- `internal/ahm/task_agents.go:22-39` ŌĆö consumer of `default_work_agent`; invalid agent values only surface later, at `task work` time.
- `internal/ahm/install.go:248-264` ŌĆö `readMetadata`/`writeMetadata` already provide the read and atomic-write primitives.

## Fix Direction

Add `ahm config` (list current settings), `ahm config get <key>`, and `ahm config set <key> <value>`, operating only on the two known keys; reject unknown keys with a usage error listing valid ones. `set` validates values (`strict_acceptance`: true/false; `default_work_agent`: must pass `parseTaskWorkAgent`), reads metadata, mutates, and writes via `writeMetadata` (atomic). Fail loudly on corrupt or missing metadata with the existing remediation messages ŌĆö never create or reset metadata implicitly. Support `--dry-run` (preview, no write) and the standard text/JSON/plain output modes. Docs: new page under `docs/references/cli/` and cross-references where hand-editing is currently documented; update the ARCHITECTURE.md module map if a new file is added. Out of scope: arbitrary keys, editing the `files` hash map or `version` field.
