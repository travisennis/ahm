# Add resumable task work sessions

## Summary

Explore and implement resumable task work sessions where supported agents can
return a session identifier and later receive follow-up prompts for the same
task.

## Background

`scripts/task-workflow.sh` depends on `cake --output-format json` returning a
`session_id`, then resumes that session with `cake --resume`. The MVP
`ahm task work` intentionally does not parse provider output or manage sessions.
Session support may be necessary before `ahm` can safely orchestrate review
feedback or follow-up completion prompts.

## Design Questions

- Which agents expose stable session identifiers and resume commands?
- Should session IDs be transient command output only, or should `ahm` persist
  them in task metadata or repository-local state?
- If sessions are persisted, what is the file format and lifecycle?
- How should `ahm` avoid leaking provider-specific credentials or sensitive
  session data?
- Should this be implemented only for `cake` first, or with an abstraction for
  multiple agent CLIs?
