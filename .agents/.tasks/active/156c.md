# Add live ahm context groom procedure scope

## Summary

Add `ahm context groom`: the backlog-grooming procedure currently installed
as the `grooming-backlog` skill, emitted from the binary and interleaved
with live backlog state.

## Problem

The grooming procedure lives in
`internal/templates/workflow/grooming-backlog-SKILL.md` (~124 lines),
installed to `.agents/skills/grooming-backlog/SKILL.md`. Per the 156a ADR,
skills stop being installed; the procedure becomes a scoped context. Making
it live is the point: a grooming prompt that already contains the current
queue state is immediately actionable, where the static skill made the
agent gather state first.

## Design

Output = procedure instructions (adapted from the SKILL.md body, dropping
skill frontmatter and any "when to use this skill" framing) interleaved
with live state:

- Open tasks (full lines) — the triage queue.
- Blocked tasks (full lines) with their `depends_on` values — staleness
  review.
- Pending count and any validation findings from the existing validation
  report (e.g. unchecked acceptance items).
- Label vocabulary (`ahm task labels` data) so grooming keeps labels
  consistent.

Follow the goals in the existing skill: every Open task triaged to
Pending/Blocked/next-action, decisions recorded, dependencies accurate,
bodies self-contained, indexes regenerated via `ahm` commands.

Mechanics: extend `scopedContextInstruction` and `contextCommands` in
`internal/ahm/context.go` with the `groom` scope. Unlike existing scopes
(static embedded files), this scope composes live sections — structure the
scoped report so instruction body and live sections are separate fields
(JSON parity), rendered together in text. Compose data in Go from the same
internal listing functions used by task commands; do not shell out to ahm
itself.

## Relevant Files

- `internal/templates/workflow/grooming-backlog-SKILL.md` — source content
  (deleted later by 156g; absorb it here)
- `internal/ahm/context.go` — scope registry (`scopedContextInstruction`
  ~line 211, `contextCommands` ~line 238), scoped report shape
- `internal/ahm/context_test.go`
- Task listing/label helpers used by `ahm task list/blocked/labels`
