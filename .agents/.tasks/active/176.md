# Remove dead ref-mode remnants from records code

## Problem

After task 172 removed ref-backed records, several ref-mode abstractions remain even though committed storage is now the only supported mode. In `internal/ahm/records.go`, `collectRecordFiles` and `isRecordRelPath` have no non-test callers; their supporting `recordFile` type and `recordSourceDirs` variable are likewise dead. In `internal/ahm/install.go`, the `recordStoreMode` enum and `recordsStorage()` method only produce the constant mode string consumed by prime and records doctor.

This dead code obscures the committed-only storage model and leaves tests asserting behavior that production no longer uses. The remnants should be removed without changing the documented CLI and JSON compatibility surfaces. Surfaced during the task 172 release review.

## Relevant Files

- `internal/ahm/records.go` — dead record-file collection helpers and supporting declarations.
- `internal/ahm/records_test.go` — tests that exist only for the dead collection helpers.
- `internal/ahm/install.go` — vestigial record storage mode abstraction.
- `internal/ahm/install_test.go` — assertions for the vestigial abstraction.
- `internal/ahm/prime.go` — emits the documented `records.mode` JSON field.
- `internal/ahm/records_commands.go` — records doctor reports the current mode.

## Fix Direction

Remove `recordFile`, `recordSourceDirs`, `collectRecordFiles`, `isRecordRelPath`, `recordStoreMode`, and `recordsStorage()`, along with tests that cover only those symbols. Preserve the existing external behavior by emitting the literal string `"committed"` at the prime and records-doctor call sites. Do not remove or rename `prime --json`'s `records.mode` field, and do not change the records-doctor mode output. Because those documented outputs remain unchanged, no user-facing documentation update is required; update internal comments or docs only if they still describe multiple current storage modes.

## Acceptance Notes

- [ ] Dead record collection declarations and helpers are removed from `internal/ahm/records.go`, with tests that exist only for them removed from `internal/ahm/records_test.go`.
- [ ] The `recordStoreMode` abstraction and `recordsStorage()` are removed from `internal/ahm/install.go`, with abstraction-specific assertions removed from `internal/ahm/install_test.go`.
- [ ] Prime JSON continues to include `records.mode` with the value `"committed"`.
- [ ] Records doctor continues to report committed storage mode with no user-visible output or exit-behavior change.
- [ ] Documentation and source comments contain no stale claim that ref-backed storage is a current mode.
- [ ] Full CI passes.
