# Implement records metadata and storage mode model

## Summary

Add the metadata and internal model needed for committed legacy records, gitignored local records, and ref-backed record sync.

## Relevant Records

- ADR: `docs/adr/013-use-ref-backed-workflow-record-storage.md`
- ExecPlan: `.agents/exec-plans/active/138-ref-backed-workflow-records.md`

## Work Direction

Extend metadata parsing and rendering so `.agents/ahm.json` can represent
legacy committed records, gitignored local records, and ref-backed records.
Absence of new metadata must preserve existing behavior. Keep field names and
defaults conservative; likely fields include `store_mode`, `records_ref`,
`records_remote`, and last successful sync metadata.

## Acceptance Notes

- [ ] Missing records metadata preserves current committed-record behavior.
- [ ] Ref-backed mode can be represented in `.agents/ahm.json` without breaking
      old metadata parsing.
- [ ] Metadata round-trip tests cover unknown fields and default behavior.
- [ ] Workflow spec updates are identified for task 145.
