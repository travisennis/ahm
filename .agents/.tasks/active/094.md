# Rename deslop review workflow to a professional name

## Problem

The review-readiness workflow is currently named `deslop`, which is too informal for project documentation, agent instructions, and CLI-facing workflow references. The underlying idea is useful, but the name makes the workflow feel like a joke in places where it should read as maintained tooling.

## Relevant Files

- `.agents/skills/deslop/`
- `.agents/skills/deslop/SKILL.md`
- `.agents/TASKS.md`
- Agent and task workflow docs that reference `deslop`
- Task 091, if it still references the current name
- Any tests or fixtures that mention the skill by name

## Fix Direction

Pick a clearer name for the workflow and update references consistently. Candidate names include `review-readiness`, `final-review`, `change-review`, or `preflight-review`.

Prefer a name that describes the job rather than the complaint. Keep compatibility in mind: if agents or docs still reference `deslop`, decide whether an alias, migration note, or one-time rename is enough.

## Acceptance Notes

- [ ] The skill directory and skill metadata use the new name.
- [ ] Project workflow docs and task references no longer use `deslop` as the primary name.
- [ ] Any compatibility behavior or migration note is documented if needed.
- [ ] Task 091 is updated or superseded so future work uses the new name.
