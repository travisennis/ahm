# Consolidate duplicate front matter parsing in task migrate

## Problem

`task_migrate.go` re-implements front-matter mechanics that already exist in
`tasks.go`:

- `splitFrontMatter` (`task_migrate.go:117-127`) duplicates the CRLF
  normalization and `---` fence splitting in `parseFrontMatter`
  (`tasks.go:141-152`).
- `frontMatterValue` (`task_migrate.go:172-183`) duplicates the
  quote-stripping logic, and `normalizeDependsOnValue`
  (`task_migrate.go:189-196`) repeats it a third time.

Two parsers for one format will eventually disagree (e.g. a future BOM or
quoting change applied to one but not the other — `readWorkflowFile` already
handles BOM, but the inline `\r\n` replaces are repeated in four places).

## Fix Direction

Extract shared helpers — fence splitting (returning raw block + body) and
value unquoting — used by both `parseFrontMatter` and the line-oriented
migration code. Keep migration's line-preserving behavior intact; it operates
on raw lines deliberately.

- `internal/ahm/tasks.go:141-180`
- `internal/ahm/task_migrate.go:77-224`
- `internal/ahm/task_acceptance.go:17` and `internal/ahm/agents.go:75`
  (other inline CRLF normalizations worth routing through one helper while
  in here)
