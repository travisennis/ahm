# Implement private ref snapshot and materialization plumbing

## Blocker

Blocked until ADR 013 and the ref-backed workflow record ExecPlan are finalized
by task 137, and until task 140 defines the records metadata/storage-mode model
this plumbing depends on.

## Summary

Create the low-level git plumbing that snapshots workflow records into refs/ahm/records and materializes them back into .agents without touching branches, the index, staging area, or HEAD.

## Relevant Records

- ADR: `docs/adr/013-use-ref-backed-workflow-record-storage.md`
- ExecPlan: `.agents/exec-plans/active/138-ref-backed-workflow-records.md`
- Research: `.agents/.research/topics/records-storage-via-git-refs.md`

## Work Direction

Build focused internal helpers before exposing CLI commands. The helpers should
select the record files, build a Git tree, create a commit, update
`refs/ahm/records`, fetch/push the remote ref, compare ref state, and
materialize records back into `.agents/`. Generated indexes must be excluded
from snapshots. Tests must prove the helpers do not move `HEAD`, stage files,
write the index, or mutate project branches during routine snapshot and
materialization operations.

## Acceptance Notes

- [ ] Internal plumbing can create a local `refs/ahm/records` snapshot from
      task, research, and ExecPlan source records.
- [ ] Materialization can restore those records into `.agents/`.
- [ ] Generated indexes are excluded from the records ref.
- [ ] Tests prove project branches, `HEAD`, the index, and staging area are not
      mutated by routine snapshot/materialization helpers.
