# Add docMap config and diff-driven doc review suggestions

## Blocker

Blocked until task 160b lands the `ahm docs check` command this extends.

## Summary

Add `projectDocs.docMap` to `.agents/ahm.json` and a diff mode to
`ahm docs check` that intersects changed paths with the map and emits
advisory `doc_review_suggested` findings: "changes touch X; review doc Y."

## Behavior

- `docMap` is a list of `{ "paths": [globs], "docs": [repo-relative docs] }`
  entries in the `projectDocs` block.
- `ahm docs check --staged` diffs the index; `--diff <range>` diffs an
  arbitrary range. Git usage is read-only within the existing safety
  boundary (see `readGitContext` precedent in `internal/ahm/context.go`).
- Changed paths matching a `docMap` entry emit `doc_review_suggested`
  (info) naming the changed path group and the docs to review — unless the
  mapped doc itself changed in the same diff (the review presumably
  happened).
- `doc_review_suggested` never affects the exit code, including under
  `--strict`. A machine knows the doc's subject changed, not that the doc
  is stale.
- `doc_map_target_missing` (error): a docMap entry references a doc that
  does not exist. Validated on every `docs check` run, not only in diff
  mode.
- Static checks still run in diff mode; `--staged`/`--diff` adds findings,
  it does not replace the health pass.
