# Add ahm prime and stale record reporting

## Blocker

Blocked until task 141 defines the records sync command behavior that
`ahm prime` builds on, and until task 156b introduces the read-only
`ahm prime` briefing command.

## Summary

Add the agent-facing ahm prime command and human-facing stale-state notes so agents sync records before work and humans can see when workflow state is missing, stale, unsynced, or diverged.

## Relevant Records

- ADR: `docs/adr/013-use-ref-backed-workflow-record-storage.md`
- ExecPlan: `.agents/exec-plans/active/138-ref-backed-workflow-records.md`

## Work Direction

Add `ahm prime` as the explicit startup command for coding agents. It should
sync or fetch records, materialize local files, regenerate indexes, validate
workflow state, and print a compact backlog briefing. Human-facing commands
should also show concise stale/unsynced notes when workflow records need
attention.

## Acceptance Notes

- [ ] `ahm prime` performs the documented records sync/materialization/index/
      validation sequence.
- [ ] `ahm prime` reports ready/open/blocked task counts, high-priority ready
      tasks, active ExecPlans, recent research, stale state, and suggested next
      commands.
- [ ] Normal human commands emit concise stale/unsynced notes without surprise
      network pushes.
- [ ] Agent guidance tells coding agents to run `ahm prime` before work.
