# Add ADR lifecycle and supersede commands

## Summary

Add ADR lifecycle commands mirroring the table-driven task status commands:
`ahm adr accept <id>`, `ahm adr reject <id>`, `ahm adr deprecate <id>`, and
`ahm adr supersede <old-id> --by <new-id>`.

## Background

Design per ADR 009 (task 074) and
`.agents/.research/topics/madr-adr-management-design.md`. Model and rendering
come from task 075. Supersession semantics follow the existing guidance in
`docs/adr/README.md`: the old record stays, both records link to each other.

## Implementation Notes

- `accept`/`reject`/`deprecate` update `status:` (and `date:`) in front
  matter via atomic writes and regenerate the ADR index; mirror the
  table-driven status command loop in `internal/ahm/task_commands.go`.
- `supersede <old-id> --by <new-id>` in one command:
  - sets the old record's status to `superseded by ADR-NNN`,
  - writes or replaces a short supersession note in the old body linking to
    the replacement (replace-not-duplicate on rerun, mirroring the
    Cancellation Reason section behavior from task 073),
  - adds a reference to the superseded ADR in the new record's More
    Information section.
- Define error behavior: unknown IDs, self-supersession, superseding an
  already-superseded record.
- ADR bodies are otherwise user-owned: commands touch only front matter plus
  the defined supersession note/reference.
- Update `docs/cli.md` with sections for all four commands in the same
  change.

## Relevant Files

- `internal/ahm/task_commands.go` (status command pattern)
- `internal/ahm/write.go`
- `docs/adr/README.md` (supersession semantics)
- `docs/cli.md`

## Acceptance Notes

- [ ] Transitions rewrite only front matter (plus the supersession note) and
      preserve the rest of the body byte-for-byte.
- [ ] `supersede` updates both records in one invocation and replaces rather
      than duplicates its note on rerun.
- [ ] Error cases are tested: unknown IDs, self-supersession,
      already-superseded targets.
- [ ] Indexes regenerate as part of each command; `docs/cli.md` updated;
      `just ci` passes.
