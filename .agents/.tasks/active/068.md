# Reuse parsed tasks during generated index validation

## Problem

`ahm status`/`doctor` parse the task tree multiple times per run:

- `validateTaskFiles` parses every task once (good — task 036).
- `validateGeneratedIndexes` (`validation.go:428-452`) then constructs a fresh
  `app{opts: options{root: root}}` and re-parses everything via
  `indexWrites` -> `collectTasks`.
- Exec-plan files are parsed once per referencing task in
  `validateTaskExecPlans` (`execPlanHasRetrospective`) and again in
  `validateExecPlans`.

Follow-up to task 036, which removed the double read inside
`validateTaskFiles` but not these cross-validator re-parses. Harmless at ~60
tasks; structural fix before it matters.

## Fix Direction

- Refactor index-content generation so it can accept already-parsed tasks
  (e.g. extract a package-level `indexWritesFor(root, tasks)` used by both
  `app.indexWrites` and `validateGeneratedIndexes`).
- Cache parsed exec-plan sections per path within a single validation run.
