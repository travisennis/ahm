# Remove dead validation-scope and output-mode helpers

## Problem

Two helpers are dead or duplicated:

- `ValidCheckScopes()` (`validation.go:36-39`) has zero callers, while
  `validateCheckScopes` (`cli.go:228-237`) hand-duplicates the same scope list
  in its switch and error message. Two sources of truth for the
  workflow/links/project-docs scope set.
- `outputMode()` (`output.go:22-33`) is called only by its own test
  (`output_test.go:274`); no production caller.

## Fix Direction

- Make `validateCheckScopes` derive both the validity check and the error
  message from `ValidCheckScopes()` (or delete the exported helper and keep a
  single unexported source). One definition of the scope list.
- Delete `outputMode` and its test, or wire it into real output (e.g. status
  diagnostics) if there is a use.

## Relevant Files

- `internal/ahm/validation.go:29-39`
- `internal/ahm/cli.go:148, 165, 228-237` (scope list also appears in flag
  help text)
- `internal/ahm/output.go:22-33`, `internal/ahm/output_test.go`

## Acceptance Notes

- [ ] Scope list is defined once and used by validation, error message, and
      ideally flag help text.
- [ ] No test-only production helpers remain.
- [ ] `just ci` passes.
