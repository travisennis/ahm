# Add live ahm context improve procedure scope

## Summary

Add `ahm context improve`: the codebase-audit procedure currently installed
as the `finding-improvements` skill, emitted from the binary with live
backlog data so audits do not duplicate existing tasks.

## Problem

The procedure lives in
`internal/templates/workflow/finding-improvements-SKILL.md` (~469 lines,
the largest of the three skills), installed to
`.agents/skills/finding-improvements/SKILL.md`. Per the 156a ADR, it becomes
a scoped context. The recurring failure mode of static audits is proposing
work that already exists in the backlog; live state fixes that.

## Design

Output = procedure instructions (adapted from the SKILL.md body — preserve
its hard rules verbatim in substance: senior-advisor/read-only posture,
never modify source, every accepted finding becomes a fully self-contained
`ahm` task, never reproduce secret values) interleaved with live state:

- Existing active task titles with ids and labels (compact lines) — the
  dedupe list. If the active set is large, cap with a pointer to
  `ahm task list`.
- Label vocabulary (`ahm task labels` data) so new tasks use consistent
  `type:*` / `area:*` / `risk:*` labels.
- Validation findings, so known workflow issues are not re-reported.

Mechanics: same pattern as `ahm context groom` (task 156c) — extend the
scope registry in `internal/ahm/context.go`, separate instruction body from
live sections in the scoped report struct for JSON parity, compose live
data in Go. If 156c lands first, follow its structure exactly.

## Relevant Files

- `internal/templates/workflow/finding-improvements-SKILL.md` — source
  content (deleted later by 156g; absorb it here)
- `internal/ahm/context.go` — scope registry and scoped report shape
- `internal/ahm/context_test.go`
