# Support task create body input from file or stdin

## Summary

Add rich body input support to `ahm task create` so agents and scripts can create complete task records in one command while `ahm` continues to own task ID allocation, front matter, file placement, and index regeneration.

## Problem

`ahm task create` currently creates valid task metadata but only accepts a short description. Callers that need required task sections such as Problem, Relevant Files, Fix Direction, and Acceptance Notes must patch the created Markdown file afterward. That makes task creation a two-step workflow and increases the chance of stale indexes or inconsistent task shape.

## Relevant Files

- `internal/ahm/cli.go` for task create flag parsing and file creation behavior.
- `internal/ahm/cli_test.go` for CLI coverage.
- `docs/cli.md` for user-facing command documentation.
- `.agents/TASKS.md` if the repository task workflow should mention rich task body creation.

## Fix Direction

Support a `--body-file <path>` flag on `ahm task create`. The provided Markdown should replace the generated placeholder body content after the task title/front matter boundary while preserving the current title, ID allocation, front matter fields, active task location, and index behavior.

Support `--body-file -` if practical so callers can pipe task content through stdin:

```bash
ahm task create "Cache Immutable Tool Definitions For Agent Turns" \
  --priority P2 \
  --effort M \
  --labels "type:task, area:agent, area:tools" \
  --body-file -
```

Keep this task focused on body input. Do not add a broad set of structured section flags as part of this task. Treat `ahm task create --edit` as a separate follow-up feature, not part of this implementation.

Decide and document whether `--body-file` content is everything after the H1 or the whole Markdown body excluding front matter. Prefer the simpler contract that is easiest to explain and test, and make validation errors explicit for unreadable files or invalid stdin usage.

Because this changes user-visible CLI behavior, update `docs/cli.md` in the same change. If the behavior is considered a durable workflow contract, create or update an ADR before implementation.

## Acceptance Notes

- [ ] `ahm task create` still allocates the next task ID exactly as today.
- [ ] Generated front matter is written exactly as today for title, status, priority, effort, labels, `exec_plan`, and `depends_on`.
- [ ] `--body-file <path>` uses the provided Markdown content instead of the default placeholder body.
- [ ] `--body-file -` reads body content from stdin, or the implementation documents why stdin support is intentionally deferred.
- [ ] Task indexes are regenerated after creation exactly as current `task create` behavior requires, or docs clearly state the caller's responsibility if that is not current behavior.
- [ ] CLI docs describe the body-file contract, including whether the file starts after the H1 or represents the whole body excluding front matter.
- [ ] Tests cover body file creation, stdin behavior if supported, unreadable body files, and preservation of current ID/front matter/index behavior.
