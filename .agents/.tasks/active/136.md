# add govulncheck to CI verification suite

## Summary

govulncheck ./... is documented in CONTRIBUTING.md as `just vuln` but is not wired into the `just ci` read-only verification suite or the GitHub Actions CI workflow. The single runtime dependency (cobra v1.10.2) could have known vulnerabilities that go undetected until noticed manually.

## Evidence

- CONTRIBUTING.md — lists `just vuln` as a manual recipe: `govulncheck ./...`
- .github/workflows/ — CI workflow runs tests, vet, lint, and release checks but never govulncheck.
- `just ci` recipe — runs go test, go vet, golangci-lint, go mod tidy check, and gofmt check, but not govulncheck.

## Fix Direction

Add govulncheck ./... to the `just ci` recipe. Update the .github/workflows/ CI config to also run govulncheck ./... (or `just vuln`).

Consider whether govulncheck should fail the build or just warn. For a CLI tool with one runtime dependency, failing the build on vulnerabilities is appropriate — the blast radius is small.

Do NOT change the `just vuln` recipe. Do NOT add dependency scanning beyond govulncheck.

## Acceptance Notes

- [ ] `just ci` includes a govulncheck ./... step that fails the recipe on vulnerability findings.
- [ ] CI workflow in .github/workflows/ includes the same check.
- [ ] `just vuln` still works as a standalone recipe.
