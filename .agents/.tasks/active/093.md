# Add label-focused task listing commands

## Problem

Task labels exist, but the CLI needs a better API for using them directly. Current task selection is too status-first and requires users or agents to infer label capabilities from generated indexes or raw files.

The desired direction is a clearer label workflow: list tasks by label, identify labels that make work ready or routeable, and list the label vocabulary currently present in the repository.

## Relevant Files

- `internal/ahm/cli.go`
- `internal/ahm/task_commands.go`
- `internal/ahm/tasks.go`
- `internal/ahm/indexes.go`
- `docs/cli.md`
- `.agents/TASKS.md`
- Task command tests under `internal/ahm/`

## Fix Direction

Design and implement a small label API for task commands. Candidate behavior:

- `ahm task list --label <label>` with repeatable or comma-separated labels.
- A ready-work filter that combines queue readiness with one or more labels.
- `ahm task labels` or equivalent to list labels used by active tasks, with counts.

Keep the command shape boring and scriptable. Avoid inventing a large query language unless a concrete use case forces it.

## Acceptance Notes

- [ ] Users can list tasks matching one or more labels from the CLI.
- [ ] Users can list ready work filtered by label without manually reading generated indexes.
- [ ] Users can list labels currently used in task files, ideally with active/open counts.
- [ ] Output is deterministic and covered by task command tests.
- [ ] `docs/cli.md` and workflow guidance document the label workflow.
