# Fix case-sensitive README.md/index.md exclusion in ADR file discovery

## Summary

`adrFilePaths` uses a case-sensitive switch to exclude `README.md` and `index.md`. On case-insensitive filesystems, case variants are not excluded and produce malformed ADR findings.

## Problem

`adrFilePaths` uses a case-sensitive switch to exclude `README.md` and `index.md` from ADR file discovery. On case-insensitive filesystems (macOS default), a file named `readme.md` or `ReadMe.md` would not be excluded, would be parsed as an ADR, fail filename validation, and produce a malformed finding.

## Evidence

- **File:** `internal/ahm/adrs.go`, line 53
- **Context:** `switch entry.Name() { case "README.md", "index.md": continue }`

## Fix Direction

Use `strings.EqualFold` for the exclusion check instead of a case-sensitive switch:

```go
if strings.EqualFold(entry.Name(), "README.md") || strings.EqualFold(entry.Name(), "index.md") {
    continue
}
```

## Acceptance Notes

- [ ] `adrFilePaths` excludes case-variant names (`readme.md`, `ReadMe.md`, `README.MD`, `INDEX.md`) when iterating ADR directories.
- [ ] Existing case-sensitive exclusion still works.
- [ ] Test creates a case-variant `README.md`/`index.md` in the ADR directory and confirms it is skipped.
