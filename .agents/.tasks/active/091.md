# Use deslop review workflow for all agents

## Summary

Make ahm task work --review use the repo-owned deslop review workflow consistently across Cake, Codex, and Cursor instead of relying on opaque agent-native review semantics.

## Problem

`ahm task work --review` currently has agent-specific review semantics:

- Cake runs the repo-owned `deslop` skill.
- Codex uses `codex review --uncommitted`, whose behavior and output contract are opaque to AHM.
- Cursor uses a generic ask-mode review prompt.

That makes review behavior inconsistent across agents. AHM should own the review contract so `--review` means the same thing regardless of the selected agent.

## Relevant Files

- `internal/ahm/task_commands.go`
- `internal/ahm/task_commands_test.go`
- `docs/cli.md`
- `docs/testing.md`
- `.agents/skills/deslop/SKILL.md`
- `scripts/capture-agent-fixtures.sh`
- `internal/ahm/testdata/agents/`

## Fix Direction

- Define the stable review contract as: `ahm task work --review` runs the AHM deslop review workflow against current uncommitted changes.
- Keep Cake using its native `--skills deslop` path if that remains the best way to invoke the skill.
- Change Codex review orchestration away from `codex review --uncommitted` and toward a normal Codex execution path that explicitly reads and follows `.agents/skills/deslop/SKILL.md`.
- Change Cursor review orchestration away from the generic ask-mode review prompt and toward the same deslop-backed review prompt.
- Parse Codex and Cursor structured output so review feedback passed back into the original session is human-readable feedback, not raw JSONL.
- Update CLI and testing docs to describe the deslop-backed review contract for every review-capable agent.
- Update tests and live smoke coverage for the new argument builders and output parsing.

## Acceptance Criteria

- `ahm task work --review --agent cake` still runs the deslop review workflow.
- `ahm task work --review --agent codex` runs deslop through Codex's normal execution path, not opaque `codex review` behavior.
- `ahm task work --review --agent cursor` runs deslop through Cursor's normal execution path, not a generic review prompt.
- Review feedback passed into the original work session is plain actionable feedback.
- `docs/cli.md` documents consistent deslop-backed review semantics.
- `docs/testing.md` reflects the smoke-test expectations for the updated review path.
- Focused tests and `just ci` pass.
- Because this task is `area:agent` and `risk:external-service`, Acceptance Notes include live-run evidence: agent versions and transcript snippets from the affected agents.
