# Stop installing managed skills, migrate consumers, update docs

## Blocker

Blocked until the command-based replacements exist: `ahm prime` (156b), the
delegation commands `ahm task groom` (156c) and `ahm audit` (156d), the
task-work review fold-in (156e), and `ahm onboard` (156f). Removing the
skills before their replacements ship would leave consumers with neither.

## Summary

Remove the three managed skills from the template registry and install
targets, migrate existing consumer repositories via the obsolete-file
mechanism, and update all agent guidance and reference docs to the
prime/delegation/onboard model.

## Problem

`ahm init`/`ahm upgrade` install and hash-track three skills
(`internal/templates/templates.go` lines 24-26 →
`.agents/skills/{preflight,grooming-backlog,finding-improvements}/SKILL.md`,
directories created in `internal/ahm/install.go` ~lines 221-223). Per the
156a ADR their content now ships in the binary, so the installed copies
become obsolete managed files. ADR 011 already built the removal machinery
when workflow guide templates were withdrawn; the `deslop` skill entry in
`obsoleteManagedFiles` (`internal/ahm/install.go` lines 52-55) is the exact
precedent, including `EmptyDirs` cleanup.

## Work Direction

Code:

- Remove the three skill entries from the template registry in
  `internal/templates/templates.go`; delete
  `internal/templates/workflow/{preflight,grooming-backlog,finding-improvements}-SKILL.md`
  (confirm 156c/156d/156e absorbed their content first); update
  `internal/templates/templates_test.go` (currently asserts managed files
  are "only skills").
- Add the three SKILL.md targets to `obsoleteManagedFiles` with `EmptyDirs`
  for their directories and `.agents/skills/` itself once empty.
- Stop creating `.agents/skills/*` directories in install
  (`internal/ahm/install.go` ~lines 221-223).
- Migration semantics follow the existing mechanism exactly: upgrade
  removes files whose on-disk content matches the recorded managed hash,
  reports locally edited ones as conflicts unless `--force`, and `init`
  never creates `.agents/skills/`. Dry-run previews all of it.
- Template/binary version implications: consult
  `docs/guides/workflow-upgrades.md` and preserve binary/template version
  separation.

Docs and guidance (consult `docs/guardrails/workflow-state-and-file-formats.md`
before editing):

- This repo's `AGENTS.md`: shrink "Managed Work Intake With `ahm`" to match
  the new model (prime as entry point; scoped contexts for
  task/plan/adr/research/docs; `ahm task groom` and `ahm audit` for the
  delegated procedures; preflight embedded in the task-work review); remove
  the
  workflow-state guardrail line "Do not remove or stop installing agent
  skills unless that is explicitly in scope" (this task is that explicit
  scope); keep the Operating Loop and Workflow Routing sections — they are
  project-owned policy, not ahm output.
- `docs/references/workflow-spec.md`, `docs/references/glossary.md`,
  `docs/references/cli/commands.md`, `docs/references/cli/task-commands.md`,
  `docs/guides/workflow-upgrades.md`, `docs/cli.md`: remove/replace skill
  references (grep for "skills" and "SKILL").
- `docs/guardrails/workflow-state-and-file-formats.md` and other guardrail
  docs that mention installed skills.
- This repo dogfoods ahm: after the change, run the upgrade here so
  `.agents/skills/` is removed from this repository too.
