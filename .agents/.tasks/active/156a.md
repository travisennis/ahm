# Write ADR: replace managed skills with command procedures and ahm prime

## Summary

Write and get accepted the ADR that records the decision to stop installing
managed agent skills, expose their procedures as live scoped contexts, add a
read-only `ahm prime` session-start command, and replace
`ahm agents suggestions` with `ahm onboard`.

## Problem

ADR 011 stopped installing workflow guide templates but explicitly deferred
the skills decision: "Keep agent skills installed and managed until there is
an explicit decision to replace that mechanism." That decision has now been
made (design discussion, 2026-07-01) and must be recorded before the
implementation tasks (156b-156g) proceed, because it changes several
compatibility surfaces: the CLI command set, the installed file set, embedded
templates, and agent guidance.

## Decisions To Record

Modeled on beadwork's `bw prime` / `bw onboard`
(github.com/jallum/beadwork). Record all of the following:

1. Stop installing and hash-managing `.agents/skills/preflight/SKILL.md`,
   `.agents/skills/grooming-backlog/SKILL.md`, and
   `.agents/skills/finding-improvements/SKILL.md`. Procedure content lives
   only in the binary. No per-repo customization of procedure content â€” no
   override mechanism.
2. Expose the procedures as live, state-aware scoped contexts:
   `ahm context groom`, `ahm context improve`, `ahm context preflight`
   (short names chosen deliberately). Scoped contexts remain the instruction
   and procedure channel.
3. Add `ahm prime` as the canonical agent session-start command. In this
   effort it is strictly read-only: warnings, live repository/backlog state,
   and the managed-work routing table. ADR 013 reserves `ahm prime` for
   records sync; task 143 later layers the sync step onto this same command.
   The ADR must state this coordination explicitly so ADR 013 and this ADR
   do not conflict.
4. Unscoped `ahm context` becomes a usage error pointing at `ahm prime` and
   listing valid scopes. Breaking CLI change.
5. Remove the `ahm agents` command group (`ahm agents suggestions`,
   including `--all`). Add `ahm onboard`, which prints a minimal paste-ready
   AGENTS.md snippet. This supersedes ADR 002 (advisory agents suggestions).
6. Add a low-severity `ahm doctor` check: AGENTS.md exists but does not
   reference `ahm prime` â†’ suggest running `ahm onboard`. This is the answer
   to snippet drift.
7. Content architecture (the rationale section):
   - Pasted snippet (always in context, static): bootstrap invariant
     ("ALWAYS run `ahm prime` before starting work; re-run after
     compaction") plus safety invariants (never hand-edit generated
     indexes; use `ahm task` / `ahm adr` for state changes). Nothing else.
   - `ahm prime` (dynamic, versioned with binary): live state plus the
     managed-work intake routing table, which moves out of AGENTS.md.
   - Scoped contexts: full instructions and procedures.
   - Operating-loop and workflow-routing guidance is project-owned; ahm
     never emits it.
8. JSON parity for all new outputs, implemented as structured data in Go
   with text rendered from the same struct (not template-executed
   subcommand output).
9. Migration: upgrade removes clean previously-managed skill files via the
   existing obsolete-managed-files mechanism (deslop precedent,
   `internal/ahm/install.go`), reports locally edited ones as conflicts
   unless `--force`; `ahm init` never creates `.agents/skills/`.
