# Add examples to --help for all commands

## Summary

Only `task comment` has examples in its `--help` output. Every other command's `--help` is just the `Short` description + usage + flags. The docs in `docs/references/cli/` already have excellent examples — they just never made it into the cobra `Long` fields.

An agent running `ahm task create --help` sees flags but no example invocations. The `writing-clis-for-agents` skill says: "Every `--help` includes Examples with real invocations. Examples do more than prose for pattern-matching."

## Evidence

- `internal/ahm/task_comment.go` — `taskCommentCommand` has a `Long` field with three examples. This is the gold standard.
- `internal/ahm/task_commands.go` — `task create`, `task list`, `task show`, `task start`, `task complete`, `task cancel`, `task accept`, `task reopen`, `task work`, `task migrate`, `task labels`, `task next`, `task ready`, `task blocked` — none have `Long` fields with examples.
- `internal/ahm/adr_commands.go` — `adr create`, `adr list`, `adr show`, `adr accept`, `adr reject`, `adr deprecate`, `adr supersede`, `adr migrate` — none have `Long` fields with examples (except `adr migrate` which has a `Long` but no examples).
- `internal/ahm/cli.go` — `init`, `upgrade`, `status`, `doctor`, `index`, `version`, `context`, `agents` — none have `Long` fields with examples.
- `docs/references/cli/commands.md` and `docs/references/cli/task-commands.md` — already contain well-crafted examples for every command.

## Fix Direction

Add a `Long` field to every cobra command that includes an `Examples:` section with real invocations. Source the examples from the existing docs in `docs/references/cli/`. Prioritize write commands first (`task create`, `task work`, `task complete`, `task cancel`, `adr create`, `adr supersede`) since those are where agents need the most guidance.

Keep examples short — 2-4 invocations per command showing the most common patterns. Don't duplicate the full docs reference; the `--help` output should be enough for an agent to construct a valid command without reading the docs.
