# Upgrade codex agent to full task work orchestration

## Decision

Use Codex non-interactive JSONL mode:

- Work args: `codex exec --json <prompt>`.
- Resume args: `codex exec resume --json <sessionID> <prompt>`.
- Session ID source: first JSONL event
  `{"type":"thread.started","thread_id":"..."}`.
- Final message source: `item.completed` events whose nested `item.type` is
  `agent_message` and whose nested `item.text` contains the message.
- Review support should use Codex's native non-interactive review command,
  `codex review --uncommitted <prompt>`, not a generic fresh work prompt.

Local verification on 2026-06-12 with Codex CLI 0.138.0 confirmed
`codex exec --json` emits JSONL with `thread.started`, `turn.started`,
`item.completed`, and `turn.completed`. `codex exec resume --help` confirms
`codex exec resume --json <SESSION_ID> <PROMPT>`.

Official OpenAI Codex manual checked:

- `codex exec` is documented as the non-interactive command.
- `--json` makes stdout a JSON Lines stream.
- Documented event types include `thread.started`, `turn.started`,
  `turn.completed`, `turn.failed`, `item.*`, and `error`.
- The sample JSONL stream shows `thread_id` on `thread.started` and final
  agent text on an `item.completed` event with `item.type: "agent_message"`.

## Problem

`ahm task work --agent codex` is basic handoff only: it runs
`codex exec <prompt>` without session capture, so `--review`, `--complete`,
and `--commit` print warnings and are skipped. The Codex CLI supports
machine-readable output and session resume, so codex can be upgraded to full
orchestration like `cake`.

## Relevant Files

- `internal/ahm/task_commands.go` — `codex` case in `parseTaskWorkAgent`,
  `taskWorkAgent` struct, `taskWorkWithSession`.
- `internal/ahm/task_commands_test.go` — agent selection and orchestration
  tests.
- `docs/cli.md` — `task work` supported-agents table (codex row currently says
  "Basic handoff only" / "Not supported").
- `docs/adr/006-task-work-agent-delegation.md` — says session-capable agents
  are "currently cake".

## Fix Direction

- Implement the verified Codex CLI contract:
  - `codex exec --json <prompt>` emits JSONL events; the session/thread ID
    arrives in `thread.started.thread_id`.
  - `codex exec resume --json <sessionID> <prompt>` resumes a prior session
    and preserves JSONL output.
- Codex output is JSONL, not the single JSON object cake returns, so
  `parseSessionID` needs a codex-specific parser that scans lines for the ID
  event. The `taskWorkAgent` per-agent parser hooks already allow this; do not
  force codex output through the cake parser.
- Add review support with `codex review --uncommitted <review prompt>`.
  This command is designed for non-interactive code review. It does not expose
  a documented `--json` flag in the installed help, so parse plain stdout as
  review feedback for now unless a later CLI version adds structured review
  output.
- Preserve the existing tee-to-terminal behavior in `taskWorkWithSession` so
  JSONL streaming output remains visible to the user.
- Update `docs/cli.md` (codex row in the supported-agents table, examples) and
  amend `docs/adr/006-task-work-agent-delegation.md` since the
  session-capable agent set changes.

## Acceptance Notes

- [ ] `ahm task work <id> --agent codex` captures a session/thread ID from

- [ ] `--complete` and `--commit` resume the codex session instead of warning.
- [ ] `--review` runs `codex review --uncommitted` and resumes the captured
      session with any review feedback.
- [ ] `--dry-run` previews the codex executable and args without invoking it.
- [ ] Unit tests cover codex arg construction and JSONL session-ID parsing.
