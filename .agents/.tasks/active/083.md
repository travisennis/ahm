# Upgrade codex agent to full task work orchestration

## Problem

`ahm task work --agent codex` is basic handoff only: it runs
`codex exec <prompt>` without session capture, so `--review`, `--complete`,
and `--commit` print warnings and are skipped. The Codex CLI supports
machine-readable output and session resume, so codex can be upgraded to full
orchestration like `cake`.

## Relevant Files

- `internal/ahm/task_commands.go` — `codex` case in `parseTaskWorkAgent`,
  `taskWorkAgent` struct, `taskWorkWithSession`.
- `internal/ahm/task_commands_test.go` — agent selection and orchestration
  tests.
- `docs/cli.md` — `task work` supported-agents table (codex row currently says
  "Basic handoff only" / "Not supported").
- `docs/adr/006-task-work-agent-delegation.md` — says session-capable agents
  are "currently cake".

## Fix Direction

- Verify the installed Codex CLI's headless interface before locking in flags.
  As of recent versions:
  - `codex exec --json <prompt>` emits JSONL events; the session/thread ID
    arrives in an early event (e.g. `thread.started` with `thread_id`).
  - `codex exec resume <sessionID> <prompt>` resumes a prior session.
- Codex output is JSONL, not the single JSON object cake returns, so
  `parseSessionID` needs a codex-specific parser that scans lines for the ID
  event. The `taskWorkAgent` per-agent parser hooks already allow this; do not
  force codex output through the cake parser.
- Decide review support: codex has no deslop equivalent, so an independent
  review pass would be a fresh `codex exec --json <review prompt>` whose final
  agent message is the feedback. If extracting that final message is not
  reliable, ship sessions + `--complete` + `--commit` first and leave
  `supportsReview` false with the existing warning; record the choice in the
  ADR amendment.
- Preserve the existing tee-to-terminal behavior in `taskWorkWithSession` so
  JSONL streaming output remains visible to the user.
- Update `docs/cli.md` (codex row in the supported-agents table, examples) and
  amend `docs/adr/006-task-work-agent-delegation.md` since the
  session-capable agent set changes.

## Acceptance Notes

- [ ] `ahm task work <id> --agent codex` captures a session/thread ID from
      codex JSONL output.
- [ ] `--complete` and `--commit` resume the codex session instead of warning.
- [ ] `--review` either runs a codex review pass or the documented warning
      behavior is intentional and recorded in ADR 006.
- [ ] `--dry-run` previews the codex executable and args without invoking it.
- [ ] Unit tests cover codex arg construction and JSONL session-ID parsing.
- [ ] `docs/cli.md` and ADR 006 updated; `just ci` passes.
