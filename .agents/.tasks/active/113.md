# Skip Go pre-commit checks when only non-Go files are staged

## Summary

The pre-commit hooks (fmt-check, tidy-check, go test, golangci-lint) run on every commit regardless of what changed. When only markdown, task, or doc files are staged, running `go test` and `golangci-lint` is wasted time (~24s). Smart wrapper scripts that inspect staged files can skip Go-specific checks when no `.go` files are staged, cutting doc-only commit latency to ~2s.

## Evidence

- `.pre-commit-config.yaml` — all 4 Go hooks use `pass_filenames: false` and `entry: just <recipe>`, so they always run.
- Real commit timing: the commit for tasks 109-111 (5 markdown files, zero `.go` changes) took ~24s waiting for `go test` + `golangci-lint` to complete.
- `pass_filenames: false` means the hooks receive no information about which files changed.

## Fix Direction

Create 4 wrapper scripts in a `hooks/` (or `scripts/`) directory that check `git diff --cached --name-only` for `.go` files and exit 0 if none found. Update `.pre-commit-config.yaml` to point at the wrappers instead of raw `just` commands.

Wrapper pattern (example for go test):
```sh
#!/bin/sh
# Skip if no .go files are staged.
if ! git diff --cached --name-only | grep -q '\.go$'; then
  exit 0
fi
just test
```

Hooks to wrap:
- `go-fmt-check` → `scripts/hooks/go-fmt-check.sh`
- `go-mod-tidy-check` → `scripts/hooks/go-tidy-check.sh`
- `go-test` → `scripts/hooks/go-test.sh`
- `golangci-lint` → `scripts/hooks/go-lint.sh`

Update `.pre-commit-config.yaml` to point `entry` at the wrapper scripts instead of `just <recipe>`. Keep `language: system` and `pass_filenames: false`.

Decision: wrap all four hooks for consistency.

Files in scope: `.pre-commit-config.yaml`, new `scripts/hooks/` directory.
Out of scope: changing the `just` recipes themselves, moving hooks to pre-push stage.
