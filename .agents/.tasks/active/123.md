# Add JSON tags to Task struct for consistent field naming

## Summary

Task JSON uses Go struct field names (PascalCase, no `json:` tags) while ADR JSON uses explicit lowercase tags. An agent parsing JSON from both `task list --json` and `adr list --json` has to handle two different key conventions.

## Evidence

Task JSON (from `task list --json`):
```json
{"ID": "109", "Title": "...", "Status": "Completed", "Priority": "P0"}
```

ADR JSON (from `adr list --json`):
```json
{"id": "001", "title": "...", "status": "accepted", "date": "2026-05-29"}
```

- `internal/ahm/tasks.go` — `Task` struct has no `json:` tags on any field. Go's `json.Marshal` uses the struct field name as-is, producing PascalCase keys.
- `internal/ahm/adr_commands.go` — `adrListEntry` struct has explicit `json:` tags (`json:"id"`, `json:"title"`, etc.), producing lowercase keys.
- `internal/ahm/task_list.go` — `taskLabelSummary` struct has explicit `json:` tags, also lowercase.

So within the same CLI, three different JSON naming conventions exist: PascalCase (Task), lowercase (ADR), and lowercase (taskLabelSummary). The Task struct is the outlier.

## Fix Direction

Add `json:` tags to the `Task` struct matching the lowercase convention used by ADR and label summaries:

```go
type Task struct {
    ID          string            `json:"id"`
    Title       string            `json:"title"`
    Status      string            `json:"status"`
    Priority    string            `json:"priority"`
    Effort      string            `json:"effort"`
    Labels      string            `json:"labels"`
    ExecPlan    string            `json:"exec_plan"`
    DependsOn   []string          `json:"depends_on"`
    Created     string            `json:"created"`
    Updated     string            `json:"updated"`
    Parent      string            `json:"parent"`
    ExternalRef string            `json:"external_ref"`
    Extra       map[string]string `json:"extra"`
    Path        string            `json:"path"`
    Bucket      string            `json:"bucket"`
    Body        string            `json:"body"`
}
```

This is a **breaking change** for JSON consumers. Update all tests that assert on JSON keys. Update `docs/references/cli/global-contract.md` and `docs/references/cli/task-commands.md` to document the new key names.
