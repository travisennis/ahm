# Rework records migrate to preserve normal Git tracking

## Problem

`ahm records migrate` currently moves source records into gitignored `.ahm/`,
seeds a custom ref, and tells the user to untrack legacy paths. The command must
instead perform a namespace/layout migration whose source records remain part
of normal project Git history.

## Relevant Files

- `internal/ahm/records_migrate.go`
- `internal/ahm/records_migrate_test.go`
- `internal/ahm/install.go`
- `docs/references/cli/commands.md`

## Fix Direction

Keep `ahm records migrate` as the explicit trigger. Preserve dry-run,
conflict detection, atomic writes, resumability, project-owned `.agents/`
content, and the no-stage/no-HEAD-mutation boundary. Move legacy task,
research, ExecPlan, and config state into committed `.ahm/` paths without
seeding custom refs or ignoring source records. Move every supported attachment
safely. Print precise normal-Git follow-up instructions.

## Acceptance Notes

- [x] Legacy `.agents/` source records migrate to committed `.ahm/` paths.
- [x] Migration performs no fetch, push, custom-ref creation, staging, branch commit, index write, or `HEAD` move.
- [x] Dry-run reports moves, tracking changes, config changes, and user-run Git follow-up accurately.
- [x] Interrupted migration is resumable and conflicting targets are never overwritten.
- [x] Project-owned `.agents/` content remains untouched.
