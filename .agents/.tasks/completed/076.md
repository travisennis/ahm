# Add ahm adr create command

## Blocker

Blocked on task 075 for the ADR model/parser foundation. Do not start
implementation until task 075 is complete.

## Summary

Add `ahm adr create <title>` mirroring `ahm task create`: allocate the next
ID, write a MADR-profile record seeded from an embedded MADR template into
`docs/adr/NNN-kebab-slug.md`, and regenerate the ADR index.

## Background

Design per ADR 009 (task 074) and
`.agents/.research/topics/madr-adr-management-design.md`. Model and rendering
come from task 075.

## Implementation Notes

- Embed a constrained-profile MADR 4.x template used to seed new records. Per
  ADR 009, decide whether it lives in `internal/templates` (embedded only, not
  an installed managed file) or as a rendered string in `internal/ahm`.
- Flags mirroring `task create`: `--status` (default `proposed`),
  `--description`/`-d` seeding Context and Problem Statement, and
  `--body-file <path|->` providing the full body below the generated H1
  (mutually exclusive with `--description`). Consider `--decision-makers`.
- Stamp `date:` with the current date; build the kebab-case slug from the
  title; title becomes the H1.
- Honor `--dry-run` (print target path and ID without writing) and output
  modes consistently with `task create`; print the new ID on success.
- Regenerate indexes after creation so no separate `ahm index` is needed.
- Update `docs/cli.md` with an `adr create` section in the same change.
