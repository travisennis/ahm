# Add ahm records sync command surface

## Blocker

Blocked until tasks 139 and 140 provide the private-ref plumbing and records
metadata model this CLI surface wraps.

## Summary

Add ahm records status, pull, push, sync, and doctor behavior for explicit GitHub-backed refs/ahm/records synchronization and diagnostics.

## Relevant Records

- ADR: `docs/adr/013-use-ref-backed-workflow-record-storage.md`
- ExecPlan: `.agents/exec-plans/active/138-ref-backed-workflow-records.md`

## Work Direction

Add an explicit `ahm records` command family for ref-backed workflow records.
The command surface should include status reporting, pull/fetch,
push, sync, and diagnostics for missing remotes, unsupported remotes,
credential failures, non-fast-forward remote refs, and stale local state. Initial
remote support is GitHub only.

## Acceptance Notes

- [x] `ahm records status` reports local/remote records state without mutating
      files.
- [x] `ahm records pull`, `push`, and `sync` work against
      `refs/ahm/records` using the explicit records remote; GitHub URLs are
      accepted for user-facing support and local filesystem remotes are covered
      in hermetic tests.
- [x] JSON/plain/text output shapes are documented and tested.
- [x] Remote failures produce actionable diagnostics for missing remotes,
      unsupported remotes, stale local state, authentication-shaped failures,
      and non-fast-forward pushes.
