# Invert ahm task work defaults: review/commit on by default with --no-* opt-out

## Summary

Flip the `ahm task work` orchestration defaults so the common workflow is the
default. Today review and commit handoffs are opt-in via `--review` and
`--commit` (default `false`). In practice they are wanted on nearly every run,
so the defaults are inverted relative to actual usage. Make review and commit
run by default, remove the positive flags, and add `--no-review` / `--no-commit`
opt-out flags.

Depends on task 119 (remove `--complete`), which should land first.

## Motivation

`ahm task work <id>` is almost always invoked as
`ahm task work <id> --review --commit`. Requiring the positive flags every time
is friction. Defaulting them on with explicit opt-out matches real usage.

## Resolved decisions

1. `--complete` is out of scope here; it is removed entirely in task 119
   because completion is agent-driven via the base work prompt.
2. Remove the existing `--review` and `--commit` flags and replace them with
   `--no-review` and `--no-commit`. Review and commit are on by default.
3. Agent-capability posture: `ahm task work` only supports agents that can run
   the full default flow. All four supported agents (cake, codex, cursor,
   claude) already set `supportsSessions` and `supportsReview` true, so the
   capability-warning branches are dead code. Remove the
   "not supported by agent" warnings. The now-unused `supportsSessions` and
   `supportsReview` fields and their gating branches should be removed as well
   unless a concrete need to keep them is found.

## Scope / Requirements

- In `internal/ahm/task_commands.go`: remove `--review` and `--commit` flag
  registration; add `--no-review` and `--no-commit`. Update `taskWorkArgs`
  accordingly (store the no-* booleans or resolve to effective review/commit).
- In `internal/ahm/task_work.go`: resolve effective review/commit (default on
  unless `--no-*`), pass them to `taskWorkWithSession`, and remove the
  capability-warning branches. Update the dry-run preview so review/commit
  appear by default and disappear when opted out.
- In `internal/ahm/task_session.go`: review/commit run by default; the only gate
  is the `--no-*` opt-out (and step success). Remove the `agent.supportsReview`
  gate in `taskWorkWithSession`.
- In `internal/ahm/task_agents.go`: remove the now-unused `supportsSessions` and
  `supportsReview` fields and their assignments unless a concrete need remains.
- Remove the dead non-session fallback path
  (`taskWorkRunCommand` direct branch) only if it is genuinely unreachable after
  this change; otherwise leave it and note why.
