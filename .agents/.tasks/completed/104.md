# Move exec-plan sections cache from global to validation-report scope

## Summary

`execPlanSectionsCache` is a package-level `var` in `internal/ahm/validation.go` that's mutated without synchronization. Two problems:

1. **Concurrent validation races.** If `status` and `doctor` were ever called concurrently, they'd race on the global map.
2. **Test leakage.** Tests in the same binary share the global. A test that parses exec plans leaves cached data for the next test.

The cache memoizes exec-plan section parsing per validation run. It belongs on the validation report or an explicit per-run cache object, not as a global variable.

## Files in scope

- `internal/ahm/validation.go` — the cache variable, `getCachedExecPlanSections`, `clearExecPlanSectionsCache`, `execPlanHasRetrospective`, and all callers
- `internal/ahm/validation_test.go` — check for any test references to the cache

## Current state

The cache lives at `validation.go:422`. Usage pattern:

1. `validateWorkflowScoped` (line 50) calls `clearExecPlanSectionsCache()` → sets global to `nil`
2. `validateTaskExecPlans` (line 237) calls `execPlanHasRetrospective(path)` for completed tasks with exec plans
3. `execPlanHasRetrospective` (line 443) calls `getCachedExecPlanSections(path)`
4. `validateExecPlans` (line 262) calls `getCachedExecPlanSections(path)` for every exec-plan file
5. `getCachedExecPlanSections` (line 424) lazily populates the global map

## Steps

### Step 1: Add a cache field to `validationReport`

```go
type validationReport struct {
    OK       bool                `json:"ok"`
    Errors   []validationFinding `json:"errors"`
    Warnings []validationFinding `json:"warnings"`
    Info     []validationFinding `json:"info"`
    execPlanSectionsCache map[string]map[string]execPlanSection  // unexported
}
```

### Step 2: Move cache operations onto `*validationReport`

Change `getCachedExecPlanSections` from a bare function to a method:

```go
func (r *validationReport) getCachedExecPlanSections(path string) (map[string]execPlanSection, error) {
    if r.execPlanSectionsCache == nil {
        r.execPlanSectionsCache = map[string]map[string]execPlanSection{}
    }
    if sections, ok := r.execPlanSectionsCache[path]; ok {
        return sections, nil
    }
    sections, err := parseExecPlanSections(path)
    if err != nil {
        return nil, err
    }
    r.execPlanSectionsCache[path] = sections
    return sections, nil
}
```

Remove the global `var execPlanSectionsCache` and the `clearExecPlanSectionsCache` function.

### Step 3: Update `execPlanHasRetrospective` to accept `*validationReport`

```go
func execPlanHasRetrospective(path string, report *validationReport) bool {
    sections, err := report.getCachedExecPlanSections(path)
    if err != nil {
        return false
    }
    return execPlanSectionHasBody(sections[normalizedExecPlanSection("Outcomes & Retrospective")])
}
```

### Step 4: Thread `report` through callers

- `validateWorkflowScoped` (line 47): remove `clearExecPlanSectionsCache()` call
- `validateTaskExecPlans` (line 223): update call to `execPlanHasRetrospective(plan, report)`
- `validateExecPlans` (line 262): update call to `report.getCachedExecPlanSections(path)`

All validators already receive `*validationReport` — only the cache-access functions need updating.
