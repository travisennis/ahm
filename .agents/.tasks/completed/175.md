# Make fresh ahm init create only the committed .ahm layout

## Summary

ahm init still creates the legacy .agents/ layout, so new adopters land on the layout being migrated away from and must immediately run ahm records migrate to reach committed .ahm/. This decision determines whether fresh installs should create the committed .ahm/ layout directly or whether legacy-first init is intentional. Surfaced during the task 172 release review.

## Acceptance Notes

- [x] Decision recorded: fresh init creates .ahm/ layout directly (2026-07-12, Travis)
- [x] Implement init to write .ahm/config.json and create .ahm/ directory structure directly
- [x] Populate managedFiles in templates.go with .ahm/ target paths
- [x] Update workflow spec to document .ahm-first init
- [x] Update CLI docs
- [x] Update release notes for the behavioral change
- [x] Update tests (TestManagedFilesAreEmpty renamed to TestManagedFilesContainScaffoldTargets, etc.)

## Relevant Files

- internal/templates/templates.go (managedFiles is empty; must be populated with .ahm/ targets)
- internal/ahm/install.go (init writes metadata to .agents/ahm.json; must write .ahm/config.json)
- internal/ahm/workflow_paths.go (derives records dir from metadata)
- internal/ahm/root.go (root detection checks .agents/ahm.json; .ahm/config.json already recognized)
- docs/references/workflow-spec.md (line 45 says install the managed .agents workflow state)
- docs/guardrails/workflow-state-and-file-formats.md

- internal/templates/templates_test.go (TestManagedFilesAreEmpty validates empty list)

## Fix Direction

Fresh ahm init must create only the committed .ahm/ layout directly. The legacy .agents/ record layout is not used for future installs; it remains only as the migration source for existing legacy repositories via ahm records migrate. Decision made 2026-07-12. Implementation requires: (1) populating managedFiles in templates.go with .ahm/ target paths for workflow scaffolding files, (2) updating install.go to write metadata to .ahm/config.json on fresh init, (3) updating the workflow spec to describe .ahm-first init, (4) updating CLI docs and release notes, (5) updating TestManagedFilesAreEmpty and init integration tests to expect .ahm/ output.
