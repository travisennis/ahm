# Fix task ID resolution to avoid substring matches

## Summary

Review issue 1. resolveTask uses unbounded substring matching, so commands such as task start 1 can become ambiguous or target the wrong task. Fix by returning exact ID matches immediately and using only constrained prefix matching as fallback. Acceptance: task commands resolve exact IDs deterministically, ambiguous prefixes error clearly, and tests cover exact, prefix, and ambiguous inputs.
