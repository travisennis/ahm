# Add ahm adr migrate for legacy ADR metadata

## Blocker

Blocked on task 075 for the ADR parser/renderer and task 079 for ADR index and
validation behavior. Migration should land after legacy-format findings have a
defined validation surface.

## Summary

Add `ahm adr migrate` to convert legacy ADR metadata (`# ADR NNN: Title`
heading plus `**Status:**` / `**Date:**` bold lines) into the constrained
MADR front matter profile, then run it on this repository's own ADRs so
self-hosted validation is clean.

## Background

Design per ADR 009 (task 074) and
`.agents/.research/topics/madr-adr-management-design.md`. Mirrors
`ahm task migrate` (`internal/ahm/task_migrate.go`). Migration is
metadata-only: decision history in body sections is never rewritten.

## Implementation Notes

- Conversion per file: derive ID/slug from the filename, take status and date
  from the bold lines, strip the `ADR NNN:` prefix from the H1, remove the
  consumed bold lines, and write MADR front matter. Legacy section bodies
  (Context / Decision / Rationale / ...) are preserved as-is.
- Status mapping: `Proposed`->`proposed`, `Accepted`->`accepted`,
  `Deprecated`->`deprecated`, `Superseded`->`superseded by ADR-NNN` resolved
  from the existing supersession note; if the replacement cannot be resolved
  unambiguously, report the file and require a manual fix instead of
  guessing. Map `Accepted, superseded in part by ADR NNN` per the ADR 009
  ruling on partial supersession.
- Idempotent: files that already carry MADR front matter are untouched;
  rerunning produces no changes.
- `--dry-run` lists files that would change, matching `task migrate`
  behavior; regenerate the ADR index after a real run.
- Migrate this repo's `docs/adr/` records in the same change so `ahm status`
  reports no legacy-format findings afterward.
- Update `docs/cli.md` with an `adr migrate` section in the same change.
