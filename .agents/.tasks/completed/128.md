# Add JSON output for task dep cycles and task dep tree

## Summary

`task dep cycles` and `task dep tree` always print text regardless of `--json` or `--plain` flags. An agent expecting JSON gets text. The docs note that `--plain` is "Ignored by commands with custom text output" but `--json` should work everywhere for consistency.

## Evidence

```
$ ahm task dep cycles --json
No dependency cycles found

$ ahm task dep tree 001 --json
001 [Completed] Fix task ID resolution to avoid substring matches
```

- `internal/ahm/task_deps.go` — `taskDepCycles()` always prints text via `fmt.Fprintln`. No check for `a.opts.json` or `a.opts.plain`.
- `internal/ahm/task_deps.go` — `taskDepTree()` always prints text via `fmt.Fprintf`. No check for output mode.
- Compare to `taskList()` in `task_list.go` which checks `a.opts.json` before printing text lines.

## Fix Direction

1. **`task dep cycles`**: When `--json` or `--plain` is set, emit a structured response. For no cycles: `[]`. For cycles: `[["001", "002", "001"], ...]` — an array of cycle paths, each path being an array of task IDs.

2. **`task dep tree`**: When `--json` or `--plain` is set, emit a structured tree. Options:
   - An adjacency list: `{"id": "001", "title": "...", "status": "Completed", "dependencies": [...]}`
   - A flat list of nodes with depth indicators
   
   The adjacency list (nested struct) is more natural for a tree. Define a struct:

   ```go
   type depTreeNode struct {
       ID           string        `json:"id"`
       Title        string        `json:"title"`
       Status       string        `json:"status"`
       Dependencies []depTreeNode `json:"dependencies,omitempty"`
   }
   ```

3. Keep text output unchanged for both commands.

## Acceptance Notes

- [x] `task dep tree <id> --json` returns a nested tree structure
- [x] `task dep tree <id> --plain` returns compact JSON

- [x] `just ci` passes

## Implementation

Modified `internal/ahm/task_deps.go`:

- `taskDepCycles()` now checks `a.opts.json || a.opts.plain` and emits `[][]string` via `a.emit()`, falling back to original text output. When no cycles exist and JSON/plain mode is active, returns `[]` instead of `null`.
- `taskDepTree()` now checks `a.opts.json || a.opts.plain` and builds a `depTreeNode` (nested struct with `id`, `title`, `status`, `dependencies`) via `buildDepTree()`, falling back to original text output.
- Added `depTreeNode` struct and `buildDepTree()` helper function.
