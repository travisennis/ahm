# Normalize CRLF line endings when reading workflow markdown

## Summary

Review issue 11. Front matter, migration, title, heading, and body helpers assume LF line endings and fail on CRLF files. Acceptance: task and workflow markdown parsing normalizes CRLF safely, tests cover CRLF task files, and rendered output behavior is documented or consistent.

  `taskMigrate`, `validateManagedFile`, `validateGeneratedIndexes`, `isStaleIndex`,
  and `install` (existing file comparison).
- [x] Added defense-in-depth CRLF normalization in `parseFrontMatter` and `splitFrontMatter`.
- [x] Added 9 CRLF-specific tests covering `readWorkflowFile`, `parseFrontMatter`,
  `splitFrontMatter`, `parseTask`, `migrateTaskFrontMatter`, `headingTitle`,
  `stripHeading`, `taskStatus`/complete round-trip, and `validateTaskFrontMatter`.
- [x] All existing tests pass; full CI passes (`just ci`).
