# Make committed .ahm the post-migration records layout

## Problem

Current path resolution treats `.ahm/` layout as a consequence of the
unreleased ref storage mode. Committed `.ahm/` records instead need one direct
post-migration layout, while legacy `.agents/` repositories continue to work
until the user runs migration.

## Relevant Files

- `internal/ahm/workflow_paths.go`
- `internal/ahm/install.go`
- `internal/ahm/root.go`
- `internal/ahm/validation.go`
- metadata and path-resolution tests

## Fix Direction

Implement the smallest metadata/path model accepted by the replacement ADR.
Support legacy committed `.agents/` before migration and committed `.ahm/`
after migration. Remove the unreleased ref-mode coupling instead of preserving
a transitional ref-backed state. Preserve unknown metadata fields and root
detection, and make normal workflow reads and writes resolve committed `.ahm/`
paths after migration.

## Acceptance Notes

- [x] Committed `.ahm/` is representable without enabling ref synchronization.
  `workflowPathsFor` now selects `.ahm/` paths when metadata lives in
  `.ahm/config.json`, decoupled from the ref storage mode value.
- [x] Legacy `.agents/` repositories continue to work before migration.
  When metadata is in `.agents/ahm.json` or absent, `.agents/` paths are used.
- [x] Unreleased ref-storage metadata is removed rather than maintained as a compatibility mode.
  The `recordsStorage().Mode` check in `workflowPathsFor` is replaced by a
  metadata-source-based check. The ref-mode enum and fields remain in the
  struct (removed in Milestone 4/172f) but no longer drive layout decisions.
- [x] Root detection, validation, install/upgrade, and workflow path tests
  cover the legacy and committed `.ahm/` layouts.
  Root detection already handles both config paths. Validation and install
  use `workflowPathsFor` which correctly resolves both layouts. The commit
  prompt is unified (both layouts use the same prompt). All 6 previously-
  failing task-work tests fixed to use correct paths for `.ahm/config.json`.
  Full suite: `go test ./internal/ahm -count=1` passes; race, fmt, lint,
  vet, build, and release-check all green.
