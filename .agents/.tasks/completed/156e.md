# Fold preflight review procedure into ahm task work review

## Summary

Embed the preflight review procedure in the binary and use it to build the
`ahm task work` review prompt, replacing the current invocation of the
installed preflight skill. Preflight gets no public command or scope.

## Problem

The task-work review pass invokes the installed skill by name: cake's review
arg builder passes `--skills preflight` (`internal/ahm/task_agents.go` ~line
53) and the review prompt is "Run the preflight skill on the current
uncommitted changes." (`taskWorkReviewPrompt`, `internal/ahm/task_session.go`
~line 59). Task 156g removes the installed skills, which would leave the
review pass referencing a skill that no longer exists. Per ADR 014 (revised
2026-07-07), preflight stops being a public procedure surface entirely: it is
internal review methodology for the handoff `ahm` already owns (ADR 006,
ADR 008), not a workflow-record procedure.

## Design

- Absorb `internal/templates/workflow/preflight-SKILL.md` (~223 lines) into a
  binary-embedded review procedure (drop skill frontmatter and "when to use"
  framing). Keep the skill's core structure: change-size scaling (XS/S one
  pass, M two, L/XL three sequential passes — rules conformance,
  correctness/source-of-truth, overengineering), then synthesize and apply
  worthwhile fixes; goal is the smallest clear diff.
- Build the review prompt from the embedded procedure. The reviewing agent
  computes the change size itself with the read-only git commands the
  procedure already prescribes (including untracked files per its caveat);
  ahm's own git surface does not grow.
- The review prompt gains task context the current generic prompt lacks:
  the task's id, title, and acceptance notes (task work already holds the
  task record), plus the managed-work completion checklist from ADR 014 —
  acceptance notes checked off, `exec_plan` (if any) updated or completed,
  documentation touched when the change requires it, no hand-edited
  generated indexes, task state moved via `ahm task` commands.
- Remove `--skills preflight` from the cake review arg builder; the
  procedure now arrives in the prompt itself, identically for every agent.
- Review-feedback parsing (`internal/ahm/task_parsers.go`) should be
  unaffected; verify rather than change.

## Relevant Files

- `internal/templates/workflow/preflight-SKILL.md` — procedure source
