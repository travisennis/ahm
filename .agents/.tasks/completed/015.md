# Add explicit human output formatters

## Summary

Review issue 15. emit mixes human, plain, and JSON behavior by value type, causing commands such as status to print JSON by default. Acceptance: commands have explicit text, plain, and JSON formatters where applicable, default output is human-friendly, and docs/tests describe format behavior.

## Acceptance Notes

- [x] Added `--text` flag for explicit human-friendly text output mode
- [x] Modified `emit()` to default to text instead of JSON for all types
- [x] Added text renderers for `map[string]any` (status/doctor), `map[string][]string` (install/upgrade), and simple dry-run maps
- [x] `ahm status` and `ahm doctor` now print human-friendly text by default
- [x] `ahm --json status` and `ahm --json doctor` still produce structured JSON
- [x] `ahm --plain` still produces compact JSON
- [x] Install/upgrade grouped text output unchanged
- [x] Task command-specific text output unchanged
- [x] Updated `docs/cli.md` with new output mode documentation
- [x] Added `output_test.go` with tests for text, JSON, plain, and nil output
- [x] All existing tests pass
- [x] Full CI (`just ci`) passes, including `go vet`, `lint`, `test-race`
