# Make task workflow guidance ahm-first

## Problem

`AGENTS.md` and `.agents/TASKS.md` still imply agents should inspect generated task indexes directly as the main task queue. That guidance conflicts with the current shape of `ahm`: task discovery, queue inspection, triage, and lifecycle changes should primarily flow through commands such as `ahm task next`, `ahm task ready`, `ahm task list`, `ahm task blocked`, `ahm task labels`, `ahm task show`, `ahm task create`, `ahm task accept`, `ahm task start`, `ahm task complete`, and `ahm task cancel`.

The generated `.agents/.tasks/index.md` is still useful as a deterministic artifact and fallback reference, but current workflow docs make manual index reading feel like the default path. `TASKS.md` also explains task details well while still implying too much manual front matter editing, file movement, and index management outside the CLI.

## Relevant Files

- `.agents/TASKS.md`
- `internal/templates/workflow/TASKS.md`
- `AGENTS.md`
- `internal/templates/workflow/AGENTS.md`
- `docs/cli.md`
- `internal/ahm/task_commands.go` or related task-work prompt builders, if delegated-agent prompts still tell agents to read the generated task index first
- Template or CLI tests that assert the affected guidance

## Fix Direction

Rewrite the workflow guidance so `ahm` commands are the primary task interface:

1. Update first-task/session instructions to read `.agents/TASKS.md`, then use `ahm` task commands to discover queue state, then open the specific task file.
2. Keep generated index documentation, but frame indexes as generated read-only artifacts and fallback/reference material, not the normal queue selection workflow.
3. Update `Choosing Work`, `Creating Tasks`, `Updating Tasks`, and `Working a Task` guidance to prefer concrete `ahm task ...` commands over manual front matter edits, manual file moves, or direct index inspection.
4. Preserve manual fallback instructions only for cases where `ahm` is unavailable or a command cannot express the needed change.
5. Update installed copies and canonical embedded templates together.
6. Update docs and task-work prompt text wherever current guidance tells delegated agents to read `.agents/.tasks/index.md` before using `ahm`.
