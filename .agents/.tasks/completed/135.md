# cleanupStaleTemps stops WalkDir on first os.Remove failure

## Summary

`cleanupStaleTemps` in `internal/ahm/write.go` walks `.agents/` looking for orphaned `.tmp` files. When `os.Remove` of a `.tmp` file returns a non-nil error (permission denied, concurrent removal by another process), the WalkDir callback returns that error, causing `filepath.WalkDir` to stop immediately. Remaining stale `.tmp` files are silently skipped since the callsite at `cli.go:226` discards the return value (`_ = cleanupStaleTemps(...)`).

## Evidence

- `internal/ahm/write.go:148` — `return os.Remove(cleanPath)` inside WalkDir callback returns the os.Remove error, terminating the walk on any failure.
- `internal/ahm/cli.go:226` — `_ = cleanupStaleTemps(a.opts.root)` discards the error, so the premature stop is invisible to the user.

## Fix Direction

In the WalkDir callback, treat `os.Remove` errors as non-fatal: when the error is `os.ErrNotExist` (concurrent removal by another process), continue walking silently. For other errors, keep walking and return a summarized cleanup error after `WalkDir` completes.

Keep the `cleanupStaleTemps(root string) error` signature. Update the `cli.go:226` callsite to capture a returned cleanup error and surface it via `a.addWarning` so the user knows stale temps weren't fully cleaned.

## Acceptance Notes

- [x] `just test ./internal/ahm` passes. Added `TestCleanupStaleTemps_ContinuesPastRemoveFailure` in `write_test.go`. Note: on Unix, `os.Remove` failure is driven by the *parent directory's* write permission, not the file's own mode, so the test locks a read-only subdirectory (`0o555`) to make its `.tmp` unremovable and confirms a removable `.tmp` in a sibling directory is still cleaned up.
- [x] `cli.go` surfaces a warning when `cleanupStaleTemps` returns a non-fatal cleanup error. The `index` callsite now captures the error and calls `a.addWarning`; `writeIndexes` defers `emitWarnings`, so it reaches the user.
- [x] Manually verified: with a read-only `.agents/locked/` dir holding `stuck.md.tmp` and a removable `.agents/writable/orphan.md.tmp`, `ahm index` removed the orphan, left the stuck file, printed `warning: could not remove 1 stale .tmp file(s): ...: permission denied`, and exited 0.
- [x] Existing `write_test.go` tests pass unchanged.

## Implementation Notes

- `cleanupStaleTemps` now collects per-file removal failures during the walk instead of returning from the callback. `os.ErrNotExist` (concurrent removal) is ignored silently; other errors are recorded, the walk continues, and a summarized error (`could not remove N stale .tmp file(s): ...`) is returned after `WalkDir`. Traversal errors handed to the callback still abort the walk as before. Signature `cleanupStaleTemps(root string) error` is unchanged.
