# Replace Cobra usage error string parsing

## Summary

Review issue 9. isCobraUsageError parses Cobra error strings, which is brittle and can false-positive. Acceptance: Args callbacks and flag errors propagate typed usage errors or Cobra-supported sentinels, heuristic string matching is removed, and tests cover usage versus runtime error exit behavior.

## Acceptance Notes

- [x] Removed `isCobraUsageError` heuristic string-matching function from `internal/ahm/cli.go`.
- [x] Added typed `noArgs` function that wraps Cobra "unknown command" errors as `usageError` (analogous to `cobra.NoArgs` but with proper error typing).
- [x] Set `Args: noArgs` on the root command to ensure unknown top-level commands produce typed usage errors.
- [x] Replaced all 6 uses of `cobra.NoArgs` with `noArgs` across version, simpleCommand, lenientCommand, task next, task migrate, taskListCommand, and task dep cycles.
- [x] Cobra `FlagErrorFunc` already wraps flag errors as `usageError` (unchanged).
- [x] Added `TestUsageErrorsExitCode2` — tests 8 usage-error scenarios (unknown command, subcommand, flag, shorthand flag, extra args, missing subcommand) all exit with code 2.
- [x] Added `TestRuntimeErrorsExitCode1` — verifies runtime errors (no managed repo) exit with code 1.
- [x] Added `TestRuntimeErrorsOnTaskCommandOutsideRepo` — verifies task list/ready/next outside repo exit with code 1.
- [x] All existing tests pass; `just ci` passes.
