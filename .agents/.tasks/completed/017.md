# Avoid rewriting task files when no changes occur

## Summary

Review issue 17. taskStatus and taskDepUpdate write files even when content is unchanged, touching mtimes and creating noisy diffs. Acceptance: no-op status and dependency commands skip writes and report stable output, with tests or timestamp-safe checks confirming no content rewrite.

## Acceptance Notes

- [x] taskStatus skips write and prints "already <status>" when target status matches current status
- [x] taskDepUpdate skips write and prints "already depends on <dep>" / "does not depend on <dep>" when dependency set is unchanged
- [x] Existing tests continue to pass
- [x] Tests added for all three no-op paths (status, dep add, dep remove)
- [x] docs/cli.md updated with no-op output messages for status transitions and dependency updates
