# Add ahm adr list and show commands

## Blocker

Blocked on task 075 for ADR collection, parsing, and ID resolution. Do not
start implementation until task 075 is complete.

## Summary

Add `ahm adr list` and `ahm adr show <id>` mirroring `ahm task list` and
`ahm task show`.

## Background

Design per ADR 009 (task 074) and
`.agents/.research/topics/madr-adr-management-design.md`. Model and ID
resolution come from task 075.

## Implementation Notes

- `adr list` prints ID, title, status, and date, sorted by ID; supports
  `--status <value>` filtering. Per ADR 009, superseded records are
  filterable by matching the `superseded` prefix because the replacement ID is
  embedded in the status.
- `adr show <id>` prints one record using `resolveADR` (`9`, `009`, or
  `009-slug`).
- Support `--json`, `--plain`, and `--text` output modes consistently with
  the task commands (`internal/ahm/output.go`).
- Legacy or malformed files must not abort either command; report them the
  way malformed tasks are reported (task 033 behavior).
- Update `docs/cli.md` with `adr list` and `adr show` sections in the same
  change.
