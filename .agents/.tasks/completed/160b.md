# Add ahm docs check command and expanded static checks

## Summary

Add the `ahm docs check` command running an expanded `project-docs` scope:
existing link/index checks plus link portability, entry-point budget, a
generalized index-coverage check, and an expanded scanned surface including
agent entry points. Deprecate `--check project-docs` with a warning.

## Behavior

- `ahm docs check [--strict] [--json|--plain|--text]`, wired like other
  commands in `internal/ahm/cli.go`; read-only, no model calls.
- Runs existing `project-docs` validators plus:
  - `project_doc_link_not_portable` (error): `file://` URIs, absolute
    filesystem paths, and home-directory paths in markdown link targets.
  - `entry_point_over_budget` (warning): root `AGENTS.md` line count over
    `projectDocs.entryPointBudget` (default 150). Skip `CLAUDE.md` when it
    is a symlink or a bare `@AGENTS.md` import.
  - `doc_unindexed` (warning): any `docs/` subdirectory containing an
    `index.md` has every sibling `.md` represented in it. Keep emitting
    `design_doc_unindexed` for `docs/design-docs/` for code compatibility.
- Scanned surface adds root `AGENTS.md`, `CLAUDE.md`, and nested
  `AGENTS.md` files to the existing roots + `docs/` set.
- `projectDocs.entryPointBudget` and `projectDocs.exclude` read from
  `.agents/ahm.json`; absent config uses defaults.
- Exit 0 when clean or warnings-only; exit 1 on errors; `--strict`
  promotes warnings.
- `--check project-docs` on `status`/`doctor` still works but prints a
  deprecation warning naming `ahm docs check`.
- JSON parity per the `textRenderer` pattern in `internal/output`.
