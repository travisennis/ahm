# Verify structured grooming against real agent CLIs

## Problem

Structured task revisions depend on external agents producing the newly expanded schema and on `ahm` extracting that result from each supported CLI's real output. Hermetic parser fixtures can validate ahm's own assumptions while still missing prompt-following or transport differences in installed agent versions.

## Relevant Files

- `internal/ahm/task_groom_test.go` — hermetic grooming integration coverage.
- `internal/ahm/task_agents.go` and `internal/ahm/task_parsers.go` — supported agent contracts.
- `internal/ahm/testdata/agents/` — captured real-agent transcripts and provenance.
- `docs/guides/testing.md` — live agent smoke requirements.
- `docs/references/cli/task-commands.md` — documented grooming behavior and examples.

## Fix Direction

Verify the structured revision contract accepted in [ADR 017](../../../docs/adr/017-apply-structured-task-revisions-during-grooming.md).

Add or extend live smoke coverage to run `ahm task groom` against a scratch repository containing a task like 176: a concrete but incomplete Open task with TODO acceptance notes and an objectively resolvable compatibility constraint. Verify each installed supported agent returns a valid structured revision, ahm preserves protected content, and the task becomes ready or Pending according to the accepted design. Capture or refresh representative transcripts when transport output changes.

Record agent versions and concise evidence. Keep live calls outside normal CI while retaining hermetic regression coverage in `just ci`.
