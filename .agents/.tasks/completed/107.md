# Split task_commands.go god file into focused files

## Summary

`internal/ahm/task_commands.go` is 1549 lines — the largest file in the project by ~500 lines. It violates the architectural invariant: "Command handlers should stay thin." The file currently mixes command wiring, four agent definitions with their executables/args/parsers, session orchestration (~300 lines), task CRUD, list/filter/sort, enum validation, dependency checking, and task ID resolution all in one file.

This refactor splits it into 8-9 focused files by concern with no behavior changes. See the ExecPlan at `.agents/exec-plans/active/107-split-task-commands-dot-go.md` for the detailed split plan, file map, and step-by-step instructions.
