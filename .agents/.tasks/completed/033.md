# Keep task commands usable with malformed task files

## Summary

Review issue 33. One malformed task file causes most ahm task commands and index generation to fail. Acceptance: list-like commands skip and warn on malformed tasks where appropriate, validation commands still hard-fail, and recovery workflows are documented and tested.

## Acceptance Notes

- [x] `collectTasks` now skips malformed task files instead of hard-failing
- [x] List-like commands (`task list`, `task ready`, `task blocked`, `task next`, `task dep cycles`, `task dep tree`) warn on stderr and continue with valid tasks
- [x] Index regeneration (`ahm index`) works despite malformed task files
- [x] Task resolution commands (`task show`, `task start`, `task complete`, `task cancel`, `task reopen`, `task dep add`, `task dep remove`) skip malformed tasks during ID resolution
- [x] Validation commands (`ahm status`, `ahm doctor`) still report malformed tasks as `task_malformed` errors and hard-fail
- [x] Recovery workflow documented in `docs/cli.md`
- [x] Tests added for malformed task resilience across all affected commands
