# Make status and doctor fail on validation errors

## Summary

Review issue 31. status and doctor report ok false but still exit 0, which prevents CI from gating on ahm doctor. Acceptance: validation errors produce non-zero exit or a documented --fail-on-error mode, with tests for clean and broken workflows.

## Acceptance Notes

- [x] `status()` returns `errValidationFailed` sentinel when `validation.Errors` > 0
- [x] `doctor()` returns `errValidationFailed` sentinel when `validation.Errors` > 0
- [x] `Main()` catches `errValidationFailed` and exits with code 1 without printing `error:` to stderr
- [x] Warnings alone (e.g., `managed_file_modified`) do not trigger non-zero exit
- [x] Existing tests updated to expect `errValidationFailed` for workflows with validation errors
- [x] `TestStatusSucceedsAfterInitInCleanDir` continues to pass (exit 0 for clean workflows)
- [x] Exit code behavior documented in `docs/cli.md`
