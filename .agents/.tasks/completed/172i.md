# Update documentation and release guidance for committed .ahm records

## Problem

Current architecture, workflow, CLI, upgrade, template, and agent guidance
describes private-ref durability and automatic synchronization. Users need one
consistent contract for project-owned `.agents/`, committed ahm-owned `.ahm/`
source records, ignored generated artifacts, migration, rollback, and release
compatibility.

## Relevant Files

- `ARCHITECTURE.md`
- `docs/references/workflow-spec.md`
- `docs/references/cli/commands.md`
- `docs/guides/workflow-upgrades.md`
- templates and managed-work context output
- `CHANGELOG.md`
- `migrate-issues.md`

## Fix Direction

Update all durable references and live guidance after implementation stabilizes.
Document `ahm records migrate` as a committed namespace migration, include
legacy committed conversion and rollback instructions, remove the unreleased
remote-sync claims, and explain which `.ahm/` files are committed versus
regenerated. Preserve `migrate-issues.md` as review evidence and mark which
findings became impossible or were resolved.
