# Preserve unknown task front matter fields

## Summary

Review issue 2. Task round-trips currently drop front-matter fields outside the fixed allowlist, causing user metadata loss on start, complete, cancel, and dependency edits. Acceptance: unknown fields such as assignee, due, tags, and ticket survive task mutations, with tests covering preservation and ordering or documented ordering.

## Acceptance Notes

- [x] Added `Extra map[string]string` field to `Task` struct to store unknown front matter fields
- [x] `parseTask` populates `Extra` via `metaExtra()` helper, which identifies unknown fields
- [x] `renderTask` writes extra fields in sorted key order after known fields
- [x] `docs/cli.md` updated to document front matter preservation behavior
- [x] Tests added: `TestTaskStatusPreservesUnknownFrontMatter` and `TestTaskDepUpdatePreservesUnknownFrontMatter`
- [x] All existing tests pass, full CI passes cleanly
