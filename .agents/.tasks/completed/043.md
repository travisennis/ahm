# Keep install dry run side-effect free in memory

## Summary

Review issue 43. install dry-run mutates meta.Version in memory before returning, which could affect downstream computations. Acceptance: dry-run avoids mutating metadata state or uses a copy, and tests cover dry-run metadata version behavior.

## Acceptance Notes

- [x] Dry-run no longer mutates `meta.Version` or `meta.Files` in memory
- [x] All meta `Files` mutations guarded by `!a.opts.dryRun` in the created, CreateOnly skipped, and force/upgrade cases (adoption case was already guarded)
- [x] `meta.Version = templates.Version` moved into the non-dry-run branch, just before `writeMetadata`
- [x] Generated index target deletion from `meta.Files` guarded by `!a.opts.dryRun`
- [x] New test `TestInstallDryRunDoesNotMutateMetadata` verifies metadata on disk is unchanged after dry-run
