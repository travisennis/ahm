# Replace managed skills with command-based procedures and ahm prime

## Summary

Track the ADR, CLI, template, migration, and documentation work to stop
installing managed agent skills into consumer repositories and instead expose
their procedures, plus a session-start briefing, directly from the `ahm`
binary.

## Context

Decided in a design discussion on 2026-07-01, modeled on beadwork's
`bw prime` / `bw onboard` approach (github.com/jallum/beadwork), and revised
in a design discussion on 2026-07-07 (before implementation of the procedure
surfaces) from scoped procedure contexts to delegation commands — see the
revised ADR 014:

- The three managed skills (`.agents/skills/preflight`,
  `.agents/skills/grooming-backlog`, `.agents/skills/finding-improvements`)
  stop being installed. Procedure content lives only in the binary; no
  per-repo customization.
- Grooming and the improvement audit become delegation commands modeled on
  `ahm task work`: `ahm task groom [<id>]` and `ahm audit`. The delegated
  agent returns a schema-constrained JSON result; `ahm` applies it
  mechanically (groom: accept/comment/dep/label fixes, never cancellation;
  audit: create `Open` tasks with a `source:audit` provenance label). `Open`
  status is the acceptance gate — no interactive gate.
- Preflight stops being a public procedure surface; its content is embedded
  in the binary and folded into the `ahm task work` review prompt.
- A new read-only `ahm prime` command becomes the canonical agent
  session-start entry point (state-rich, instruction-light). This aligns with
  ADR 013, which reserves `ahm prime` for records sync; task 143 later layers
  the sync step on top of the command built here.
- Unscoped `ahm context` becomes a usage error pointing at `ahm prime` and
  the valid scopes. Scoped `ahm context <scope>` remains the managed-work
  reference channel (task/plan/adr/research/docs); no procedure scopes are
  added.
- `ahm agents suggestions` is removed in favor of `ahm onboard`, which prints
  a minimal paste-ready AGENTS.md snippet. A low-severity `ahm doctor` check
  flags an AGENTS.md that does not reference `ahm prime`.
- Content architecture: the pasted snippet carries only the bootstrap
  invariant (always run `ahm prime`) and safety invariants (never hand-edit
  generated indexes); the prime output carries live state plus the
  managed-work routing table; scoped contexts carry reference instructions;
  delegation commands carry procedures. Operating-loop guidance is
  project-owned and never emitted by ahm.
- Autonomy model (recorded deliberately in ADR 014): audit → groom → work can
  run end-to-end without a human decision. Human touchpoints are priority,
  task selection, the task-work review/commit handoff, and cancellation.
  Safety is observability and reversibility (records only, provenance labels,
  session capture, required cancel reasons), not interactive gates.
- JSON parity is kept for all new outputs (structured data in Go, text
  rendered from the same struct).
