# Remove automatic ref sync from prime and workflow mutations

## Problem

`ahm prime` and supported workflow mutations currently snapshot, fetch, push,
materialize, and update sync metadata in ref mode. This creates hidden network
and Git-ref side effects, makes linked worktrees unsafe, and dirties committed
config with `records_last_sync`.

## Relevant Files

- `internal/ahm/prime.go`
- `internal/ahm/records_commands.go`
- `internal/ahm/indexes.go`
- task mutation paths and tests

## Fix Direction

Return `prime` to local repository preparation and briefing: ensure local
workflow directories exist when appropriate, regenerate ignored indexes,
validate state, and print the briefing without network or custom-ref operations.
Remove automatic post-mutation snapshots and stop writing machine-local sync
state to committed config. Because ref mode is unreleased, its sync fields do
not need a compatibility period.

## Acceptance Notes

- [x] `prime` does not fetch, push, create, update, or materialize custom refs.
  - `primeSyncRecords()` removed entirely; `prime()` now builds and emits the report directly.
  - `--no-sync` flag removed from CLI (no sync to skip).
- [x] Normal task/research/ExecPlan mutations do not snapshot custom refs.
  - `snapshotRecordsAfterWrite()` removed from `writeIndexes()` and from the codebase.
  - Ref-snapshot tests (`TestTaskCreateInRefModeWritesAhmRecordsAndSnapshotsRef`,
    `TestTaskCompleteInRefModeMovesRecordAndSnapshotsRef`,
    `TestIndexInRefModeSnapshotsHandEditsAndStaysIdempotent`) removed.
- [x] Routine commands leave committed config clean.
  - `primeSyncRecords()` never writes `RecordsLastSync` to config.
  - `emitRecordsStaleStatus()` removed from `status()` and `doctor()`.
- [x] Linked worktrees cannot publish record deletion through `prime`.
  - Prime no longer has any ref operations that could affect linked worktrees.
- [x] Local briefing, index regeneration, validation, dry-run, and warning behavior remains covered.
  - All existing prime, status, doctor, index, and task tests pass.
  - Covered by `just ci` (fmt, vet, test -race -cover, lint, vulncheck).
