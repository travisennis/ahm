# Retire ref-backed records commands and sync metadata

## Problem

Once source records remain committed, the ref-backed `records status`, `pull`,
`push`, `sync`, and sync-focused `doctor` behavior has no durable role. Keeping
it would preserve the highest-risk maintenance surface and leave conflicting
storage semantics in the CLI and metadata model.

## Relevant Files

- `internal/ahm/records.go`
- `internal/ahm/records_commands.go`
- records command and plumbing tests
- CLI and workflow metadata references

## Fix Direction

Retain `ahm records migrate` for the committed namespace migration and remove or
replace the ref-sync-only command surface according to the replacement ADR.
Delete the unreleased `store_mode: ref`, `records_ref`, `records_remote`, and
`records_last_sync` model and its private-ref plumbing outright. There is no
adopter state, custom ref, or released CLI compatibility to preserve.

## Acceptance Notes

- [x] General CLI help no longer presents unsupported ref synchronization as usable behavior.
  - `ahm records --help` shows only `migrate` and `doctor` subcommands.
  - `records status`, `pull`, `push`, `sync` subcommands removed entirely.
- [x] Ref-only plumbing, metadata, remote restrictions, and tests are removed rather than deprecated.
  - `snapshotRecordsRef`, `materializeRecordsRef`, `fetchRecordsRef`, `pushRecordsRef`,
    `compareRecordsRefs`, `recordsRemoteTrackingRef`, `buildRecordsTree`, `writeGitTree`,
    `insertRecordBlob`, `sortedRecordTreeKeys`, `recordsTreeNode`, `recordsSnapshot`,
    `recordsRefComparison` removed from `records.go`.
  - `resolveGitRef`, `isGitAncestor`, `validateRecordsRef`, `gitIdentityEnv`, `isGitExitError`,
    `errGitRefMissing` removed as no longer needed by migrate.
  - `runGit`/`runGitBytes` simplified (removed unused `stdin`, `env` params).
  - `StoreMode`, `RecordsRef`, `RecordsRemote`, `RecordsLastSync` removed from `metadata` struct.
  - `recordStoreModeRef`, `defaultRecordsRef`, `defaultRecordsRemote` constants removed.
  - `recordsStorageConfig` simplified to only `Mode` field.
  - `records_commands_test.go` and `records_workflow_test.go` removed (all ref-mode tests).
  - `records_test.go` stripped to only the two collection tests + shared helpers.
  - Remote URL validation (`isSupportedRecordsRemoteURL`, `requireSupportedRecordsRemote`,
    `lsRemoteRecordsRef`, `classifyRecordsRemoteError`, `classifyRecordsPushError`) removed.
  - `primeRecords.Synced` field removed (was always true after 172e).
  - `recordsDoctor()` simplified to only migration diagnostics.
  - `migrate` command help text updated (no longer mentions ref fields).
- [x] Unknown non-ref metadata fields remain preserved where promised.
  - Ref fields like `store_mode`, `records_ref` read as unknown `Extra` fields.
  - `TestMetadataRoundTripPreservesUnknownFields` passes (ref fields survive round-trip
    as extra keys).
- [x] The remaining `records migrate` command matches the replacement ADR (ADR 015).
  - Migration writes clean `.ahm/config.json` without ref fields.
  - No ref seeding, no custom-ref operations.
