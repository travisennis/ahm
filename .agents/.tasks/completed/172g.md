# Keep source records committed and generated indexes local-only

## Problem

Committed source records restore robust Git behavior, but committing regenerated
indexes would retain avoidable branch-history churn. The ownership and
validation model must distinguish durable source records from deterministic
local derivatives and machine-local artifacts.

## Relevant Files

- `internal/ahm/indexes.go`
- `internal/ahm/install.go`
- `internal/ahm/validation.go`
- `.ahm/.gitignore` generation
- workflow specification and upgrade tests

## Fix Direction

Track config and all source task, research, ExecPlan, and supported attachment
files. Ignore generated task, research, and ExecPlan indexes under `.ahm/`, plus
locks, temporary files, session state, and other explicitly machine-local
artifacts. Keep `docs/adr/index.md` committed because it is project-facing
documentation browsed without ahm. Ensure `init`, `upgrade`, `index`, and
`prime` recreate required workflow directories and indexes deterministically
without making a clean project worktree dirty.

Ignored indexes can remain stale across branch checkout, so `prime` must
regenerate them before validation and briefing. Commands and validation must
treat source records, never an existing index, as authoritative.

## Acceptance Notes

- [x] `.ahm/.gitignore` never ignores source records or supported attachments. Entries `index.md`, `.lock/`, `*.tmp` match only generated indexes and machine-local artifacts.
- [x] Workflow indexes under `.ahm/` and machine-local artifacts are ignored and reproducible via the managed `.ahm/.gitignore`.
- [x] `docs/adr/index.md` remains committed and deterministically generated (listed in `generatedIndexTargetsList`, not under `.ahm/`).
- [x] Fresh clones can regenerate the complete local index set: `init`/`upgrade` call `ensureWorkflowGitignore` + `writeIndexes`; `prime` calls `ensureWorkflowGitignore` + `regenerateIndexes`.
- [x] `prime` replaces stale cross-branch workflow indexes via `regenerateIndexes()` before `buildPrimeReport()`.
- [x] Validation and commands use source records as authoritative state via `indexWritesFor`/`validateGeneratedIndexes`.
- [x] Routine index regeneration does not dirty normal Git status. `prime` only prepares the worktree when metadata is present, and gitignore keeps generated files untracked.
