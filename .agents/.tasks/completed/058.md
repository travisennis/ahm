# Add opt-in task work completion handoff

## Decision

Implement task completion handoff as a later prompt to the original working
agent session, built on task 056's in-process session capture/resume capability.

- Keep this task separate from session capture, review, and commit handoff.
- `ahm` delegates completion work to the external agent; it does not silently
  mark tasks completed on its own.
- The completion prompt should instruct the agent to fill Acceptance Notes, run
  required verification, and run `ahm task complete <id>` when acceptance is
  satisfied.
- Strict acceptance failures remain surfaced by `ahm task complete`.
- `cake` is the first full-workflow implementation, with room for `codex`,
  `claude`, and `cursor` once their session/resume contracts are implemented.
- Tasks already `In Progress` before `task work` started are allowed.

## Summary

Explore and implement an opt-in task completion handoff that asks the delegated
agent to fill acceptance notes, run verification, and complete the task through
`ahm` when work is done.

## Background

The MVP `task work` prompt tells the external agent to follow the workflow, but
`ahm` does not orchestrate or enforce completion after the handoff. The shell
workflow asks the agent to complete the task before committing. This task brings
that behavior into `ahm` while keeping the completion action owned by the
delegated agent and the normal `ahm task complete` command.

## Provider Direction

- Use the provider/session boundary from task 056.
- Implement the completion follow-up prompt for `cake` first.
- Keep command structure compatible with later `codex`, `claude`, and `cursor`
  full-workflow support.
