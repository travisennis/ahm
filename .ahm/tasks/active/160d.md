## Blocker

## Summary

Document and wire the three-tier enforcement pattern for `ahm docs check`:
blocking static checks in pre-commit, advisory diff mode in the coding
agent's commit path, and `--strict` in CI. Update the documentation
guardrail and dogfood the setup in this repo.

## Behavior

- Guidance: update `docs/guardrails/documentation.md` (Enforcement and
  Required Checks sections) to name `ahm docs check` as the mechanical
  check, replacing the ad-hoc `--check project-docs` mention.
- Hook recipes (documented in the workflow-upgrades or a guides doc, and
  applied to this repo):
  - pre-commit: `ahm docs check` (blocking; catches broken/non-portable
    links before commit).
  - Claude Code `PreToolUse` hook matching `git commit`: run
    `ahm docs check --staged` and surface findings as advisory context —
    never blocking; suggestions must not train agents to game the map.
  - CI: `ahm docs check --strict`.
- Preserve the completed task 156 guidance model: the onboard snippet stays
  minimal (bootstrap plus safety invariants), while docs-check guidance belongs
  in guardrails and hooks.
