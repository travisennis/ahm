# Add semantic contract checks for project documentation

## Summary

`ahm docs check --strict` currently reports this repository as healthy even when live documents advertise nonexistent flags and context scopes or contradict the implemented record model. Structural link, index, portability, and line-budget checks are valuable but cannot prevent recurrence of the specific semantic drift found in the audit. After the current documentation contract is reconciled, add targeted machine-checkable invariants against commit `5d231a4` rather than attempting natural-language validation.

## Evidence

- Running `ahm docs check --strict` at `5d231a4` returned `ok` while `docs/references/workflow-spec.md:30-56` listed nonexistent `--quiet`/`--verbose` flags and incomplete commands.
- `docs/VISION.md:31-65` advertises unsupported context scopes and contradictory committed/out-of-history record properties without triggering a finding.
- `internal/ahm/cli.go:287-307` describes docs checking as structural: relative links, portable targets, entry-point budgets, and index coverage.
- Searches under `internal/`, `scripts/`, and CI configuration found no semantic contract test tying this repository's authoritative documentation blocks to executable behavior.
- Task 235 is responsible for first reconciling the live documentation; this task must encode the corrected result, not preserve current contradictions.

## Fix Direction

Add targeted semantic documentation contract checks after task 235 lands. Prefer explicit machine-readable inventories, exact invariant assertions, or repository-specific tests for facts that have an authoritative implementation source: global flags, top-level/subcommand inventories, context scopes, fresh-init metadata/layout, and the membership of `just ci`. Keep generic `ahm docs check` free of ahm-repository-specific product assumptions unless a clean configurable mechanism already exists. Do not attempt to infer truth from unrestricted English or add model/network calls. If the only viable approach is brittle whole-paragraph matching, STOP and report a narrower proposed contract.
