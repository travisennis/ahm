# Remove obsolete record storage mode output

## Problem

Ref-backed record storage was abandoned before release, and both supported record
layouts now contain ordinary committed files. The remaining storage-mode output is
therefore not a meaningful discriminator: `prime --json` always emits
`records.mode: "committed"`, and `records doctor` always includes
`checks.mode: "committed"`. Treating `.agents/` and `.ahm/` as separate modes would
reintroduce the discarded abstraction; they are legacy and post-migration layouts
with the same storage semantics.

Remove the obsolete mode fields instead of inventing a `migrated` mode. This is a
deliberate structured-output compatibility change and should be documented as such.

## Relevant Files

- `internal/ahm/prime.go`
- `internal/ahm/records_commands.go`
- `internal/ahm/prime_test.go`
- `internal/ahm/records_migrate_test.go`
- `docs/references/cli/commands.md`
- `docs/references/workflow-spec.md`
- `docs/guides/workflow-upgrades.md`
- `ARCHITECTURE.md`

## Fix Direction

Remove `primeRecords`, `primeReport.Records`, and `buildPrimeRecords`, so JSON and
plain prime output no longer contain a `records` object whose only content is a
constant mode. Remove the constant `mode` entry from records-doctor checks while
preserving its migration diagnostics and `ok` behavior. Update user-facing docs and
nearby architecture terminology to describe legacy `.agents/` and post-migration
`.ahm/` layouts rather than active storage modes. Keep path selection based on the
metadata location; this task does not remove legacy-layout compatibility or the
`records migrate` command.

## Acceptance Notes

- [ ] `prime --json` and `prime --plain` omit the obsolete `records` object; text output remains unchanged
- [ ] `records doctor` omits `checks.mode` in text, JSON, and plain output while preserving migration findings and exit behavior
- [ ] No production code computes or emits a record storage mode
- [ ] CLI, workflow, upgrade, and architecture docs consistently describe `.agents/` and `.ahm/` as layouts with committed source records, not distinct current storage modes
- [ ] Focused prime and records-doctor tests cover the revised structured output, and existing tests pass
