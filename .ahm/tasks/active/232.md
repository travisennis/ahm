# Remove dead wrappers and duplicated helpers in internal/ahm

## Problem

Several package-level functions have no non-test callers and exist only as test-only indirection: `nextTaskID`, `nextChildTaskID`, `acquireWorkflowRecordLock`, `acquireWorkflowLock`, `indexWritesFor`, `validateWorkflowScoped`, `collectTasks`. `metadataReadRelPath` and `metadataWriteRelPath` are byte-identical duplicates. The containment check in `cleanupStaleTempsIn` uses strings.HasPrefix without a path-separator boundary (unreachable in practice since the walk is rooted at stateDir, but incorrect as written).

## Relevant Files

- `internal/ahm/task_create.go` (nextTaskID, nextChildTaskID)
- `internal/ahm/lock.go` (acquireWorkflowRecordLock, acquireWorkflowLock)
- `internal/ahm/indexes.go` (indexWritesFor)
- `internal/ahm/validation.go` (validateWorkflowScoped)
- `internal/ahm/tasks.go` (collectTasks)
- `internal/ahm/install.go` (metadataReadRelPath / metadataWriteRelPath)
- `internal/ahm/write.go` (cleanupStaleTempsIn prefix check)

## Fix Direction

Delete the dead wrappers and point tests at the ForPaths variants, collapse the duplicated metadata path helpers into one, and either remove the redundant containment check or make it separator-safe. Pure cleanup; no behavior change.

## Acceptance Notes

- [ ] Dead wrappers removed with tests updated to call the remaining variants
- [ ] Single metadata rel-path helper
- [ ] go vet and full test suite pass with no behavior change
