# Align AGENTS.md operating loop with ahm task lifecycle

## Summary

The `AGENTS.md` operating loop is the top-level procedure an agent follows, but it omits two lifecycle transitions that `ahm` expects:

1. **`ahm task start <id>`** — the loop goes from intake (`ahm task show <id>`) straight to editing ("Select the route..."), never calling `ahm task start`. The detailed "Work A Task" guidance in `internal/templates/workflow/TASKS.md` includes it as step 2, but `AGENTS.md` never directs agents to follow that procedure.

2. **`ahm task complete <id>` before git commit** — step 6 says "After completing a task, run preflight checks before handoff," which is ambiguous (implementation done vs. lifecycle complete). Combined with the missing `start` step, this allowed task 221's implementation to be committed while the task file stayed `Pending` in the active bucket.

**Fix:** Rewrite the `AGENTS.md` operating loop to include `ahm task start` after intake and `ahm task complete` before preflight checks. Add a note in `TASKS.md` that `ahm task complete` must precede any git commit for the task. The loop should read as a clear sequence that matches the lifecycle `ahm` enforces.

## Acceptance Notes
