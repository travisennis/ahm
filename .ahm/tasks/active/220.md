# Detect concurrent target edits before applying grooming verdicts

## Summary

At survey baseline commit `5ead0ae`, task grooming builds its agent prompt from one task snapshot, waits through a potentially long external delegation, then reloads tasks under the mutation lock without comparing the targets to that original snapshot. The old verdict is merely revalidated against current state and may replace labels, dependencies, section content, or timestamps that a human or another ahm process changed during delegation. Implement the stale-record rejection required by ADR 017.

## Evidence

- `internal/ahm/task_groom.go:139-167` — `taskGroom` captures `tasks` and `targets`, builds the prompt, and delegates before acquiring the workflow mutation lock.
- `internal/ahm/task_groom.go:191-205` — the locked phase reloads `current` and calls `validateGroomResult`, but performs no equality, timestamp, or content-fingerprint comparison with the original targets.
- `internal/ahm/task_groom.go:600-668` — applying a stale verdict can replace current dependencies, labels, revision sections, comments, and `updated`.
- `docs/adr/017-apply-structured-task-revisions-during-grooming.md:143-148` — the accepted design requires verifying that targeted records have not changed since they were read before writing.
- Existing state-change coverage around `internal/ahm/task_groom_test.go:432-455` only catches changes that make the verdict semantically invalid; it does not reject a concurrent edit that remains structurally valid.

## Fix Direction

Capture a deterministic representation or fingerprint of every target record used to build the prompt, then compare it with the freshly loaded target under the workflow mutation lock before applying any verdict. Abort the entire batch without task or index writes when any target differs, naming the changed task IDs. Compare only target records; unrelated task changes may still be revalidated for dependency correctness without automatically invalidating the batch. Do not hold the workflow lock across the external agent call.
