# Kill delegated agent process trees on task work timeout

## Problem

`runTaskWorkCommand` uses `exec.CommandContext` with an `io.MultiWriter` stdout and sets neither `cmd.WaitDelay` nor any process-group handling. On timeout Go SIGKILLs only the direct child; agent CLIs (cake, claude, codex, cursor) spawn subprocesses that inherit the stdout pipe, so `cmd.Wait` blocks until every descendant exits. A hung agent subtree can block ahm forever, defeating the stated purpose of the timeout ("prevents hung agent CLIs from blocking ahm indefinitely"). The same path is used by groom and audit delegation.

## Relevant Files

- `internal/ahm/task_work.go` (runTaskWorkCommand, taskWorkDefaultTimeout comment)
- `internal/ahm/task_session.go` (io.MultiWriter stdout)

## Fix Direction

Set `cmd.WaitDelay` so Wait cannot block indefinitely on inherited pipes after cancellation, and start the child in its own process group with a `cmd.Cancel` that signals the group so descendants are terminated too. Cover with a test that spawns a child which itself spawns a sleeping grandchild inheriting stdout.

## Acceptance Notes

- [ ] A timed-out delegated command returns promptly even when a grandchild holds the stdout pipe open
- [ ] Descendant processes are terminated (or at least detached) on timeout
- [ ] Regression test exercises the grandchild-inherits-pipe case
