# Enforce CLI documentation parity with Cobra

## Summary

The detailed CLI references manually repeat nearly two thousand lines of command names, scopes, flags, aliases, and behavior already defined in Cobra. The audit found stale global flags and context behavior in canonical documentation even though the detailed references are otherwise comprehensive. This task adds a mechanical parity boundary against the executable command tree as of commit `5d231a4` so future command-surface changes cannot silently leave the documentation behind.

## Evidence

- `internal/ahm/cli.go`, `internal/ahm/task_commands.go`, `internal/ahm/adr_commands.go`, and `internal/ahm/records_commands.go` — authoritative Cobra command, scope, alias, and flag registration.
- `docs/references/cli/global-contract.md`, `docs/references/cli/commands.md`, `docs/references/cli/task-commands.md`, and `docs/references/cli/task-file-format.md` — 1,972 lines of manually maintained CLI contract.
- `docs/references/workflow-spec.md:30-56` — already contains nonexistent `--quiet` and `--verbose` flags, omits `--text`, and has an incomplete command inventory.
- `internal/ahm/cli.go:337-368` — restricts context to five scopes, while current documentation elsewhere advertises additional or unscoped modes.
- Existing tests cover CLI behavior, but searches under `internal/` and `scripts/` found no test that compares current documentation inventories with the registered command tree.

## Fix Direction

Create a small, deterministic parity mechanism sourced from the Cobra tree. Prefer a focused test or generated inventory for command names, aliases, global flags, context scopes, and selected command flags over generating all prose. Keep user-authored explanations and examples in Markdown. Put implementation and tests near the existing CLI tests or in a narrowly named documentation-contract test; document the update workflow in `CONTRIBUTING.md`. Do not introduce a new runtime dependency or parse arbitrary prose. If reliable parity would require scraping unconstrained English text, STOP and narrow the contract to explicit machine-readable inventory blocks.

## Acceptance Notes

- [ ] A test or generation check derives the supported top-level commands and persistent flags from the actual Cobra command tree.
- [ ] The parity boundary also covers context scopes and at least the task/ADR/records subcommand inventories, including aliases where documented as compatibility surfaces.
- [ ] Deliberately adding a fake documented global flag or removing a registered documented command makes the focused check fail with a useful message.
- [ ] Current prose examples remain human-maintained; no broad documentation generator or new runtime dependency is introduced.
- [ ] `CONTRIBUTING.md` identifies the exact command contributors run after changing CLI wiring.
- [ ] Focused CLI/documentation-contract tests and `go test ./internal/ahm` pass.
- [ ] `ahm docs check --strict`, `just docs-md-lint`, and `just ci` pass, or the handoff records exact environmental blockers.
