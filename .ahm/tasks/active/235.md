# Reconcile current product documentation with executable behavior

## Summary

The current entry-point and policy documentation contains contradictory descriptions of record storage, context behavior, supported flags, and the product's maturity. This task reconciles the live documentation with executable behavior and tests as of commit `5d231a4`, establishing one consistent current contract without rewriting historical ADR text.

## Evidence

- `README.md:3-15,53-63` — describes a `.agents` workflow, calls the project an initial implementation, says unscoped `ahm context` is the briefing, and says writes use `.agents` unless a future command changes that.
- `internal/ahm/install.go:271-305` — fresh init actually selects the committed `.ahm/` layout and writes `.ahm/config.json`.
- `internal/ahm/cli.go:337-368` — unscoped context is rejected; only `task`, `adr`, `research`, `plan`, and `docs` scopes exist, with `ahm prime` owning the briefing.
- `docs/references/workflow-spec.md:30-56` — lists nonexistent `--quiet` and `--verbose` flags, omits `--text`, describes context as a briefing, and omits current top-level commands including `prime`, `audit`, and `docs`.
- `docs/VISION.md:31-65,104-115` — advertises nonexistent context scopes and simultaneously describes workflow records as committed and outside branch history.
- `docs/references/cli/global-contract.md:36-40` — says fresh init creates `.agents` scaffolding.
- `docs/guardrails/safety-and-permissions.md:10-16` and `docs/guardrails/workflow-state-and-file-formats.md:3-13` — describe current metadata and root behavior primarily through legacy `.agents/ahm.json`.
- `internal/ahm/cli.go:134-137` — root help itself still labels the current product as a `.agents` workflow, so user-visible code strings also need reconciliation.

## Fix Direction

Update `README.md`, `docs/VISION.md`, `docs/references/workflow-spec.md`, `docs/references/cli/global-contract.md`, the affected guardrails, and the root CLI help text so they consistently distinguish fresh `.ahm/` repositories from supported legacy `.agents/` repositories. Make `ahm prime` the only briefing command, list only implemented context scopes and global flags, and describe committed source records versus gitignored generated indexes accurately. Keep historical ADRs unchanged and leave the chronological restructuring of `docs/guides/workflow-upgrades.md` to its separate task; only adjust links into that guide if needed. Preserve CLI compatibility except for correcting descriptive help text. If the implementation and tests disagree on behavior, STOP and report the conflict rather than choosing one silently.
