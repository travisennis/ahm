# Restore the architecture module map as a reliable codemap

## Summary

The module map in `ARCHITECTURE.md` has begun to diverge from the production package layout and incorrectly describes at least one listed file. Because project routing explicitly tells agents to use this map for architecture and implementation work, incomplete or inaccurate ownership descriptions cause avoidable source discovery and increase refactor risk. Restore a clear, maintainable codemap against commit `5d231a4`.

## Evidence

- `ARCHITECTURE.md:45-113` — presents a file-by-file module map for `internal/ahm`.
- Production files `internal/ahm/audit.go`, `internal/ahm/prime.go`, `internal/ahm/records_migrate.go`, and `internal/ahm/markdown_sections.go` are absent from that map.
- `ARCHITECTURE.md:65-66` — describes `internal/ahm/records.go` as workflow-record selection plumbing.
- `internal/ahm/records.go:11-31` — actually provides Git subprocess wrappers used by records helpers.
- `internal/ahm/workflow_paths.go` and `internal/ahm/install.go` contain the record-layout selection described by the stale `records.go` entry.

## Fix Direction

Correct the inaccurate ownership descriptions and add the missing production modules, grouping tightly related helpers if that makes the map easier to maintain. State whether the map intends exhaustive production-file coverage or only architectural components; make its contents match that stated contract. A lightweight repository test is acceptable if it can enforce an explicit exhaustive inventory without hardcoding prose, but do not build a general documentation framework in this task. Keep test-only files out of scope unless they define an important verification boundary.

## Acceptance Notes

- [ ] Every production module intended by the map's stated coverage is represented, including audit, prime, records migration, and Markdown section parsing.
- [ ] The `records.go`, `git.go`, `workflow_paths.go`, and records-command descriptions match their actual responsibilities.
- [ ] Newly added entries describe architectural ownership rather than merely repeating filenames.
- [ ] A future contributor can determine from the document whether adding a production module requires updating the map.
- [ ] `ahm docs check --strict` and `just docs-md-lint` pass.
