# Normalize task work phase output and outcomes

## Blocker

Blocked until tasks 189b, 189c, and 189g resolve commit authorization, durable
run records, and acceptance enforcement.

## Problem

`ahm task work` streams provider-specific JSON or text directly while also emitting ahm status messages. Users and automation must infer which phase is active, how long it took, whether it completed, why it was skipped, and whether the overall command is awaiting authorization or genuinely done. Raw provider transcripts are valuable for diagnosis but are not a stable CLI contract.

## Relevant Context

- `internal/ahm/task_session.go` — phase sequencing and ad hoc stderr messages.
- `internal/ahm/task_parsers.go` — provider event normalization opportunities.
- `docs/guardrails/cli-and-user-output.md` — stable output and exit-code requirements.
- Durable task-work run records — intended owner for full raw transcripts and phase history.

## Fix Direction

Define normalized phase-start, phase-complete, phase-failed, phase-skipped, and awaiting-authorization events for implementation, review, finalization, verification, and commit. Human output should be concise and provider-neutral; `--json` should expose stable structured events and a terminal summary. Preserve an opt-in path to raw provider output or store it in the local run record. Establish exit-code categories without leaking provider-specific codes as ahm's public contract.

## Acceptance Notes

- [ ] Human output clearly reports each requested phase, agent/model, duration, result, and next action.
- [ ] JSON output has versioned stable events and exactly one terminal outcome summary.
- [ ] Complete, incomplete, interrupted, unavailable-provider, malformed-output, verification-failed, and awaiting-commit outcomes are distinguishable.
- [ ] Raw provider output remains available for diagnosis without becoming the default stable CLI surface.
- [ ] Dry-run previews the phase plan and authority requirements using the same normalized vocabulary.
- [ ] Golden output tests cover success, skip, failure, interruption, and resume paths across supported agents.
- [ ] CLI/reference documentation and compatibility notes are updated, with required live smoke evidence recorded.
