# Allow audits with an empty task label vocabulary

## Summary

At survey baseline commit `5ead0ae`, `ahm audit` cannot create a finding in a newly initialized repository with no existing tasks. Audit validation requires every finding to carry type and area labels, but it also rejects every label not already present in the task-derived vocabulary; an empty backlog therefore makes every non-empty result unsatisfiable after the external agent has already run.

## Evidence

- `internal/ahm/audit.go:231-266` — `validateAuditResult` builds vocabulary only from existing task labels, rejects unknown labels, and independently requires both `type:` and `area:` labels.
- `internal/ahm/audit.go:144-147` — the prompt emits no usable label choices when the vocabulary is empty.
- `internal/ahm/audit_test.go:13-25` — the helper used by audit tests always seeds an existing task with `type:task, area:unknown`.
- `internal/ahm/audit_test.go:27-104` — no successful non-empty audit test covers a repository with zero tasks.
- `docs/references/cli/commands.md:9-25` documents audit as a general repository survey and does not state that an existing task is required.

## Fix Direction

Define a deterministic bootstrap vocabulary for audits with no usable task labels: `type:task` and `area:unknown`, matching the existing defaults used when creating ordinary tasks. Include those choices in the prompt and accept them during validation only when the task-derived vocabulary is empty. Once a repository has a usable vocabulary, retain the current strict rejection of unknown labels. Keep general `task create` label behavior and unrelated grooming label rules out of scope.

## Decision

An empty task vocabulary bootstraps to `type:task` and `area:unknown`; it does not authorize the provider to invent arbitrary well-formed labels. This lets a fresh repository complete an audit deterministically while preserving vocabulary enforcement for established repositories.

- [ ] Existing repositories continue rejecting genuinely unknown labels according to the chosen policy.
