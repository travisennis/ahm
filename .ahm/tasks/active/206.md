# Consolidate Markdown heading section helpers

## Problem

Markdown heading-section replacement logic is duplicated across `upsertCancellationReason`, `upsertADRSection`, `replaceGroomSection`, and `groomSectionContent`. The groom scanners already differ in repeated-match `start` handling, creating avoidable drift risk.

## Relevant Files

- `internal/ahm/task_status.go` — cancellation section upsert.
- `internal/ahm/adrs.go` — ADR section upsert.
- `internal/ahm/task_groom.go` — groom section read/replace scans.
- Corresponding task, ADR, and groom tests.

## Fix Direction

Extract a concrete shared heading-section locator/edit primitive that preserves each caller's newline and ambiguity semantics. Keep higher-level domain helpers small and explicit.

## Acceptance Notes

- [ ] Section boundary scanning has one tested implementation.
- [ ] Repeated matching headings are handled consistently and explicitly.
- [ ] Cancellation, ADR, and groom output remains byte-compatible for valid inputs.
- [ ] LF/CRLF, missing section, final section, and nested heading cases are covered.
