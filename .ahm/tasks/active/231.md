# Stream groom and audit delegation output to the terminal

## Problem

`runGroomDelegation` and `audit` capture agent output into a bytes.Buffer without teeing it to the user's terminal (unlike task work, which uses io.MultiWriter). A groom or audit run appears frozen for up to the full timeout (default 30 minutes) with no progress indication. The full stream-json output is also buffered in memory in all delegation paths, which is unbounded for long sessions.

## Relevant Files

- `internal/ahm/task_groom.go` (runGroomDelegation)
- `internal/ahm/audit.go` (audit)
- `internal/ahm/task_session.go` (existing MultiWriter pattern)

## Fix Direction

Tee delegation output to stderr while still capturing it for parsing, matching
the existing task-work session pattern. Retain the current capture behavior in
this task; buffer bounding or spill-to-disk requires a separate change with an
explicit storage and truncation contract.

## Decision

Use stderr for live provider progress so stdout remains available for the
command's stable result, especially under structured output modes. Keep this
task focused on eliminating the frozen-terminal behavior rather than combining
it with a new transcript-retention policy.

## Acceptance Notes

- [ ] Groom and audit stream agent output live while preserving parse behavior
- [ ] Structured-output parsing and golden tests still pass
