# Spike: design worktree isolation for ahm task work

## Summary

Parallel task execution via git worktrees works (validated 2026-07-12: task
181 ran in a manually created worktree while task 175 ran in the main tree,
merged back with only generated-index conflicts), but `ahm task work` has no
worktree support, and agent capabilities are uneven: cake has a native
`--worktree [name]` flag (auto-named worktrees, `.worktreeinclude` copy list,
cleanup guard); claude, codex, and cursor CLIs have no equivalent (verify
against current docs during the spike). Decide how ahm should offer worktree
isolation for delegated task work.

## Design Options

1. Delegate to agent-native worktrees (cake-only today, capability-gated per
   agent). Catch: `ahm task work` marks the task In Progress and regenerates
   indexes in the current tree before launching the agent; a worktree cut
   from committed HEAD will not contain that uncommitted record mutation, so
   the agent finds stale task state. Record sequencing must move inside the
   worktree.
2. ahm creates and manages the worktree itself (agent-agnostic, works for
   all four agents; matches the validated manual flow). Catch:
   `git worktree add -b` creates a branch, and ahm's compatibility contract
   says ahm never mutates branches or performs ref operations. This is a
   deliberate git-safety boundary extension and requires an ADR before code.

## Known Constraints

- cake task 257 (sandbox lacks gitdir/commondir rules for linked worktrees)
  currently breaks git for any cake session inside a worktree, including
  cake's own `--worktree` feature; it is a likely prerequisite either way.
- Records are branch-scoped: task status changes on the work branch merge
  back through normal git; generated task indexes conflict on merge and are
  resolved by rerunning `ahm index` (validated). The workflow docs for any
  worktree feature must state this merge-back procedure.
- Commit handoff inside a worktree could not run git under cake's sandbox
  (2026-07-12); the agent handed back manual commands. Handoff behavior
  needs a defined fallback when the agent cannot commit.
