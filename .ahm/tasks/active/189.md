# Improve ahm task work orchestration safety and recoverability

## Problem

`ahm task work` has grown from a single delegation into a multi-agent lifecycle spanning implementation, review, finalization, verification, and optional commit handoff. The current flow is useful but still relies on implicit authority, in-memory orchestration state, free-form review output, and broad worktree context. Failures or interruptions can therefore produce ambiguous success, lose resumable session state, or expose unrelated user changes to review and commit prompts.

## Existing Related Work

- Task 163 — ensures every supported agent invocation creates and exposes a resumable session.
- Task 171 — adds workflow-status validation to delegated completion and commit handoff.
- Task 185 — designs optional worktree isolation for delegated work.
- Task 187 — strengthens falsifiable evidence requirements in delegated reviews.
- Task 188 — designs environment capability gates for delegated verification.
- Task 118 — established the current default-on review and commit behavior; any change to commit authorization must explicitly reconcile or supersede that accepted decision.

## Scope

Track the focused child tasks that make phase outcomes correct, commit authority explicit, runs recoverable, review ownership unambiguous, pre-existing changes protected, and terminal/machine output coherent. Preserve ahm's guarantees that it does not patch project source, stage files, move HEAD, mutate branches, create project commits, or perform network/ref operations unless a separately accepted decision explicitly changes a boundary.
