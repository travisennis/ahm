# Require workflow status in delegated task work handoff

## Summary

`ahm task work` delegates implementation, embedded review, completion, and optional commit handoff to an external agent, but its built prompts do not consistently require `ahm --check workflow status` before finalization or commit handoff. Workflow-record drift can therefore survive an otherwise successful delegated run. Update the current task-work prompt sequence so the normal delegated path surfaces workflow validation warnings before completion is considered ready.

Written against commit `5cbd271`.

## Evidence

- `internal/ahm/task_work.go:199-220` - the implementation prompt asks the agent to load context, implement, verify, and complete the task without requiring workflow-scoped status validation.
- `internal/ahm/task_session.go:134-178` - the embedded review procedure and checklist are now the active review path.
- `internal/ahm/task_session.go:234-246` - the finalization prompt handles task completion and acceptance-note updates.
- `internal/ahm/task_session.go:249-272` - the current commit handoff prompt requires completed task state but not workflow-scoped validation.
- Completed task 156g removed the former shipped preflight template; this task must update the current prompt sequence rather than recreate that legacy surface.

## Fix Direction

Update `buildTaskWorkPrompt`, `taskWorkReviewProcedure`, `buildFinalizationPrompt`, and `buildTaskWorkCommitPrompt` so delegated agents run `ahm --check workflow status` at the appropriate completion boundary and address reported workflow issues before handoff. Keep this scoped to workflow validation rather than default `ahm status`, because link-scope warnings currently include known false positives tracked by task 161. Update current docs and focused prompt tests. Do not recreate the deleted preflight template or add a template-version change.

## Acceptance Notes

- [ ] The delegated review/finalization sequence requires `ahm --check workflow status` after task-record finalization and requires workflow errors or warnings to be addressed before considering the task done.
- [ ] The current commit handoff prompt requires `ahm --check workflow status` before committing completed work.
- [ ] The implementation prompt names workflow validation early enough that the agent can preserve required evidence and avoid surprising finalization failures.
- [ ] Tests covering the implementation, review, finalization, and commit prompts are updated or added without restoring the deleted preflight template.
- [ ] CLI docs for `task work` describe the workflow-status check in the delegated completion/commit path.
- [ ] Verification: `go test ./internal/ahm -run 'Test(TaskWork|Finalization|CommitPrompt)'` passes, then `just ci` passes before handoff.
