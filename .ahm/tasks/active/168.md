# Add ahm task edit command

## Problem

Users need a first-class way to edit an ahm-managed task file without manually
finding its path under the active, completed, or cancelled task buckets. Add
`ahm task edit <id>` so a user can open the resolved task file in the shell's
registered editor, save changes, exit, and have ahm regenerate indexes from the
updated task record.

This is user-visible CLI behavior. Before implementation, confirm whether the
change needs a small ADR under the project's ADR rules for durable CLI command
contracts.

## Fix Direction

- Add `ahm task edit <id>` under the existing `task` command family.
- Resolve `<id>` through the same task lookup rules used by `ahm task show <id>` so active, completed, and cancelled records can be located consistently.
- Read the editor command from the shell environment, using `EDITOR` as the required registered editor unless the project already has a standard `VISUAL`/`EDITOR` fallback convention.
- If no editor is configured, fail with a clear usage or environment error and do not rewrite indexes.
- Launch the editor attached to the current stdin, stdout, and stderr so interactive terminal editors work normally.
- After the editor process exits successfully, run the normal ahm index regeneration path so generated task, research, and ExecPlan indexes reflect any task metadata/body changes.
- If the editor exits non-zero or cannot be started, return a non-zero command result and do not regenerate indexes unless the implementation can prove the file was successfully edited and the existing command contract says otherwise.
- Keep dry-run behavior explicit: `--dry-run` should not launch an editor or mutate indexes, and should describe the task file that would be edited.
- Update help text, command reference docs, and tests/golden fixtures for success and failure cases.

## Acceptance Notes

- [ ] `ahm task edit <id>` opens the resolved task file in the configured editor.
- [ ] Saving and exiting the editor successfully regenerates indexes.
- [ ] Missing task IDs, unknown task IDs, missing editor configuration, editor start failures, and non-zero editor exits produce clear non-zero failures.
- [ ] `--dry-run` does not launch the editor and does not write indexes.
- [ ] Help text and task command reference docs include `ahm task edit <id>`.
- [ ] Tests cover successful editor invocation, index regeneration, dry-run behavior, and failure modes.
