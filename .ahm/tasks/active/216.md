# Resolve Claude Code permission denials during delegated task work

## Problem

During live smoke evidence for task 215, `ahm task work --agent claude` successfully started a session (`claude session started: ce3bb0d9`) and ran the independent review pass, but the Claude Code session denied every mutating operation. The agent could not:

- Run `ahm task complete 001` (the `Bash` tool was denied).
- Edit `.ahm/tasks/active/001.md` directly as a fallback (the `Edit` tool was denied).

This means the `claude` agent cannot currently perform delegated implementation work or finalize tasks through `ahm task work`. The restriction is Claude Code's own tool-use approval mechanism in non-interactive print mode (`-p`), not ahm. Until it is resolved, the `claude` agent is effectively unusable for real task work despite the `ANTHROPIC_API_KEY` fix in task 215.

## Relevant Files

- `internal/ahm/task_agents.go` — `claude`/`claudecode` argument builder; the place to add any required flags or config.
- `internal/ahm/task_work_smoke_test.go` — live smoke test that currently fails final validation for `claude` because the agent cannot mark the task complete.
- `docs/guardrails/external-agent-orchestration.md` — agent orchestration compatibility surface and required checks.
- `docs/adr/006-task-work-agent-delegation.md` — durable contract for delegated task work.

## Fix Direction

Investigate Claude Code's non-interactive permission model and determine the supported way to allow mutating `Bash` and `Edit` operations during `ahm task work --agent claude`. Likely approaches include:

- A Claude Code CLI flag (e.g., `--dangerously-skip-permissions`, `--permission-mode`, or similar) that pre-approves tool use in print mode.
- A repository-local or user-level Claude Code configuration that grants the agent permission to run `ahm` and edit task files.
- Passing explicit `--yes` or `--no-interactive` style flags if supported.

Do not bypass safety boundaries that the user has not explicitly opted into. The goal is to make `ahm task work --agent claude` work as reliably as the other agents (`cake`, `codex`, `cursor`) without requiring manual per-tool approval during a non-interactive run.
