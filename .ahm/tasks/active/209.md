# Detect asterisk checklist items in completed ExecPlan progress

## Problem

`execPlanSectionHasOpenProgress` in `validation.go` only recognizes `- [ ]`
items. The task-acceptance contract and helper also recognize `* [ ]`, but a
completed ExecPlan using that equivalent checklist marker does not produce
`exec_plan_completed_with_open_progress`. The two workflow validators should
apply the same unchecked-item grammar.

## Relevant Files

- `internal/ahm/validation.go`
- `internal/ahm/task_acceptance.go`
- `internal/ahm/validation_test.go`
- `docs/references/cli/task-file-format.md`

## Fix Direction

Reuse the existing unchecked-item helper from the ExecPlan progress check (rename
the helper if needed so its name is not task-specific). Preserve the established
grammar of optional leading indentation plus `- [ ]` or `* [ ]`; do not broaden
the file-format contract to other markers in this task. Update the documented
ExecPlan finding and add focused lifecycle-validation coverage.

## Acceptance Notes

- [ ] Completed ExecPlans with indented or unindented `- [ ]` and `* [ ]` Progress items emit `exec_plan_completed_with_open_progress`
- [ ] Checked items do not emit the finding
- [ ] Task acceptance behavior remains unchanged while both validators share one unchecked-item grammar
- [ ] `docs/references/cli/task-file-format.md` documents both accepted markers for the ExecPlan finding
- [ ] Focused validation tests and the existing package tests pass
