# Bound the project-doc discovery walk in docs check

## Problem

`projectDocFiles` claims its repo walk for nested AGENTS.md files is "depth-limited to avoid scanning vendor, node_modules, etc." but no depth limit exists — it walks the entire repository, skipping only dot-directories, vendor, and node_modules. Common build output directories (build/, dist/, target/, out/) are fully traversed on every `ahm docs check` and `--check project-docs` run, which is slow on large repositories.

## Relevant Files

- `internal/ahm/validation.go` (projectDocFiles walk and its comment)

## Fix Direction

Implement an explicit, named depth limit for the nested `AGENTS.md` walk and
skip common build-output directories (`build`, `dist`, `target`, and `out`).
Document the chosen depth beside the constant and retain coverage for root and
conventionally nested instruction files.

## Decision

Use both controls. A depth limit provides the actual upper bound promised by
the current comment, while the skip list avoids needless traversal within the
allowed depth. The implementation must choose and test a depth that preserves
the repository's existing nested-`AGENTS.md` convention.

## Acceptance Notes

- [ ] Walk has a tested explicit depth limit, skips common build-output directories, and has comments matching behavior
- [ ] Existing nested-AGENTS.md discovery tests still pass
