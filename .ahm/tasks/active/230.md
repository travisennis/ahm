# Bound the project-doc discovery walk in docs check

## Problem

`projectDocFiles` claims its repo walk for nested AGENTS.md files is "depth-limited to avoid scanning vendor, node_modules, etc." but no depth limit exists — it walks the entire repository, skipping only dot-directories, vendor, and node_modules. Common build output directories (build/, dist/, target/, out/) are fully traversed on every `ahm docs check` and `--check project-docs` run, which is slow on large repositories.

## Relevant Files

- `internal/ahm/validation.go` (projectDocFiles walk and its comment)

## Fix Direction

Implement an actual depth limit and/or extend the skip list with common build-output directory names, and make the comment describe the real behavior. Keep discovery results identical for repositories that keep AGENTS.md files at conventional depths.

## Acceptance Notes

- [ ] Walk is bounded (depth limit or expanded skip list) with the comment matching behavior
- [ ] Existing nested-AGENTS.md discovery tests still pass
