# Prevent dependency slice aliasing during task groom

## Problem

`validateGroomResult` shallow-copies `[]Task`, then truncates and appends to `DependsOn` using the original backing array. This mutates the `all` slice later passed to `applyGroomVerdicts`, so map iteration order and slice capacity can cause valid dependencies to disappear. `applyGroomVerdicts` repeats the in-place `[:0]` pattern.

## Relevant Files

- `internal/ahm/task_groom.go` — dependency validation and application.
- `internal/ahm/task_groom_test.go` — groom result regression coverage.

## Fix Direction

Build fresh dependency slices when validating and applying groom verdicts. Sort them deterministically and ensure validation cannot mutate its input task set.

## Acceptance Notes

- [ ] Groom validation and apply allocate fresh `DependsOn` slices.
- [ ] A regression test preserves all original dependencies while adding and removing dependencies with spare and full slice capacity.
- [ ] Repeated runs are deterministic and pass under the race detector.
