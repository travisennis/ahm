# Add live provider smoke coverage for ahm audit

## Blocker

Blocked on tasks 218 and 221. The live smoke must assert the finalized read-only advisory invocation contract and finalized empty-backlog label policy rather than encoding behavior those prerequisites are changing.

## Summary

At survey baseline commit `5ead0ae`, all `ahm audit` execution tests stub executable lookup and provider output. The opt-in live smoke suite exercises `task work` and `task groom` but not audit's provider-specific structured-output arguments, so provider CLI drift can break audit while the complete hermetic CI suite remains green. Add bounded live audit coverage after the advisory-permission and empty-vocabulary contracts are corrected.

## Evidence

- `internal/ahm/audit_test.go:37-104` — successful, invalid-result, and output-mode audit tests use stubbed runners rather than real provider CLIs.
- `internal/ahm/task_groom.go:319-349` — audit and grooming share provider-specific structured-result construction, including native schema flags and fallback argument builders.
- `justfile:70-74` — `just smoke-agents` selects only `TestAgentSmoke` and `TestTaskGroomAgentSmoke`.
- `docs/guides/testing.md:51-66` — the live checklist describes task-work and grooming smoke coverage, with no audit path.
- `docs/guides/testing.md:77-80` — golden parser fixtures cover task-work/review transport but do not establish that live audit schema flags still work.

## Fix Direction

Add an `AHM_AGENT_SMOKE=1`-gated audit smoke test using a throwaway initialized repository and every installed supported agent. Require a minimal schema-valid audit result, creation of a valid Open task with `source:audit`, and no project-source or Git-state mutation by the advisory subprocess. Extend the existing `just smoke-agents` recipe and testing guide rather than adding an always-on paid CI job. Do not record provider responses containing repository-sensitive data; follow the fixture scrubbing conventions in `internal/ahm/testdata/agents/README.md`.
