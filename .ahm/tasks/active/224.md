# Apply task and ADR migrate writes under --json and --plain

## Problem

`taskMigrate` and `adrMigrate` treat `--json` and `--plain` as preview modes: `if a.opts.dryRun || a.opts.json || a.opts.plain { return preview }`. An output-format flag silently suppresses writes, contradicting the documented contract (docs/references/cli/global-contract.md describes --json/--plain purely as output modes; docs/references/cli/task-commands.md lists --dry-run and --json/--plain as independent flags). A scripted `ahm --json task migrate` migrates nothing with no indication writes were skipped.

## Relevant Files

- `internal/ahm/task_migrate.go` (taskMigrate)
- `internal/ahm/adr_migrate.go` (adrMigrate)
- `docs/references/cli/task-commands.md`
- `docs/references/cli/global-contract.md`

## Fix Direction

Apply migrations under `--json` and `--plain` and emit the report in the
requested format. Only `--dry-run` suppresses writes. Update docs and golden
tests to match.

## Decision

Preserve the documented global contract that `--json` and `--plain` select
output formats rather than changing command semantics. This avoids a silent
compatibility trap for scripted callers and keeps preview behavior owned by
`--dry-run`.

## Acceptance Notes

- [ ] `ahm --json task migrate` and `ahm --json adr migrate` behavior matches the documented contract
- [ ] Structured output distinguishes applied vs previewed migrations
- [ ] CLI reference docs updated
