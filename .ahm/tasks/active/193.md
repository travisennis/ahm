# Make rendered task front matter round-trip safe

## Problem

`renderTask` emits unescaped front-matter scalars that the parser cannot reliably read back. Titles beginning with block scalar/list syntax become malformed, and quoted titles are silently changed during parsing. Newline rejection from task 148 does not cover these single-line values.

## Relevant Files

- `internal/ahm/tasks.go` — front-matter parser and renderer.
- `internal/ahm/task_create.go` — create-boundary validation.
- `internal/ahm/tasks_test.go`, `task_commands_test.go` — round-trip and CLI tests.
- `docs/references/workflow-spec.md` — supported scalar grammar.

## Fix Direction

Choose and document a deterministic scalar quoting/escaping scheme, or reject every scalar shape the renderer cannot round-trip. Preserve unknown-field behavior and existing valid files.

## Acceptance Notes

- [ ] Every accepted task title and label value round-trips without corruption.
- [ ] Block-scalar, block-list-like, quote, colon, and whitespace edge cases have tests.
- [ ] Existing task files and unknown front-matter fields remain compatible.
- [ ] The workflow specification documents the supported representation.
