# Memoize workflow path config resolution per command

## Problem

`workflowPathsFor` calls `readMetadataWithSource` on every invocation, repeatedly reading and JSON-parsing repository config throughout one command. Hot paths call it during task collection, per-task ExecPlan checks, ID allocation, locking, and index generation.

## Relevant Files

- `internal/ahm/workflow_paths.go` — metadata-backed path resolution.
- `internal/ahm/cli.go` — per-command app state.
- Workflow command and validation call sites using `workflowPathsFor`.

## Fix Direction

Resolve workflow paths once per command/app and pass or memoize the result, while keeping package-level helpers available for standalone validation where appropriate. Define invalidation only for commands that create or migrate config.

## Acceptance Notes

- [ ] Normal commands read workflow config once for path selection.
- [ ] Fresh init, records migration, and corrupt-config behavior remain correct.
- [ ] Tests or instrumentation pin the reduced config reads.
