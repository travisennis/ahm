# Strip ANTHROPIC_API_KEY from claude agent invocation environment

## Problem

When `ahm task work` invokes the `claude` agent (Claude Code) in non-interactive print mode (`-p`), the child process inherits the parent environment, which includes `ANTHROPIC_API_KEY`. In non-interactive mode, Claude Code prioritizes the `ANTHROPIC_API_KEY` environment variable for billing and ignores the active Claude subscription. For users whose API key has no balance (because they use a subscription plan), this causes every `ahm task work --agent claude` invocation to fail with `Credit balance is too low`.

This was observed during the live smoke evidence for task 163: `claude` returned `Credit balance is too low` while `cake`, `codex`, and `cursor` all succeeded.

The root cause is in `runTaskWorkCommand` (`internal/ahm/task_work.go`): it does not set `cmd.Env`, so the child process inherits `os.Environ()` verbatim, including `ANTHROPIC_API_KEY`.

## Relevant Files

- `internal/ahm/task_work.go` — `runTaskWorkCommand` builds and runs the `exec.CommandContext` for agent invocations; currently does not filter the environment.
- `internal/ahm/task_agents.go` — agent definitions; the `claude`/`claudecode` case is the affected agent.
- `internal/ahm/git.go` — `cleanGitEnvironment()` is the existing pattern for filtering inherited environment variables before spawning a child process.
- `internal/ahm/task_work_smoke_test.go` — live smoke tests for agent invocations.
- `internal/ahm/task_commands_test.go` — unit tests for task work command wiring and dry-run output.
- `docs/guardrails/external-agent-orchestration.md` — guardrail for agent orchestration compatibility surfaces.
- `docs/adr/006-task-work-agent-delegation.md` — ADR for task work agent delegation.

## Fix Direction

Introduce a per-agent environment filtering mechanism so that `runTaskWorkCommand` can strip specific environment variables before spawning the agent child process. For the `claude` agent, strip `ANTHROPIC_API_KEY` so that Claude Code falls back to subscription-based authentication in non-interactive mode.

Design considerations:

- Follow the existing `cleanGitEnvironment()` pattern: build a filtered copy of `os.Environ()` rather than constructing an allowlist from scratch, so other required environment variables (PATH, HOME, etc.) are preserved.
- Add a field to `taskWorkAgent` (e.g., `envFilter func([]string) []string` or a `blockedEnvVars []string` slice) so each agent can declare which environment variables it wants stripped. Only the `claude` agent needs filtering today, but the design should be extensible for future agents.
- Wire the filtered environment into `runTaskWorkCommand` by passing it through the call chain (either via the `taskWorkAgent` struct or an additional parameter).
- Keep the change surgical: do not alter argument builders, parsers, session capture, or resume behavior.
- Add focused unit tests verifying that `ANTHROPIC_API_KEY` is absent from the spawned environment for `claude` and present for other agents.
- Update the dry-run output if the environment filtering is visible there (it should not be, but verify).
