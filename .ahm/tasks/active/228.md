# Warn when ADR parse errors are skipped during index generation

## Problem

`indexWritesForPaths` ignores the error from `collectADRs` whenever any ADR parsed (`if err != nil && len(adrs) == 0`), with no warning — unlike the task path, which warns about skipped unparseable files. Because `renderADRIndex` skips malformed ADRs, `ahm index` can silently drop entries from docs/adr/index.md.

## Relevant Files

- `internal/ahm/indexes.go` (indexWritesForPaths, renderADRIndex)
- `internal/ahm/adrs.go` (collectADRs)

## Fix Direction

Surface the partial-parse error as a warning on the same channel the task path uses ("some ADR files could not be parsed and were skipped: ..."), threading a warning sink or returning the error alongside the writes so app-level callers can report it.

## Acceptance Notes

- [ ] Index generation with a malformed ADR present emits a warning naming the file
- [ ] Test covers the partial-parse warning path
