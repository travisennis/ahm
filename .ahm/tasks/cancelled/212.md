# cleanGitEnvironment re-allocates environment slice on every call

## Problem

cleanGitEnvironment in git.go calls os.Environ() and filters out Git repository-location variables on every invocation. It is called from runGitBytes (every ahm-owned git subprocess) and readGitContext (context/prime git status). For commands that run multiple git operations (prime, status, context), this scans all environment variables multiple times unnecessarily.

## Relevant Files

- `internal/ahm/git.go`

## Fix Direction

Cache the cleaned environment at package level: compute it once via sync.OnceValue or at init time, and return the cached slice from cleanGitEnvironment. This eliminates repeated os.Environ() scans on every git subprocess invocation and context read.

## Acceptance Notes

- [ ] cleanGitEnvironment returns the identical filtered environment as before
- [ ] The environment is captured once and reused for the process lifetime
- [ ] All git subprocess calls continue to work correctly

## Cancellation Reason

Caching os.Environ would freeze later environment changes and return shared mutable slice state for an unmeasured micro-optimization; per-invocation filtering is part of the Git subprocess isolation boundary.
