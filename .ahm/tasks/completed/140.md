# Implement records metadata and storage mode model

## Ready Note

ADR 013 and the ref-backed workflow record ExecPlan are finalized. This is the
first implementation task for the accepted storage boundary.

## Summary

Add the metadata and internal model needed for committed legacy records, gitignored local records, and ref-backed record sync.

## Relevant Records

- ADR: `docs/adr/013-use-ref-backed-workflow-record-storage.md`
- ExecPlan: `.agents/exec-plans/active/138-ref-backed-workflow-records.md`

## Work Direction

Introduce committed `.ahm/config.json` configuration while preserving legacy
`.agents/ahm.json` parsing so repositories that have not opted in keep their
current committed-record behavior. The internal model should represent legacy
committed records, gitignored local `.ahm/` records, and ref-backed records.
Absence of new metadata must preserve existing behavior. Keep field names and
defaults conservative; likely fields include `store_mode`, `records_ref`,
`records_remote`, and last successful sync metadata.

## Acceptance Notes

- [x] Missing records metadata preserves current committed-record behavior.
- [x] Ref-backed mode can be represented in committed `.ahm/config.json`
      without breaking old `.agents/ahm.json` metadata parsing.
- [x] Metadata round-trip tests cover unknown fields and default behavior.
- [x] Workflow spec updates are identified for task 145; this task added the
      initial `.ahm/config.json` compatibility notes to `docs/references/workflow-spec.md`,
      `docs/guides/workflow-upgrades.md`, and `ARCHITECTURE.md`, while task 145
      remains responsible for full user-facing ref-backed records documentation.
