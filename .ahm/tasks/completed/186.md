# Finalize Delegated Tasks Only After Review Is Resolved

## Summary

At commit `b6af9f7`, the default `ahm task work` sequence asks the implementation agent to mark a task Completed before the independent review phase starts. Review feedback is then sent back to that implementation session, but there is no post-review lifecycle gate ensuring the task is finalized only after feedback is resolved. A failed implementation can therefore leave authoritative task metadata claiming completion while review is still pending or while corrective work remains.

## Evidence

- `internal/ahm/task_work.go:158-164` — the base implementation prompt tells the delegated agent to fill Acceptance Notes, verify, and mark the task complete.
- `internal/ahm/task_session.go:47-63` — `taskWorkWithSession` reloads that already-mutated task and only then invokes the reviewer.
- `internal/ahm/task_session.go:72` — the reviewer prompt says it is reviewing "before task completion," contradicting the lifecycle state it is commonly given.
- `internal/ahm/task_session.go:140-167` — actionable feedback resumes the implementation session, but no subsequent task reload or lifecycle validation occurs before the optional commit handoff.
- `.agents/.tasks/completed/119.md` — prior design deliberately removed ahm-owned `--complete` plumbing in favor of agent-driven completion; this task must preserve that decision rather than reintroduce the removed flag.

## Fix Direction

Make the review-enabled workflow explicitly two-phase: implementation reaches a review-ready state without final completion, independent review runs, and the implementation session receives a finalization handoff whether the reviewer found issues or not. That handoff must require resolving feedback, re-running relevant verification, updating Acceptance Notes, and only then completing the task; ahm must reload and validate the resulting task state before commit handoff. Preserve agent-driven lifecycle changes and existing `--no-review` semantics, and do not restore the removed `--complete` option. If this lifecycle change conflicts with ADR 006, ADR 008, ADR 012, or task 119's accepted decision in a way that cannot be reconciled, stop and record the required ADR update before implementation.
