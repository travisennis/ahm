# Expose suggested AGENTS.md additions

## Summary

`ahm` currently ships a starter `AGENTS.md` for repositories that do not already
have one. If a repository already has `AGENTS.md`, `ahm init` and `ahm upgrade`
correctly skip it because the file is project-owned and must not be overwritten.

Some of the starter guidance is still useful for existing projects. Add an
advisory CLI surface that lets a project's agent ask `ahm` for suggested
`AGENTS.md` additions, then review and incorporate those suggestions into the
existing project-owned file.

## Proposal

Add an advisory command such as:

```bash
ahm agents suggestions
ahm agents suggestions --json
ahm agents suggestions --all
```

The command should print Markdown snippets by default and structured suggestion
objects with `--json`. The suggestions should be reusable blocks such as task
workflow, research workflow, ExecPlans, documentation workflow, generated
indexes, task state transitions, and no implicit commits.

The command must not modify `AGENTS.md`. It should preserve the existing
ownership model: `ahm` proposes, the repository's agent or maintainer decides
how to merge the guidance.

## Implementation Notes

- Keep `AGENTS.md` create-only during `init`, `upgrade`, and `--force`.
- Prefer one canonical source for starter `AGENTS.md` content and suggestion
  snippets so they do not drift.
- Document the new command in `docs/cli.md`.
- Update workflow semantics docs if needed to clarify advisory suggestions.
- Add tests for human output, JSON output, and the no-write behavior.

## Acceptance Notes

- Added `--all` to show blocks that already appear present, and `--json` for

- Kept `AGENTS.md` create-only for `init`, `upgrade`, and `--force`.
