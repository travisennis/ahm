# Harden task front matter parsing

## Summary

Review issue 3. parseFrontMatter is an ad hoc YAML-like parser that mishandles quoted edge values, comments, block forms, and values containing punctuation. Choose a real YAML parser or define and validate a strict supported grammar. Acceptance: valid task front matter with punctuation round-trips correctly, unsupported shapes fail with clear errors, and docs or tests define the supported format.

## Acceptance Notes

- [x] Valid front matter with colons in values round-trips correctly (labels: type:bug, area:tasks)
- [x] Double-quoted values have only surrounding quotes stripped (title: "My Task: The Reckoning")
- [x] Blank lines and comment lines (#) in front matter are silently skipped
- [x] Block scalar indicators (|, >) produce clear task_malformed errors
- [x] Keys containing spaces or colons produce clear task_malformed errors
- [x] validateTaskFrontMatter reports task_malformed (not misleading task_missing_field) for unsupported shapes
- [x] frontMatterValue and normalizeDependsOnValue use consistent quote-stripping logic
- [x] Strict grammar is documented in the parseFrontMatter func comment; comprehensive tests define the supported format
