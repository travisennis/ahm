# Scrub hook-injected GIT_DIR env so test fixtures cannot mutate the host repo

## Summary

Go test fixtures shell out to git without scrubbing hook-injected environment. When the suite runs under a git hook (pre-commit go-test), GIT_DIR is exported and overrides the fixtures' git -C <tempdir> isolation, so fixture git commands operate on the REAL repository: observed writes of core.worktree (breaking git status with fatal invalid-path), user.name/user.email "Test User <test@example.com>" into .git/config (real commits then carry the fixture identity), staged fixture files (.ahm/config.json version test, tracked.txt), staged deletions of docs/, and an emptied generated ADR index. Reproduced 2026-07-12 during task 181/175 runs; commit d2ad369 was authored as Test User and needed --reset-author. Does not reproduce outside hooks because GIT_DIR is unset. go test caching hides it until tests actually execute.

## Relevant Files

- internal/ahm/test_helpers_test.go (add a TestMain or helper that scrubs GIT_DIR, GIT_WORK_TREE, GIT_INDEX_FILE, GIT_OBJECT_DIRECTORY, GIT_COMMON_DIR via os.Unsetenv/t.Setenv)
- internal/ahm/prime_test.go, context_test.go, records_migrate_test.go (fixtures that exec git)
- any code under test that shells out to git (readGitContext) must also pass a scrubbed env

## Fix Direction

Scrub GIT_* repo-location variables once for the test process (TestMain) and/or set cmd.Env with a clean environment wherever tests or production helpers exec git. Add a regression test that sets GIT_DIR to a bogus path and asserts fixture git commands still act on their -C directory. Until fixed, committing with hooks that run go test can corrupt the host repository; SKIP=go-test with a manual test run is the workaround.
