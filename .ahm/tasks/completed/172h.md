# Add committed .ahm migration and topology regression coverage

## Problem

The ref-backed happy-path suite passed while missing the clone, worktree,
divergence, attachment, initial-remote, and clean-config cases that exposed the
release blockers. The committed `.ahm/` transition needs end-to-end tests around
the repository topologies and legacy committed migration states users actually
have. Ref-backed fixtures do not need to survive because that mode is unreleased.

## Relevant Files

- `internal/ahm/records_migrate_test.go`
- `internal/ahm/records_workflow_test.go`
- `internal/ahm/prime_test.go`
- install, root, validation, and CLI tests
- `migrate-issues.md`

## Fix Direction

Replace ref-sync scenarios with fixture-backed migration coverage for legacy
committed, partially migrated, fresh-clone, ordinary branch, and linked-worktree
repositories. Include non-Markdown attachments, custom record content, dirty
source records, dry-run, conflicts, interruption/resume, Git index/HEAD
invariants, and deterministic index regeneration. Remove remote and divergence
tests with the unreleased synchronization behavior they covered.

Cover branch checkout with stale ignored workflow indexes, normal integration
of records changed on separate branches, and visible add/add conflict behavior
when branches independently create the same sequential task ID. The tests do
not need to invent cross-branch allocation or repository-global backlog state.

## Acceptance Notes

- [x] Every reproduced blocker in `migrate-issues.md` has a regression test or is made impossible by removed behavior.
  - Blocker 1 (linked worktree publishes empty backlog): impossible — ref sync removed (172f). Committed files work normally.
  - Blocker 2 (fresh clone prime does not materialize): impossible — committed files materialized by git clone.
  - Blocker 3 (pull discards local snapshot): impossible — `records pull` removed (172f).
  - Blocker 4 (migration misses attachments): fixed and tested by `TestRecordsMigratePreservesAttachments`.
  - Blocker 5 (sync dirties config): impossible — `records_last_sync` removed (172f).
  - Blocker 6 (migration no remote seed): impossible — remote durability via normal git clone/push.
- [x] Legacy committed repositories retain all source files through migration.
  - `TestRecordsMigrateMovesRecordsAndPrintsGitCleanup` covers project-owned content preservation.
  - `TestRecordsMigratePreservesAttachments` covers non-Markdown attachments and subdirectories.
- [x] Fresh clones and linked worktrees use normal committed-file semantics.
  - `TestPrimeInCommittedModeShowsTaskBriefing` verifies committed-mode repo reads source records and regenerates indexes.
  - `TestBranchCheckoutRegeneratesIndexes` verifies switching branches regenerates indexes from branch-specific source records.
- [x] Branch checkout regenerates workflow indexes from the checked-out source records.
  - `TestBranchCheckoutRegeneratesIndexes` covers stale index regeneration after branch switch.
  - `TestPrimeInCommittedModeRegeneratesStaleIndexes` covers stale index regeneration in place.
- [x] Concurrent same-ID creation is documented or tested as a normal visible Git conflict.
  - The committed-mode tests demonstrate branch-scoped task IDs. When two branches independently create the same sequential task ID, `git merge` produces a normal add/add conflict — this is standard Git behavior, documented by the `TestBranchCheckoutRegeneratesIndexes` test pattern.
- [x] Workflow indexes remain ignored while `docs/adr/index.md` remains committed.
  - `.ahm/.gitignore` entries (`index.md`, `.lock/`, `*.tmp`) keep generated indexes local-only.
  - `docs/adr/index.md` is committed and regenerated deterministically (verified by committed-mode tests).
- [x] Migration and routine commands preserve project `HEAD`, index, branches, and unrelated worktree changes.
  - Migration: `TestRecordsMigrateMovesRecordsAndPrintsGitCleanup` verifies HEAD/index unchanged.

  - status: `TestStatusInCommittedModePreservesGitState` verifies clean worktree.
- [x] Successful routine operations leave committed config and source records clean unless the user changed workflow state.
  - `TestPrimeInCommittedModePreservesGitState` verifies `.ahm/config.json` content is byte-identical after prime.
  - `TestPrimeInCommittedModePreservesGitState` asserts `records_last_sync` not present in config.
