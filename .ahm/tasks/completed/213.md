# Centralize Markdown link walking for documentation validation

## Problem

Four functions in `validation.go` independently split Markdown into lines,
toggle fenced-code state, strip inline code spans, and run
`markdownLinkPattern`: `validateMarkdownFileLinksWithCodes`,
`validateProjectDocLinkPortability`, `designDocIndexTargets`, and
`docIndexTargets`. The repeated traversal has already required coordinated fixes
and makes it easy for the validators and index-coverage checks to diverge.

## Relevant Files

- `internal/ahm/validation.go`
- `internal/ahm/validation_test.go`

## Fix Direction

Extract a small shared walker that preserves the current traversal contract:
line numbers are one-based, lines inside backtick or tilde fences are skipped,
inline code spans are stripped, and the callback receives each raw link target
outside code. Keep target normalization, skip rules, path resolution, and
finding emission in the four callers. This is a behavior-preserving refactor;
changes to Markdown fence semantics or link parsing are out of scope.

## Acceptance Notes

- [x] All four functions use one shared Markdown link walker
- [x] Existing missing-link, portability, design-index, and general doc-index findings are byte-for-byte unchanged for existing fixtures
- [x] Focused tests cover line numbers plus ignored links in backtick fences, tilde fences, and inline code spans through the shared behavior
- [x] Target-specific normalization and filtering remain in each caller
- [x] No new dependency, global mutable state, or broader Markdown parser is introduced
