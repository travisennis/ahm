# Add prompt field to task work dry-run preview

## Summary

The `task work --dry-run` preview shows executable, args, task ID, agent, and orchestration flags — but not the actual prompt text that gets sent to the agent. The prompt is buried inside `args` as a long string. An agent previewing the command can't easily see what instructions the external agent will receive.

## Evidence

```
$ ahm task work 001 --agent codex --dry-run
{
  "task": "001",
  "agent": "codex",
  "executable": "/usr/local/bin/codex",
  "args": ["exec", "--dangerously-bypass-approvals-and-sandbox", "--json", "Work on task 001.\n\nBefore making changes, run ahm context task..."],
  "status": "In Progress",
  "commit": true,
  "review": true
}
```

The prompt is the last element of `args` but it's long and not labeled. Extracting it as a top-level `"prompt"` field would make the preview more readable and help agents understand what `task work` actually sends.

- `internal/ahm/task_work.go` — `taskWork()` builds the dry-run preview map with `task`, `agent`, `executable`, `args`, `status`, `commit`, `review` keys. The prompt is constructed via `a.buildTaskWorkPrompt(task)` but only passed into `agent.args(prompt)` — it's not included as a separate field in the preview.

## Fix Direction

Add `"prompt": prompt` to the dry-run preview map in `taskWork()`, between `"args"` and `"status"`:

```go
preview := map[string]any{
    "task":       task.ID,
    "agent":      agent.name,
    "executable": executable,
    "args":       args,
    "prompt":     prompt,
    "status":     taskWorkDryRunStatus(task.Status),
}
```
