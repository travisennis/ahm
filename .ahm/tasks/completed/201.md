# Stabilize install JSON result keys including adopted

## Problem

Install JSON results initialize `created`, `updated`, `removed`, `skipped`, and `conflicts` as empty arrays, but add `adopted` only when populated. Structured consumers therefore see a schema whose keys vary with data.

## Relevant Files

- `internal/ahm/install.go` — install result construction.
- `internal/ahm/install_test.go` — init/upgrade JSON output tests.
- `docs/cli.md` and `docs/references/cli/commands.md` — install output contract.

## Fix Direction

Initialize `adopted` with the other result-list keys and document a stable structured output shape. Preserve text rendering behavior unless explicitly documented.

## Acceptance Notes

- [x] JSON always includes `adopted` as an array, including when empty.
- [x] A table-driven test pins the complete array-valued result schema for
  both `init` and `upgrade`, rejecting missing, extra, or mistyped fields.
- [x] `docs/cli.md` and the command reference describe the stable key set and
  the additional dry-run `directories` array.
