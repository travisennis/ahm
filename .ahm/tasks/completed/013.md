# Remove splitTaskID magic numeric sentinel

## Summary

Review issue 13. splitTaskID uses 999999 as a sentinel for non-numeric task IDs, which collides with a possible real task ID. Acceptance: splitTaskID returns an explicit ok boolean or equivalent, nextTaskID ignores non-numeric IDs without sentinel values, and tests cover 999999 and non-numeric IDs.

## Acceptance Notes

- [x] `splitTaskID` now returns `(int, string, bool)` instead of using 999999 sentinel
- [x] `nextTaskID` uses the `ok` boolean to skip non-numeric IDs
- [x] `resolveTaskFromTasks` requires both pattern and task ID to have numeric prefixes before matching
- [x] `taskLess` handles non-numeric IDs explicitly (numeric sorts before non-numeric; non-numeric compares by string)
- [x] Tests added for `splitTaskID` covering: 001, 010, 999999 (real ID), 002a, 047b, abc (non-numeric), empty string, and bare numeric
