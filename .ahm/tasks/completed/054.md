# Add ahm-owned file editing guidance

## Summary

Projects using `ahm` need explicit agent-facing guidance that separates
project-owned files from files owned, maintained, generated, or upgraded by
`ahm`. The current starter `AGENTS.md` and `ahm agents suggestions` guidance
already says not to edit generated indexes by hand, but it does not state the
broader rule: do not manually update `ahm`-provided workflow files just to
customize or repair them. Use `ahm` commands, update source workflow records
where allowed, or change this repository's templates so future installs and
upgrades carry the correction.

## Proposed Instruction Language

Add a reusable advisory block along these lines:

```markdown
`ahm` owns the workflow files it installs, maintains, generates, and upgrades.
Do not hand-edit ahm-owned generated indexes or managed workflow templates in a
consumer project to change ahm behavior or guidance. Update the source task,
research note, ExecPlan, or other supported project-owned record, then run the
appropriate `ahm` command such as `ahm index`, `ahm task complete <id>`, or
`ahm upgrade`. If the installed guidance itself needs to change, update the
canonical templates in the `ahm` repository and let projects receive the change
through `ahm upgrade`.
```

When implementing, tighten this wording so it is clear without being too broad:

- `AGENTS.md` remains project-owned and advisory after creation; projects may
  edit it.
- Task files, research notes, ExecPlans, and ADRs are normal workflow source
  records that agents may update through the documented workflow.
- Generated indexes must be regenerated, not hand-edited.
- Managed template files under `.agents/` and `docs/adr/README.md` should not be
  customized locally as a way to change ahm-provided process guidance; durable
  guidance changes belong in `internal/templates/workflow/` and should flow via
  `ahm upgrade`.

## Likely Implementation Locations

- Add a new `AgentSuggestion` block in `internal/templates/templates.go`, likely
  near or before the existing `generated-indexes` block.
- Regenerate/align `internal/templates/workflow/AGENTS.md` through
  `RenderAgentsMarkdown` expectations.
- Update `docs/cli.md` for `ahm agents suggestions` if the suggestion set or
  wording changes.
- Update `docs/spec.md` and/or `docs/upgrades.md` to document the ownership
  boundary between project-owned workflow records, generated indexes, managed
  templates, and create-only `AGENTS.md`.
- Add or adjust tests in `internal/templates/templates_test.go` and
  `internal/ahm/agents_test.go`.

## Acceptance Notes

- `ahm agents suggestions` includes guidance telling consumer projects not to
  hand-edit ahm-owned generated or managed files as a substitute for using ahm
  commands or updating ahm templates.
- Starter `AGENTS.md` for new installs includes equivalent guidance.
- Documentation describes the ownership boundary clearly, including that
  `AGENTS.md` is project-owned after creation.
- Tests cover the new suggestion block and starter AGENTS rendering.
- Verification includes focused template/agents tests and `just ci`.
