# Standardize app methods on pointer receivers

## Summary

Review issue 5. app methods mix value and pointer receivers, creating a footgun for option or state mutation. Acceptance: command, install, status, task, index, and validation methods consistently use pointer receivers where they operate on app state, with tests passing and no behavior changes.

## Acceptance Notes

- [x] All 19 value receiver methods on `app` (`install`, `ensureWorkflowDirs`, `status`, `doctor`, `emit`, `taskCreateParsed`, `taskMigrate`, `taskList`, `taskNext`, `printTaskLine`, `taskShow`, `taskStatus`, `resolveTask`, `taskDepUpdate`, `taskDepTree`, `taskDepCycles`, `writeIndexes`, `indexWriteTargets`, `indexWrites`) converted to pointer receivers (`func (a *app)`).
- [x] Remaining 8 pointer receiver methods already used `*app` — now all 27 `app` methods are consistent.
- [x] `go fmt`, `go vet`, `go test -race -cover ./...`, `golangci-lint`, `govulncheck`, and GoReleaser all pass.
- [x] No behavior changes — all 42 tests pass unchanged with same coverage (82.4%).
