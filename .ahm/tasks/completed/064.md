# Skip rewriting unchanged generated indexes

## Problem

`writeIndexes` (`indexes.go:24-35`) writes all 7 generated index files
unconditionally on every mutating command; the change-detection helper
`isStaleIndex` is consulted only in dry-run mode. Every write goes through
`writeFileAtomic`, which fsyncs the temp file and the parent directory — about
16 fsyncs per `task complete` even when most indexes are byte-identical. This
also churns mtimes (dirtying editors/watchers/build tools) and is the main
reason `go test ./internal/ahm` takes ~11s (fsync-bound test I/O).

## Fix Direction

- In the non-dry-run path of `writeIndexes`, skip the write when the existing
  file content already matches (reuse `isStaleIndex`).
- Keep dry-run behavior identical (task 045 semantics: report only stale
  indexes).
- Optional follow-on: a package-level test hook to skip fsync in tests if the
  suite is still slow after skip-unchanged lands.

## Relevant Files

- `internal/ahm/indexes.go:12-48`
- `internal/ahm/write.go` (fsync cost context)
- `docs/spec.md` (atomic write guarantee wording, if it mentions
  unconditional rewrites)
