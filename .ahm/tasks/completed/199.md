# Handle front matter closing delimiters at EOF

## Problem

`splitFrontMatter` recognizes a closing delimiter only when followed by a newline. A Markdown file ending exactly at `---` is treated as though it has no front matter, producing misleading downstream behavior instead of a precise malformed/unsupported boundary result.

## Relevant Files

- `internal/ahm/tasks.go` — shared front-matter splitter.
- `internal/ahm/adrs.go`, `task_migrate.go` — splitter consumers.
- `internal/ahm/tasks_test.go`, `task_migrate_test.go` — parsing tests.

## Fix Direction

Define EOF behavior for a closing delimiter and make the splitter distinguish absent front matter from malformed front matter where callers need an error. Cover LF and CRLF inputs.

## Acceptance Notes

- [x] A closing delimiter at EOF is handled according to the documented grammar.
  - `splitFrontMatter` now recognizes `\n---` at EOF and `---` immediately after the opening delimiter.
  - `splitRawFrontMatter` handles the same EOF cases for both LF and CRLF newline styles.
  - `docs/references/workflow-spec.md` documents the delimiter/EOF behavior.
- [x] Unterminated front matter produces an explicit parse/validation error rather than appearing absent.
  - `parseFrontMatter` returns `front matter is not closed` when the opening delimiter is present without a closing delimiter.
  - `parseADRFromData` returns `ADR front matter is not closed` for unterminated ADR front matter.
  - `migrateTaskFrontMatter` reports `front matter is not closed; fix manually` as a migration change instead of silently treating the file as legacy.
  - A lone `---` (with or without a trailing newline) is now treated as an unclosed opening delimiter rather than "no front matter".
- [x] Task, ADR, and migration consumers have consistent tests.
  - Added `TestParseFrontMatter_EOF` in `internal/ahm/tasks_test.go`.
  - Added `TestMigrateTaskFrontMatter_EOF` in `internal/ahm/task_migrate_test.go`.
  - Added `TestParseADR_EOF` and `TestSplitRawFrontMatter_EOF` in `internal/ahm/adrs_test.go`.
  - Full suite passes: `go test ./...`, `go vet ./...`, `gofmt -l internal/ahm/*.go`.
