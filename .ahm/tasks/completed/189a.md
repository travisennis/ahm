# Make task work success require completed orchestration

## Problem

Several `ahm task work` paths can return success without completing the lifecycle the command claims to run. If session-ID parsing fails while review is enabled but commit is disabled, the command warns and returns without review or finalization. Review-feedback parse failure is treated as empty feedback and therefore as a no-findings review. The `--no-review --no-commit` path does not reload the task and verify that direct agent-driven completion actually occurred.

## Relevant Files

- `internal/ahm/task_session.go` — session capture, review parsing, finalization, and postcondition checks.
- `internal/ahm/task_work.go` — flag-derived orchestration requirements and direct-completion path.
- `internal/ahm/task_commands_test.go` — orchestration failure and exit-code coverage.
- `docs/references/cli/task-commands.md` — documented success and warning semantics.

## Fix Direction

Define required phases from the requested flags and make their prerequisites mandatory. A missing or unparseable implementation session ID must fail whenever review, finalization, or commit requires resume. Unparseable review output must be a review failure, not "no findings." Reload and validate the task after both reviewed and `--no-review` completion paths. Preserve the existing repository acceptance policy until child task 189g decides whether delegated work should enforce a stronger default.
