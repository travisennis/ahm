# Sort recent research globally before prime caps output

## Problem

`primeRecentResearch` documents the newest five notes by filename but never sorts. `os.ReadDir` yields ascending names, and notes are appended bucket-by-bucket, so output can show the oldest files and let the first bucket consume the entire cap.

## Relevant Files

- `internal/ahm/prime.go` — recent research collection.
- `internal/ahm/prime_test.go` — prime output ordering and caps.
- `docs/references/cli/commands.md` — prime output contract.

## Fix Direction

Collect eligible notes across all buckets, sort them by the documented recency key with a deterministic tie-breaker, then apply the global cap.

## Acceptance Notes

- [x] Prime returns the globally newest five research notes by sorting eligible
  filenames in descending order before applying the cap.
- [x] The regression test includes notes from every bucket and confirms that
  multiple buckets participate in the capped result.
- [x] Equal filenames use the note link as a deterministic tie-breaker, covered
  by the regression test and documented in the CLI reference.
