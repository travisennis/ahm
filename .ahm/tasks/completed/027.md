# Avoid repeated allocation of static template slices

## Summary

Review issue 27. Files and generatedIndexTargets allocate fresh slices on every call even though contents are static. Acceptance: use package-level static data or another allocation-free pattern without exposing mutable shared state unsafely.

## Acceptance Notes

- [x] `templates.Files()` returns a package-level var instead of allocating a fresh slice on each call
- [x] `templates.AgentSuggestions()` returns a package-level var instead of allocating a fresh slice on each call
- [x] `generatedIndexTargets()` (in `internal/ahm/indexes.go`) returns a package-level var instead of allocating a fresh slice on each call
- [x] All callers are read-only iterators; no caller mutates the returned slices
- [x] All existing tests pass (`go test ./internal/templates ./internal/ahm`)
