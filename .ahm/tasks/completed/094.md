# Rename review workflow to preflight

## Problem

The review-readiness workflow is currently named `deslop`, which is too informal for project documentation, agent instructions, and CLI-facing workflow references. The underlying idea is useful, but the name makes the workflow feel like maintained tooling when exposed as `preflight`.

## Relevant Files

- `.agents/skills/deslop/`
- `.agents/skills/deslop/SKILL.md`
- `.agents/TASKS.md`
- Agent and task workflow docs that reference `deslop`
- Task 091, if it still references the current name
- Any tests or fixtures that mention the skill by name

## Fix Direction

Rename the workflow from `deslop` to `preflight`. The name describes the systematic self-review pass before handoff — analogous to a preflight checklist. One rename, no alias or migration wrapper needed.

Changes required:
1. Rename `.agents/skills/deslop/` → `.agents/skills/preflight/` and update `SKILL.md` front matter (`name:` field, `description:`).
2. Update all references to `deslop` in workflow docs (`.agents/TASKS.md`, agent instructions, docs).
3. Update the embedded skill definition in `internal/templates/` if the skill is referenced from templates.
4. Update any task files that mention `deslop` in their body or notes.
5. Update `docs/cli.md` and test fixtures if they reference `deslop`.
6. Update `ahm upgrade` so existing managed installs remove the old `.agents/skills/deslop/SKILL.md` entry and directory when it is still the ahm-managed copy. Do not remove a user-modified `deslop` skill silently; preserve it or report the conflict using the existing managed-file conflict behavior.
7. Update install/upgrade tests to cover the managed-file removal path.
8. Run `just ci` after the rename to verify nothing is broken.
