# Add optional task work review orchestration

## Decision

Implement review orchestration as an optional step in the task-work workflow,
built on task 056's in-process session capture/resume capability.

- Keep the task separate from session capture, completion, and commit handoff.
- Start with the `scripts/task-workflow.sh` behavior: run an independent
  preflight review, then feed the review feedback back into the original working
  session.
- `cake` is the first full-workflow implementation.
- The design must leave room for `codex`, `claude`, and `cursor` full-workflow
  implementations by keeping review invocation and session resume behavior
  provider-specific.
- If review produces no actionable feedback, skip the feedback-resume step.
- If review fails, surface the failure clearly and do not proceed as if the
  review passed.

## Summary

Explore and implement an opt-in review orchestration step for `ahm task work`.
The goal is to let a task work handoff run an independent review pass, such as
the existing preflight workflow, while preserving a clean delegation boundary to
external agents.

## Background

`scripts/task-workflow.sh` currently runs a separate `cake --no-session
--skills preflight` review after the initial work session, then feeds that review
back to the original session using the captured session ID. ADR 006
intentionally kept this out of the MVP so `ahm task work` could establish a
clear delegation boundary first.

## Provider Direction

- Implement the initial review mode for `cake` and preflight.
- Keep the interface capable of describing whether an agent supports:
  independent review invocation;
  resuming the original work session with review feedback.
- Treat `codex`, `claude`, and `cursor` as planned providers for the full
  workflow, not as unsupported forever.
