# Reject YAML block list syntax in task front matter

## Problem

`parseFrontMatter` (`tasks.go:141-180`) silently skips any front-matter line
without a colon. A hand-written task using YAML block list syntax:

    depends_on:
      - 001
      - 002

parses as `depends_on: ""` with the item lines dropped — the task silently
loses its dependencies and appears ready instead of blocked. Block scalars
(`|` and `>`) are already rejected with an explicit error; block lists are
not, even though they are at least as likely to be written by hand.

## Fix Direction

Return an explicit parse error when a front-matter line is a list item
(starts with `- ` after trimming), mirroring the block-scalar error. Document
the supported front-matter grammar (single-line values; inline `[a, b]` or
comma-separated lists) in `docs/spec.md`. Mirror the check in the migration
splitter if needed so `task migrate` reports rather than mangles such files.
