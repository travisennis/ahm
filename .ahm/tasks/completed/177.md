# Supply Cake native output schema for delegated procedures

## Problem

`ahm task groom` and `ahm audit` use `delegatedResultArgs`, but Cake currently falls through to its ordinary `--output-format stream-json` arguments without receiving the result schema. Cake 0.1.0 supports `--output-schema <path>` and validates or corrects the complete final response. Without that flag, normal agent handoff prose can surround a valid fenced verdict, causing ahm to reject the result and apply no changes.

Observed in Cake session `231c1305-2283-4591-8285-635b2267796a` while grooming task 176.

## Relevant Files

- `internal/ahm/task_groom.go` — builds schema-constrained delegated arguments shared by groom and audit.
- `internal/ahm/task_groom_test.go` — grooming argument, parsing, and application coverage.
- `docs/adr/014-replace-managed-skills-with-command-based-procedures.md` — still says Cake is expected to gain native schema support.
- `docs/references/cli/task-commands.md` — documents grooming delegation.
- `docs/guides/testing.md` — external-agent verification expectations.

## Fix Direction

Pass the result schema to Cake through a temporary JSON Schema file and `--output-schema`, while retaining `stream-json` transport and model selection. Share temporary-schema creation where practical with Codex. Keep result parsing strict; do not scrape fenced JSON from prose. Update stale documentation and add regression coverage for Cake arguments, schema-file contents and cleanup, plus realistic Cake stream output application.
