# Add env-gated live agent smoke test

## Problem

Golden fixtures (task 088) verify output parsing, but they cannot verify
argument construction — whether `codex exec resume --json <id>` exists in that
flag shape, whether `cake --no-session --skills preflight` is still valid, whether
a captured session ID is actually resumable. Those assumptions can only be
checked by running the real binaries, and nothing in the repo does. The
bf6c1ae cake regression (task 087) shipped because no real agent run happened
between implementation and use.

## Relevant Files

- `internal/ahm/task_commands.go` — `taskWorkAgent` arg builders and
  orchestration paths (`taskWorkWithSession`, `runReview`, `runCompletion`,
  `runCommit`)
- `internal/ahm/task_commands_test.go` — existing stubbed orchestration tests
- `justfile` — home for the `smoke-agents` recipe

## Implementation Notes

- Add a Go test gated on an environment variable (e.g. `AHM_AGENT_SMOKE=1`)
  that is skipped by default, surfaced as `just smoke-agents`. Keep it out of
  `just ci`.
- For each supported agent whose binary is on PATH (skip per-agent when
  missing): set up a temp git repo with an installed `.agents` workflow and a
  trivial do-nothing task, then drive the real `ahm task work` path with a
  prompt that requires no code changes. Use the cheapest model/limits each
  CLI supports.
- Assert: stderr contains "session started:"; stderr does not contain
  "no session ID returned by" or "could not capture session ID"; a resume
  step round-trips (the `--complete` handoff is the cheapest path that
  exercises `resumeArgs` against a real session).
- Print per-agent versions in test output so a failure log records what was
  exercised.
- Document the cost expectation (a few real LLM calls per agent) and when to
  run it: any change to agent arg builders, parsers, or orchestration flow,
  per the checklist in task 090.

- [x] `just smoke-agents` runs the gated test; without `AHM_AGENT_SMOKE=1`
      the test skips (verified: `go test ./internal/ahm -run TestAgentSmoke`
      reports SKIP) and `just ci` is unaffected.
- [x] Smoke test covers session capture and one resume per installed agent
      (`--complete` exercises resumeArgs against the real session), skipping
      agents not on PATH per-subtest. Live run on 2026-06-12 passed for
      cake 0.1.0 and codex-cli 0.138.0 in 51.5s: "session started:" present

- [x] Usage and cost documented in the test header comment and the justfile

      agent; run after changing arg builders, parsers, or orchestration);
      `just ci` passes.
