# Clarify and enforce the atomic write path contract

## Problem

The `writeFileAtomic` comment says it rejects path traversal, but the implementation only rejects non-canonical spellings. An already-clean `../...` or absolute path passes. The mismatch obscures the actual caller-owned safety boundary.

## Relevant Files

- `internal/ahm/write.go` — helper contract and canonical-path check.
- `internal/ahm/write_test.go` — path behavior tests.
- `docs/guardrails/safety-and-permissions.md` — path safety expectations.

## Fix Direction

Choose the intended contract: either rename/document the check accurately and make caller scoping explicit, or pass an allowed root and enforce containment. Do not imply containment that is not enforced.
