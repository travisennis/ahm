# Protect active atomic temp files from stale cleanup

## Problem

`cleanupStaleTemps` removes every matching `*.tmp` file regardless of age. Unique atomic-write temp names have no real non-`.tmp` counterpart, so cleanup can delete a temp file another process is actively writing and make its rename fail with `ENOENT`.

## Relevant Files

- `internal/ahm/write.go` — atomic writer and stale-temp cleanup.
- `internal/ahm/write_test.go` — cleanup and concurrency tests.
- `docs/adr/001-atomic-writes-and-concurrency.md` — atomic-write guarantees.

## Fix Direction

Only reclaim temps old enough to be confidently stale, with a documented threshold and conservative stat-error handling. Test cleanup racing an active writer.

## Acceptance Notes

- [x] Fresh atomic-write temp files are never removed by cleanup.
  - Verified by `TestCleanupStaleTemps_SkipsFreshTemp`.
- [x] Old orphaned temp files are still reclaimed.
  - Verified by `TestCleanupStaleTemps_RemovesOldOrphanTemp` and by aging stale `.tmp` files in existing cleanup tests.
- [x] A deterministic concurrency test covers cleanup during an active write.
  - Verified by `TestCleanupStaleTemps_RaceWithActiveWriter`, which holds a temp file open while `cleanupStaleTemps` runs.
- [x] Atomic-write documentation reflects the stale-temp policy.
  - Updated `docs/adr/001-atomic-writes-and-concurrency.md` to describe the five-minute age threshold and conservative stat-error handling.
