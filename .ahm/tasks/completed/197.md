# Keep idempotent ADR status commands from changing dates

## Problem

`adrSetStatus` rewrites `date` to today even when the ADR already has the requested status. An idempotent `ahm adr accept` therefore mutates a record while reporting a no-op transition.

## Relevant Files

- `internal/ahm/adr_commands.go` — ADR status transitions.
- `internal/ahm/adr_commands_test.go` — lifecycle and idempotency tests.
- `docs/references/cli/adr.md` — ADR command behavior.

## Fix Direction

Return a stable no-op result before computing or writing a new date when status already matches. Preserve dry-run and structured output compatibility.

## Acceptance Notes

- [x] Repeating an ADR status command leaves the file byte-for-byte unchanged; covered by `TestADRStatusCommandNoOpPreservesFileAndReportsUnchanged`.
- [x] Text, JSON, plain, and dry-run output clearly represent the no-op, including `changed: false` for structured previews.
- [x] Real status transitions still update status and date; covered by the focused ADR lifecycle tests and the full `just ci` suite.
