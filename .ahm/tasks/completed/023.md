# Make templates version immutable or build-injected

## Summary

Review issue 23. templates.Version is a mutable package variable. Acceptance: version is a const or intentionally injected via build flags, with tests adjusted and release behavior documented if build injection is used.

## Acceptance Notes

- Changed `var Version = "0.2.0"` to `const Version = "0.2.0"` in `internal/templates/templates.go`.
- No test changes needed — all references are read-only and `const` string values work identically.
- `go test ./internal/templates ./internal/ahm` passes, `just ci` passes.
- Build-injection approach was not taken; `const` is adequate since the version is maintained in-source and there's no separate release pipeline that tags differently.
