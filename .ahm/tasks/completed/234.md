# Remove vestigial template version tracking

## Summary

`internal/templates.Version` (currently `0.6.5`) is a vestigial parallel version to the ahm binary version. It was introduced when ahm installed workflow instruction files (TASKS.md, PLANS.md, etc.) to disk under `.agents/`. Those files are now embedded in the binary and delivered exclusively through `ahm context` commands — no instruction files are written to the repo anymore.

The template version:
- Is displayed in `ahm prime` output: `workflow: installed 0.6.5 (templates 0.6.5)`
- Is included in `ahm status` and `ahm doctor` JSON as `template_version`
- Is stamped into `.ahm/config.json` metadata on every `ahm upgrade`
- Is never compared to drive any upgrade behavior
- Requires a bump-and-document dance for every template change

Remove `templates.Version` and all its display/stamping. The ahm binary version (`internal/version.Version`) is sufficient.

## Scope

- [x] Remove `Version` constant from `internal/templates/templates.go`
- [x] Remove `TemplateVersion` from `contextWorkflow` struct in `internal/ahm/context.go` and `primeReport` in `internal/ahm/prime.go`
- [x] Remove `template_version` output from `ahm prime` text rendering
- [x] Remove `template_version` from `ahm status` and `ahm doctor` JSON output
- [x] Remove `meta.Version = templates.Version` stamp from `internal/ahm/install.go` upgrade flow
- [x] Simplify the metadata `version` field — fresh metadata omits it, existing values round-trip for compatibility, and `internal/version.Binary` supplies installed-version output
- [x] Clean up version-bump entries from `docs/guides/workflow-upgrades.md` that only document template version advancement
- [x] Keep `internal/templates/templates.go` because `FS embed.FS` supplies scoped `ahm context` references
- [x] All existing tests pass after removal
