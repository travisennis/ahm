# Trim and modernize AGENTS.md entry-point guidance

## Problem

`AGENTS.md` is 143 lines, near the ~150 ceiling and well past the 60–90
line target for a routing layer with focused docs. It also describes this
repository through a legacy record layout that it no longer uses: the project
summary contrasts `.agents/` repositories with migrated `.ahm/` repositories,
the compatibility list names `.agents/ahm.json`, and the workflow-state route
continues that backward-looking framing. This repository's ahm records live
only under `.ahm/`; `.agents/` is project-owned agent guidance, not an ahm
record store.

The bootstrap guidance can recreate the same problem. `ahm onboard` currently
says that ahm records are "stored as files under `.agents/` and `docs/adr/`".
That wording is stale for current repositories and conflates ahm-owned workflow
records with project-owned ADR documentation.

Product-behavior requirements have also crept into the routing layer. The
"Workflow State, File Formats, And Upgrades" route specifies how `ahm prime`
and scoped `ahm context` commands must behave. Those requirements belong in
the workflow-state guardrail or specification, not in the entry point.

## Relevant Files

- `AGENTS.md`
- `internal/ahm/onboard.go`
- `internal/ahm/onboard_test.go`
- `docs/guardrails/workflow-state-and-file-formats.md`
- `docs/references/workflow-spec.md`
- `docs/guides/workflow-upgrades.md`
- `internal/templates/templates.go` if the embedded template version must
  advance for the onboard guidance change

## Fix Direction

- Rewrite this repository's `AGENTS.md` to describe only its current state:
  ahm workflow records live under `.ahm/`; `.agents/` contains project-owned
  agent guidance. Remove the legacy-layout and migration narrative from this
  entry point without changing the product's compatibility behavior.
- Update `ahm onboard` so its paste-ready snippet gives current, unambiguous
  storage guidance. Treat `.ahm/` as the only ahm workflow-record directory
  and describe `docs/adr/` separately as project-owned durable documentation
  if ADR location remains useful in the snippet. Do not advertise `.agents/`
  as an ahm record location.
- Move the `ahm prime` and scoped `ahm context` behavior requirements from
  `AGENTS.md` into the authoritative workflow-state guardrail or specification,
  leaving the route as a short pointer.
- Scan the rest of `AGENTS.md` for similar spec-level content and relocate it
  rather than dropping durable product requirements. Target 60–90 lines and
  keep the file at or below 100 lines.
- Update focused tests and versioned workflow-upgrade documentation required
  by the onboard template change.
