# Omit duplicate top-level heading from task create body files

## Problem

When `ahm task create` is used with `--body-file`, the CLI already generates
`# <title>` from front matter via `renderTask`. If the body file also includes
a matching `# <title>` line (either because the agent authoring the body
included it, or because the body was extracted from a full task file), the
rendered task ends up with two top-level headings — one from the CLI and one
from the body.

The fix is deduplication, not removal. The body file can contain any H1 it
needs; only an exact match of the task title gets stripped to avoid duplication.

## Relevant Files

- `internal/ahm/task_commands.go` — `taskCreateParsed` calls
  `resolveTaskCreateBody` and then renders via `renderTask`
- `internal/ahm/tasks.go` — `stripHeading` already handles exact-match H1
  removal during parsing; reuse it during creation too
- `docs/cli.md` — document the dedup behavior
- Task creation tests under `internal/ahm/`

## Fix Direction

1. In `taskCreateParsed`, add `body = stripHeading(body, parsed.title)` after
   `resolveTaskCreateBody` to strip any H1 that exactly matches the task title.
   `stripHeading` already exists and handles leading blank lines, exact match,
   and trailing blank lines correctly.
2. Update `docs/cli.md` to note that a matching H1 is automatically stripped
   while a different H1 is preserved as intentional body content.
3. Add focused tests:
   - Body file with matching H1 → exactly one H1 in output.
   - Body file with matching H1 and leading blanks → exactly one H1.
   - Body file with different H1 → both H1s preserved.

## Acceptance Notes

- [x] `taskCreateParsed` strips matching H1 from body file content before rendering.
- [x] A body file containing `# <title>` matching the task title produces exactly one H1 in the rendered task.
- [x] A body file containing a different H1 preserves both H1s as intentional content.
- [x] `docs/cli.md` documents the dedup behavior.
- [x] Focused tests cover matching H1, matching H1 with leading blanks, and different H1.
