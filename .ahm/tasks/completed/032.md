# Stamp created and updated task metadata

## Summary

Review issue 32. task create does not set created and task status transitions do not update updated, despite fields being preserved. Acceptance: create and mutation commands stamp dates consistently or remove managed-date expectations, with docs and tests updated.

## Implementation Notes

- Added `Created: time.Now().Format(time.RFC3339)` in `taskCreateParsed` so new tasks get a `created` timestamp.
- Added `task.Updated = time.Now().Format(time.RFC3339)` in `taskStatus` so status transitions (start, complete, cancel, reopen) stamp `updated`.
- Added `task.Updated = time.Now().Format(time.RFC3339)` in `taskDepUpdate` so dependency edits stamp `updated`.
- `renderTask` already conditionally writes `created`/`updated` when non-empty — no changes needed there.
- `created` is set once on create and never overwritten; `updated` is overwritten on every mutation.

## Acceptance Notes

- [x] `task create` writes a valid `created: <RFC3339 timestamp>` in the created task file.
- [x] `task start`, `task complete`, `task cancel`, `task reopen` overwrite `updated` with the current timestamp.
- [x] `task dep add`, `task dep remove` overwrite `updated` with the current timestamp.
- [x] `created` is preserved unchanged during mutations.
- [x] Existing optional front matter (`parent`, `external_ref`, unknown fields) is preserved alongside stamped dates.
- [x] All existing tests pass; updated `TestTaskCreateAllowsFlagsAfterTitle`, `TestTaskStatusPreservesOptionalFrontMatter`, and `TestTaskDepUpdatePreservesOptionalFrontMatter` to verify stamping behavior.
- [x] Full CI (`just ci`) passes.
- [x] Docs (`docs/cli.md`) already correctly describe `created` and `updated` as optional preserved fields; no doc update needed.
