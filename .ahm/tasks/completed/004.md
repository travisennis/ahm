# Make task dependency cycle output deterministic

## Summary

Review issue 4. taskDependencyCycles and taskDepCycles seed DFS from Go maps, making cycle output unstable. Acceptance: cycle detection sorts IDs before traversal, command and validation output are deterministic, and tests assert one stable cycle order.

## Acceptance Notes

- [x] `taskDependencyCycles` now sorts `byID` keys before DFS traversal for deterministic output
- [x] `taskDepCycles` (CLI `task dep cycles` command) now sorts `byID` keys before DFS traversal
- [x] Cycle detection in validation (`ahm status`, `ahm doctor`) is deterministic
- [x] Tests assert one stable cycle order ("001 -> 002 -> 001") instead of accepting either permutation
