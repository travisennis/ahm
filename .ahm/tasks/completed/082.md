# Add Claude Code agent support to ahm task work

## Decision

Use Claude Code print mode with realtime JSON streaming:

- Work args: `claude -p --verbose --output-format stream-json <prompt>`.
- Resume args: `claude -p --verbose --resume <sessionID>
  --output-format stream-json <prompt>`.
- Session ID source: first `system` event with `subtype: "init"` and
  `session_id`.
- Final result source: `result` event with `result`, `is_error`, and
  `session_id`.
- Review support should be included by running a fresh Claude Code
  stream-json review prompt and parsing the final `result` event, then
  resuming the original session with that feedback.

Local verification on 2026-06-12 with Claude Code 2.1.175 confirmed
`stream-json` requires `--verbose`. The probe emitted a `system/init` event
with `session_id`, then an `assistant` event, then a final `result` event with
the same `session_id`. The probe hit an account billing error after init, but
that still confirmed the stream shape needed for session capture.

Official docs checked:

- https://code.claude.com/docs/en/cli-reference documents `-p`, `--resume`,
  `--output-format stream-json`, and the `--verbose` requirement for stream
  options.
- https://code.claude.com/docs/en/agent-sdk/overview documents session capture
  from an init system message's `session_id`.

## Problem

`ahm task work` supports `cake`, `codex`, and `cursor`, but not Claude Code.
Claude Code's CLI (`claude`) supports headless print mode with JSON output and
session resume, so it can participate in full orchestration (sessions, review,
completion handoff, commit handoff) the same way `cake` does today.

## Relevant Files

- `internal/ahm/task_commands.go` — `taskWorkAgent` struct,
  `parseTaskWorkAgent`, `taskWorkWithSession`, cake session/review parsers.
- `internal/ahm/task_commands_test.go` — agent selection, dry-run, and
  orchestration tests.
- `docs/cli.md` — `task work` supported-agents table, flags, examples, and
  agent selection precedence.
- `docs/adr/006-task-work-agent-delegation.md` — enumerates the supported
  agent set and session-capable agents.

## Fix Direction

- Add a `claude` case to `parseTaskWorkAgent`:
  - executable `claude`
  - work args `-p --verbose --output-format stream-json <prompt>`
  - resume args `-p --verbose --resume <sessionID> --output-format
    stream-json <prompt>`
- Add a JSONL parser that scans events line by line for
  `{"type":"system","subtype":"init","session_id":"..."}` and extracts final
  review feedback from the last `{"type":"result","result":"..."}` event.
- Set `supportsSessions` and `supportsReview` so `--review`, `--complete`, and
  `--commit` work without warnings.
- Update the `--agent` flag help text, the unsupported-agent error message,
  and `default_work_agent` validation surface if any.
- Update `docs/cli.md` (supported-agents table, precedence list, examples) and
  amend `docs/adr/006-task-work-agent-delegation.md` per the supersession
  workflow, since the ADR enumerates the supported agent set.

### Live-run evidence

Per the external-agent-orchestration guardrail (`area:agent` +
`risk:external-service` need live-run evidence), `claude` was run end-to-end
through `ahm task work` on 2026-06-17 with `claude 2.1.178 (Claude Code)` via
`AHM_AGENT_SMOKE=1 go test ./internal/ahm -run TestAgentSmoke/claude`.

The account hit a billing error (`Credit balance is too low`), so the run
exited non-zero before resume, but it confirmed the real output schema the
parsers depend on:

- `{"type":"system","subtype":"init",...,"session_id":"6726dbdb-...","..."}`
  — matches `parseClaudeSessionID` (type `system`, subtype `init`,
  `session_id`).
- `{"type":"result","subtype":"success","is_error":true,"result":"Credit
  balance is too low","session_id":"6726dbdb-..."}` — matches
  `parseClaudeReviewFeedback` (final `result` event's `result` field).

Both parsers were re-verified against those exact captured lines, ruling out
the "fixture matches an invented schema" failure mode. A full resume/review
round-trip could not be exercised because of the account billing limit (same
limitation noted in this task's Decision probe); re-run `just smoke-agents`
on a funded account to confirm resume.

`internal/ahm/task_work_smoke_test.go` now includes `claude` in its
session-capable agent list so future `just smoke-agents` runs exercise it.
